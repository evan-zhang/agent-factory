# 经验沉淀机制（FIP-001-R v2）

## 启动时注入

每个 sub-agent 启动时：
1. 读取 `_runtime/experience/EXPERIENCE.md` 最近 5 条
2. 在 task 中注入："工厂经验：[最近5条摘要]"
3. 不按类型过滤——所有经验对所有 sub-agent 可见

## 完成时有条件写入

满足**任一**条件时写入：
- 任务耗时 >10 分钟
- 遇到阻塞/回退/重试
- 做了否定式决策（"没选X因为Y"）
- Evan 说"记住这个"

**不写入**（过滤噪音）：
- 拼写错误/预期内验证失败
- 立刻解决的小问题
- 例行文件创建/提交

## 写入流程

1. 先读 `EXPERIENCE.md`
2. 关键词匹配检查是否已有同类
3. 有→更新已有条目；无→追加到文件顶部（最新在前）

## 写入格式

```
## [日期] [类型] [一句话摘要]
- **决策**：选了什么，没选什么，为什么
- **前车之鉴**：下次怎么做
- **上下文**：项目ID、技术栈、环境
```

类型：`[技术]` / `[流程]` / `[决策]`。每条 ≤150 字。位置：`_runtime/experience/EXPERIENCE.md`。
