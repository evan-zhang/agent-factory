## HTML转PDF技术踩坑记录（2026-05-15）

### PDF边距双倍问题
**现象**：Chrome CLI传的边距参数 + HTML @page CSS的边距叠加，导致实际边距是预期的2倍。
**根因**：HTML已有`@page { margin: 26mm 24mm 22mm 24mm }`，Chrome CLI又传了相同边距参数。
**解决**：html-to-pdf.js新增`htmlHasPageMargin()`检测——如果HTML里有@page margin，CLI边距参数跳过，只用HTML @page的边距。打印效果由HTML @page单独控制。

### HTML @page margin = 0 还是非0？
**结论**：改成 `margin: 0`（内容撑满A4整页），不再额外预留边距。
**原因**： Evan 确认，预留边距会导致白边过大，内容反而被压缩。
**操作**：所有模板骨架（style-01~06）的@page规则统一改为`margin: 0`。

### Chrome CLI参数优先级实测结论
```
Chrome CLI margin参数 + HTML @page margin → 双倍边距（错误）
Chrome CLI margin参数（HTML无@page） → CLI边距生效
HTML @page margin（无CLI参数） → HTML边距生效 ✅
```
**最终方案**：html-to-pdf.js自动检测HTML是否有@page margin，有则跳过CLI参数。
