# "Agent Harness" 开源方案调研（2026.6）

## 一、概念先校准

**Agent Harness**（最近被密集使用，但定义不统一）大致指这几类东西，按层级排：

| 层级 | 是什么 | 例子 |
|------|--------|------|
| L0 协议 | 客户端 ↔ Agent 标准协议 | Agent Protocol (e2b-dev) |
| L1 SDK | 让 Agent 跑起来的官方 SDK | Claude Agent SDK、Codex SDK |
| L2 沙箱/执行环境 | Agent 安全跑代码的地方 | E2B、Modal、Daytona |
| L3 框架 | 多 Agent 协作/工具调用框架 | LangGraph、AutoGen、CrewAI、smol-developer |

注意区别：**Claude Agent SDK ≠ Claude Code**。SDK 是给别人用 Claude Code 能力的库，Claude Code 是 Anthropic 自己的 IDE Agent。

## 二、值得认真看的几个

### 1. [e2b-dev/E2B](https://github.com/e2b-dev/E2B) — **12.6k stars**

定位：**沙箱基础设施**。"在云端安全沙箱里跑 AI 生成的代码"。

- 强项：Python + JS SDK，开箱即用，自托管有 Terraform 模板
- 适合：需要给 Agent 配"执行代码的安全环境"时
- **跟 Agent Protocol（e2b-dev 自己提的）配合，能用统一协议跟不同 Agent 通信**
- 516 个 release，最新 6.17（昨天）

### 2. [anthropics/claude-agent-sdk](https://github.com/anthropics/claude-agent-sdk) — **1.5k stars**

定位：**官方 SDK**，给开发者"用 Claude Code 能力造自己的 Agent"。

- 之前叫 "Claude Code SDK"，v0.3.x 后改名
- TypeScript 实现（也有 Python 版，独立 repo）
- 适合：要把 Claude Code 嵌入自己产品时
- 144 个 release，最新 v0.3.181（6.17）

### 3. [smol-ai/developer](https://github.com/smol-ai/developer) — 历史地位重要

定位：**第一波 prompt→整库的范式**（2023 年），整个 `smol-*` 系列（"smol developer"、"smol researcher"）的源头。

- 老牌 reference，今天不直接用但概念是"骨架"
- smol-developer 走的是"plan → specify_file_paths → generate_code_sync"三步范式
- 衍生：E2B 团队的 [Agent Protocol](https://github.com/e2b-dev/agent-protocol) 是它 API 模式的标准化

## 三、不在 GitHub 但该知道的

- **CrewAI**：角色扮演式协作
- **OpenAI Agents SDK / Codex SDK**：跟 Claude Agent SDK 正面竞争
- **Daytona**、**Modal**：E2B 同类竞品

1. **"Harness" 这个词目前没有标准定义**。Anthropic 的"Agent SDK"、E2B 的"Agent Protocol"、LangChain 的"LangGraph"，都是不同维度的"harness"。**不要被一个词绑死，先看你要解决什么**。

2. **你当前 OpenClaw 自带的"openclaw-code-agent"插件**（config 里有）≈ Claude Agent SDK 的轻量封装 + Codex SDK 桥。**已经是 harness 思路的现成实现**，没必要再引外部 harness 替代它。

3. **真正缺的不是 harness，是协议层**。Agent Protocol（e2b-dev 推）正在做 LLM 无关的"Agent 通信标准"，值得跟一下。

4. **沙箱层 E2B 是最成熟的**。如果未来 Agent 要执行用户提交的代码（不只是写代码），E2B 几乎是首选。
