## [决策] 方案 Z = 双层报告结构（确认采纳）

报告结构最终采用 Z 而非 X/Y：

- **Part 1: One-pager（1 页）** —— 决策者看完能拍板
- **Part 2: Executive Summary（3-5 页）** —— 看完能理解 95% 判断
- **Part 3: Gate 详解** —— 详细尽调
- **Part 4: 附录** —— 原始数据、参考文献

**否定方案 X（简单组装）**：原 7 章直接拼，最终报告 ≈ 章节内容之和，
高层无法快速决策。

**否定方案 Y（高级提炼）**：摘要是 LLM 即时生成的，会丢信息或误读，
原版必须保留。

**理由**：Z 既保留原文（Part 3/4），又前置摘要（Part 1/2），
投委会看 5 页、尽调看 35 页、各取所需。

## [决策] TL;DR 强制字段方案 C 改进版（确认采纳）

每章 MD 顶部强制结构：

```markdown
# G{N} {章名}

## TL;DR
评级: ...
评分: ...
关键风险: ...
推荐路径: ...
下一步: ...

---

## 详细评估
### {N}.1 {子节}
...
```

**关键 schema 字段**（chapter-schema.yaml）：

```yaml
chapters:
  - id: G2
    name: 定调门
    weight: gate
    required_tldr_keys: [评级, 临床评分, 关键风险, 推荐路径, 下一步]
```

**否定方案 A**（每次即时生成）—— 每次跑消耗 token
**否定方案 B**（单独存 _summary.md）—— 需要 watcher 检测同步

**C 改进版优势**：写一次 = 摘要原文同源；改 1 章 = 摘要自动跟着改；
无 watcher；sub-agent 写 G2 时顺手把 TL;DR 写了。

## [决策] 承接主体分析的"处方来源权重法"（A-8 样板首创）

按科室处方占比推算院内方案上限：

- 处方来源权重 = 各科室对该产品的处方占比
- 院内方案覆盖率 = Σ(各主体覆盖科室权重)
- 例：非索非那定干混悬剂 → 儿科 45-50% + 耳鼻喉 20-25% + 皮肤科 20-25%
- 方案 A（天津康哲，覆盖儿科/耳鼻喉/全科）→ 60-70% 覆盖率
- 方案 B（德镁，仅皮肤科）→ 20-30% 覆盖率
- **2-3 倍差距 = 决定承接主体选择**

v0.7.1 SOP 必须包含这个方法论。

## [经验] "回归 Spec 现状"的对照方法论

未来做 SOP 改造的**标准流程**：

1. **先读 Spec/SOP 现状**（不是先看模板）
2. **列出 Spec 已有什么**（强制完整性表）
3. **列出模板新增了什么**（与 Spec 对比）
4. **判断"模板的新增"是机制层还是样式层**（机制可抄，样式不抄）
5. **判断"实际跑 JMKX003948 时缺什么"**（实证驱动，不是模板驱动）
6. **判断"改这块对 v0.7.1 是不是可持续"**（不是一次性升级）

**反面案例**：看到 3 份样板就开列 8 项 P0 改造，不读 Spec。
**正面案例**：先读 Spec 发现 v0.7.0 相当完整，再列"实际缺什么"。
