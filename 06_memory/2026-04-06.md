## bp-auditor Critical Bug Fix (2026-04-05)

### Problem
Original flow spawned one sub-agent per downstream BP. If many downstream BPs existed, this caused timeout.

### Solution
Batch processing with progress reporting:
- Split downstream BPs into batches (max 3 per batch)
- Process one batch at a time
- Report progress after each batch completes

### Code Change (SKILL.md flow)
```markdown
## 两级联动审计流程

```
Step 1: spawn 数据获取执行者 → fetch.py 获取根 BP 数据
Step 2: spawn 分析执行者 → 对根 BP 执行 Level 1 评估（Goal + KR + 举措）
Step 3: 识别下游 BP 列表
Step 4: **分批处理下游 BP**（每批最多 3 个）
  - 每批 spawn 分析执行者执行 Level 2 评估
  - 每批完成后汇报进度
Step 5: 汇总所有 P0/P1/P2 问题
Step 6: 生成报告 → 发给 Evan
```
```

## ClawHub CLI Limitations (2026-04-05)

### Authentication Issue
`clawhub login` requires browser callback to localhost, which fails on:
1. **VPS without browser access** - Cannot open browser at all
2. **Mobile devices** - Browser opens but cannot redirect to localhost (mobile has no server listening)
3. **Headless environments** - No graphical interface

### Error Encountered
```
Error: {"code":"[Request ID: 1dde7fcb84131716] Server Error","data":"Unauthorized"}
```
Repeated rate limit and unauthorized errors when trying to publish from VPS.

### Workarounds
1. **Run `clawhub login` on a machine with browser access** (Mac mini, laptop, etc.)
2. **Copy token** to VPS and configure manually (if supported)
3. **Wait for ClawHub CLI fix** to support `--token` parameter or Web UI token generation

### Publishing Workflow Reminder
Complete skill publishing requires:
1. Git commit and push to GitHub
2. `clawhub publish` to ClawHub registry
3. **Write release notes** (what changed, install command, feedback channel)
4. **Send release notification to Evan** for distribution to other agents

### Pre-publish Validation (Critical)
Before publishing, verify in **clean directory**:
```bash

clawhub install <skill-name>
ls -la <skill-name>/
```
