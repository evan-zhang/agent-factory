## 三、详细分析

### 问题B：GRV章节内容深度严重不足

**现象**：
- 当前 BD-Eval 流水线产出的 GRV 章节，每章约 300-500 字
- 同品种参考报告，每章约 3,000-5,000 字（含大量表格、具体数据、竞品对比）
- 内容差距约 **6-10倍**

**示例：G-3竞争者数量预测章节**

当前版本（500字）：
> 已获批产品≥8款：修美乐、百奥泰格乐立、信达苏迈欣、复宏汉霖汉达远、海正安健宁、君实君迈康、神州细胞尚贾、国药集团。🔴在审品种未核实。

参考报告版本（3,000字）：
> 含完整表格：7款获批产品逐一列出商品名、企业、获批时间、适应症数；详细分析竞争格局、市场份额变化趋势；附市场规模数字和集采影响预测。

**根本原因**：
1. Phase 3 GRV 子Agent 在超时压力下大幅压缩章节内容
2. SKILL.md 中没有对章节设置最低字数要求
3. 子Agent prompt 没有明确要求展开每个维度

**修复建议**：
1. SKILL.md 中明确要求"每章≥2000字"（正文内容）
2. Phase 3 改用分段执行（每3章一组，避免超时截断）
3. 子Agent prompt 增加质量标准说明：
   ```
   每章必须包含：
   - 至少3个数据表格（关键数字、对比矩阵、历史趋势）
   - 至少5个具体数据点（含数字、来源、时间）
   - 每个核心论点都有正向和反向两面
   ```

### 问题C：Battle承诺修正未执行，最终报告仍含错误数据

**现象**：
- Phase 4 Battle 中，审查层提出异议1（财务模型量价数学矛盾）
- 执行层接受异议，承诺"重新校准三情景的量价关系"
- 但 Phase 5 报告合并立即执行，合并时**没有检查修正是否完成**
- 最终报告（04-final-report.md）中三情景数字仍是错误版本
- Battle汇总中写了"⚠️ [重要声明]"标注，但正文数字没有修正

**根本原因**：
SOP.md 的 Phase 5 合并规范没有设置前置检查：
```
当前流程：
  Battle R1完成 → 立即进入 Phase5 合并
正确流程：
  Battle R1完成 → 检查 battle/BATTLE-R1-EXECUTOR.md 中"修正承诺" →
  逐条确认修正已写入对应 GRV 章节 → 才进入 Phase5 合并
```

**修复建议**：
1. SOP.md Phase 5 增加前置条件检查：
   ```
   Phase 5 启动条件：
   ✅ 所有 GRV 章节文件存在（state.json 验证）
   ✅ Battle 执行层的"修正承诺"已逐条确认写入对应章节
   ✅ 无未关闭的 TODO/FIXME 标记
   ```
2. Phase 5 第一步：主Agent 读取 battle/BATTLE-R1-EXECUTOR.md，逐条核对修正是否已执行

### 问题D：标的方"定制"定义缺失，导致所有假设脱离实际

**现象**：
参考报告在 B-9（综合结论）章节特别指出：

> 🔴 **异议5（极高风险）**：缺少标的方画像——不知道在评估谁，所有假设基于"行业平均"，而非标的方的实际能力。

参考报告还提出关键问题：
1. "定制"的具体含义是什么？（自研？CMO生产？License-in？）
2. 标的方的核心优势是什么？
3. 标的方是否有现有产能（2,000L反应器）？

当前 BD-Eval 流程直接开始执行，根本没有向用户确认这些信息。

**根本原因**：
Phase 1 DISCOVERY 的执行步骤中，缺少"标的方画像确认"节点：
```
当前：
  用户输入品种 → 立即开始 web_search → 输出 01-discovery.md

正确：
  用户输入品种 → 询问标的方情况 → 用户回复 → 开始搜索
```

**修复建议**：
1. Phase 1 增加"标的方画像确认"步骤，在开始前询问：
   - 标的方是甲方自有研发还是外部引进？
   - 标的方是否有现有生物药产能？（有/无/计划中）
   - "定制"的具体含义？
2. 新增文件：`00-target-profile.md`（标的方画像），作为第0章写入报告
3. 财务模型（G-6）必须基于标的方实际数据，不得以"行业平均"替代

---

### 问题E：差异化能力全为"待核实"却支撑乐观IRR

**现象**：
- G-8 差异化包装策略章节中，5项差异化能力（S1-S5）全部标注"⚠️ 待核实"
- 但 G-6 财务模型中，乐观IRR（18-25%）完全依赖这5项差异化假设
- 这是"用假设支撑假设"——乐观IRR实际上是一个没有被验证的结论

参考报告的处理方式：
> 审查层主动将乐观IRR降权，并在报告中明确说明"乐观情景已被自身数据否定，但仍保留供决策者参考"。

当前 BD-Eval 的处理：
> 直接在报告中保留乐观IRR 18-25%，没有置信度标注，也没有对假设进行降权处理。

**根本原因**：
GRV 子Agent 在撰写 G-8 时，只是列出了"潜在差异化方向"，没有评估每项能力的置信度，也没有要求对"待核实"项目进行降权处理。

**修复建议**：
1. G-8 章节规范增加"差异化能力置信度评估"维度：
   ```
   对每项差异化策略，评估：
   - 置信度：已确认 / 初步可行 / 待核实 / 不可行
   - 证据等级：高（合同/已有产品）/ 中（有参考案例）/ 低（仅假设）
   - 对IRR的影响：乐观+5% / 基准 / 悲观-3%
   ```
2. G-6 财务模型规范增加"假设降权规则"：
   ```
   如果某情景的支撑假设中"待核实"项目超过50%，
   则该情景IRR应标注为"⚠️ 低置信度"，不做主要结论依据
   ```
3. Battle 审查层应将"乐观IRR依赖未验证假设"列为优先异议

## 四、修复优先级建议

| 优先级 | 问题 | 理由 |
|---|---|---|
| **P0** | C（Battle修正未执行） | 直接导致最终报告含错误数据，比没有报告更危险 |
| **P1** | D（标的方定义缺失） | 根本性方向错误，基于错误前提的所有分析都是无效的 |
| **P2** | B（内容深度不足） | 影响报告使用价值，但有参考报告可供比对 |
| **P3** | E（乐观IRR未降权） | 已有CONDITIONAL裁定标注，属于边界情况 |
