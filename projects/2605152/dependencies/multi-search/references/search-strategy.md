# 检索策略：三轮递进

## 核心原则

搜索不是一次性的——一条查不到不代表没有。三轮递进确保最大覆盖率。

## 第一轮：精准查询

### 模式
```
site:{目标域名} {关键词1} {关键词2} {年份}
```

### 要点
- 使用 `site:` 限定到具体官方域名
- 每个指标至少构造 2 条不同查询
- 适合已知目标网站、需要精准定位的场景
- 如果不知道具体域名，跳过 site: 用裸关键词

### 示例
```
site:lzsbx.lanzhou.gov.cn 异地就医 报销比例 2026
site:lanzhou.gov.cn 医保 异地就医 直接结算 2026
```

## 第二轮：泛搜补充

### 模式
```
{城市/主体} {关键词} {年份}
{城市/主体} {别名/简称} {关键词}
{省份/上级} {部门名} {关键词}
```

### 要点
- 去掉 `site:` 限定，扩大搜索范围
- 加入城市别名、省份简称、部门全称/简称
- 适合第一轮查不到或结果不完整的场景
- 可尝试去掉年份，搜最新政策

### 示例
```
兰州 异地就医 政策 2026
兰州 跨省异地就医 报销比例
甘医保 异地就医 直接结算
兰州市医保局 异地就医 2026
```

## 第三轮：兜底来源

### 模式
```
site:{辅助来源域名} {关键词}
{城市}本地宝 {关键词}
```

### 要点
- 辅助来源（本地宝等）作为最后兜底
- **必须在来源明细中标注来源类型**（官方 A / 辅助 B / 线索 C）
- 辅助来源只作为线索和辅助，不作为最终政策依据
- 适合官方来源确实不存在的场景

### 示例
```
site:lanzhou.liebiao.com 异地就医 2026
兰州本地宝 异地就医 报销
```

## 降级规则

| 规则 | 说明 |
|------|------|
| 轮次递进 | 第一轮查不到 → 第二轮 → 第三轮 |
| 每轮至少 2 条 | 每个指标每轮至少 2 条不同查询 |
| 引擎切换时从头开始 | 切换搜索引擎后，从第一轮重新开始 |
| 记录所有尝试 | 每条查询都要记录，无论是否有结果 |
| 不放弃过早 | 一条查不到不直接放弃，试完三轮再标记缺口 |

## 搜索引擎 × 检索轮次矩阵

| | MiniMax | Tavily | Exa |
|---|---|---|---|
| 第一轮精准 | ✅ 主力 | ✅ 回退 | ⚠️ includeDomains |
| 第二轮泛搜 | ✅ 主力 | ✅ 回退 | ⚠️ 按需 |
| 第三轮兜底 | ✅ 可用 | ✅ 可用 | ❌ 不适合 |

Exa 最适合第一轮精准查询（includeDomains 限定域名），不适合泛搜和兜底。
