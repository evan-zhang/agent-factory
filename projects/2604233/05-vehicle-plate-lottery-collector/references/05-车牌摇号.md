# 05-车牌摇号

本文件是“车牌摇号”模块的内置模块文档与字段约定，用于帮助 skill 理解常见渠道结构、新能源与燃油车区分、摇号与上牌资格区分，以及社保年限映射。

真实执行时，本文件就是默认模块文档。执行代理必须先读本文件，并按本文件的渠道模板、标签和指标映射进行检索。这里的占位符用于生成目标城市查询，不是已采集来源。

## 数据采集要求

- 采集内容必须来自已打开的官方页面，搜索结果摘要不能当作官方原文
- 搜索工具按 EXECUTION-PROMPT Phase 0 探测结果选择（优先 MiniMax web_search）
- 实际使用的搜索工具和抓取工具，须在来源明细的备注中记录
- 遇到 JS 渲染页面无法采集时，在缺口表中标注原因（JS渲染失败）
- 搜索不到目标时，按检索词模板的三轮策略递进：第一轮 site: 精准 → 第二轮泛搜补充 → 第三轮本地宝兜底
- 泛搜时加入城市别名/简称、{省份简称}+部门名（如"甘医保"）、政策文号等变体
- 每轮每个指标至少尝试 2 条不同查询，不要一条查不到就放弃
- 使用本文件中的渠道模板、渠道顺序和检索词模板构造查询
- `目标城市` 和 `年份/时间范围` 必须来自用户输入；搜索查询、来源明细和缺口记录都必须保留该年份口径
- 搜索工具只用于发现和定位页面；采集内容必须来自已打开页面
- 搜索结果摘要、片段或未打开页面不能当作官方原文
- 候选 URL 必须实际打开并验证可读；404、错误页、空白页、登录拦截页或不可读页面不能作为来源证据
- 采用来源时，`02-来源明细表.md` 的 `备注` 必须记录 URL 验证结果，以及 搜索结果标题/摘要/排名

## 与 SKILL.md 的配合方式

- 本 reference 不单独执行，必须配合上级 `SKILL.md` 使用
- `SKILL.md` 决定固定指标、输出文件、来源字段和追溯规则；本文件提供内置渠道模板、模块标签、关键词和指标映射
- 执行时先读本文件，再根据目标城市替换占位符并使用搜索工具定位真实官方页面
- 如用户额外提供城市材料，只能作为补充线索；不得替代本文件的渠道顺序、固定指标和输出约束

## 检索与存储提醒

- 对每个指标至少尝试使用对应关键词进行检索；找到相关页面后必须打开页面再采集
- 采用的每个页面都要写入 `02-来源明细表.md`，并保留完整 URL、URL 验证结果、检索关键词、搜索结果标题/摘要/排名、原文摘录和提取数据
- 搜索结果 URL 如无法打开，只能写入 `03-缺口与待补充.md` 的备注，记录 URL、搜索摘要和失败原因；不得作为来源证据
- 没有找到明确依据的指标，要在 `03-缺口与待补充.md` 写明已检索渠道、关键词和当前边界
- 本 skill 只做搜索和写入文件，不做政策裁决

## 1. 文件用途

用于当输入 `{城市} + 车牌摇号 + 年份 + 标签` 时，检索小客车指标申请、摇号、竞价、新能源指标和配置公告。

## 2. 本文件结构

本文件作为“车牌摇号”内置模块文档，包含以下几段：

1. 文件用途
2. 渠道清单
3. 标签库
4. 检索词模板
5. 输出要求

## 5. 指标与关键词映射

| 指标编号 | 采集指标 | 优先关键词 | 重点抽取内容 |
|---|---|---|---|
| 1 | 电动车牌是否需要社保 | 新能源指标、纯电、插电混动、增程式、社保 | 必须区分新能源指标、上牌资格、以及纯电/插混/增程是否区分 |
| 2 | 燃油车车牌是否需要社保 | 普通小客车、燃油车指标、摇号、竞价、社保 | 必须区分普通小客车指标、摇号资格和上牌资格 |
| 3 | 社保缴纳年限要求（多少年） | 连续缴纳、累计缴纳、社保年限、居住证 | 明确连续或累计、多少年/月、是否要求最近连续不断缴 |

## 6. 对象与车辆类型拆分提醒

遇到以下对象或情形时必须分别记录，不要混写：

- 本地户籍
- 非本地户籍
- 持居住证人员
- 个人申请
- 家庭申请
- 单位申请
- 新能源指标
- 普通小客车指标
- 纯电
- 插电混动
- 增程式
- 燃油车

## 7. 检索词模板

### 第一轮：精准查询（site: 限定）
```text
site:{交通主管部门域名} {城市} 小客车指标 {年份}
site:{指标系统域名} 指标申请 摇号 竞价
site:{指标系统域名} 配置公告 {年份}
site:{市政府域名} {城市} 小客车增量指标
```

### 第二轮：泛搜补充（无 site: 限定）
```text
{城市} 车牌摇号 政策 {年份}
{城市} 小客车指标 申请条件 {年份}
{城市} 车牌 竞价 摇号 最新
{城市} 机动车增量指标 {年份}
{城市} 摇号 中签率 {年份}
```

### 第三轮：降级兜底
```text
site:{本地宝域名} {城市} 车牌摇号 {年份}
{城市}本地宝 车牌 摇号
```
## 8. 输出要求

- 必须区分 `新能源指标`、`普通小客车指标`、`摇号资格`、`上牌资格`
- 若城市同时存在 `摇号` 和 `竞价` 路径，必须分别记录，不能只按摇号处理
- 若新能源车辆类型未见明确区分，也要明确写 `当前未见区分`
- 若页面只写申请流程而未写社保条件，也要保留为有效来源，但备注要写清边界
