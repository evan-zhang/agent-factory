# 06-子女上学

本文件是“子女上学”模块的内置模块文档与字段约定，用于帮助 skill 理解常见渠道结构、义务教育与中高考区分、区级差异、积分入学条件和时间节点要求。

真实执行时，本文件就是默认模块文档。执行代理必须先读本文件，并按本文件的渠道模板、标签和指标映射进行检索。这里的占位符用于生成目标城市查询，不是已采集来源。

## 数据采集要求

- 采集内容必须来自已打开的官方页面，搜索结果摘要不能当作官方原文
- 搜索工具按 EXECUTION-PROMPT Phase 0 探测结果选择（优先 MiniMax web_search）
- 实际使用的搜索工具和抓取工具，须在来源明细的备注中记录
- 遇到 JS 渲染页面无法采集时，在缺口表中标注原因（JS渲染失败）
- 搜索不到目标时，先尝试不同关键词组合和 site: 限定，再标记缺口
- 使用本文件中的渠道模板、渠道顺序和检索词模板构造查询
- `目标城市` 和 `年份/时间范围` 必须来自用户输入；搜索查询、来源明细和缺口记录都必须保留该年份口径
- 搜索工具只用于发现和定位页面；采集内容必须来自已打开页面
- 搜索结果摘要、片段或未打开页面不能当作官方原文
- 候选 URL 必须实际打开并验证可读；404、错误页、空白页、登录拦截页或不可读页面不能作为来源证据
- 采用来源时，`02-来源明细表.md` 的 `备注` 必须记录 URL 验证结果，以及 搜索结果标题/摘要/排名

## 与 SKILL.md 的配合方式

- 本 reference 不单独执行，必须配合上级 `SKILL.md` 使用
- `SKILL.md` 决定固定指标、输出文件、来源字段和追溯规则；本文件提供内置渠道模板、模块标签、关键词和指标映射
- 执行时先读本文件，再根据目标城市替换占位符并使用搜索工具定位真实官方页面
- 如用户额外提供城市材料，只能作为补充线索；不得替代本文件的渠道顺序、固定指标和输出约束

## 检索与存储提醒

- 对每个指标至少尝试使用对应关键词进行检索；找到相关页面后必须打开页面再采集
- 采用的每个页面都要写入 `02-来源明细表.md`，并保留完整 URL、URL 验证结果、检索关键词、搜索结果标题/摘要/排名、原文摘录和提取数据
- 搜索结果 URL 如无法打开，只能写入 `03-缺口与待补充.md` 的备注，记录 URL、搜索摘要和失败原因；不得作为来源证据
- 没有找到明确依据的指标，要在 `03-缺口与待补充.md` 写明已检索渠道、关键词和当前边界
- 本 skill 只做搜索和写入文件，不做政策裁决

## 5. 指标与关键词映射

| 指标编号 | 采集指标 | 优先关键词 | 重点抽取内容 |
|---|---|---|---|
| 1 | 非本地户籍入学是否要社保（硬性门槛 vs 加分项） | 非户籍、随迁子女、社保、积分入学 | 区分硬性门槛、积分项、推荐条件 |
| 2 | 社保缴纳年限要求（多少年/个月） | 连续缴纳、累计缴纳、父母一方、截至时间 | 明确连续或累计、父母单方或双方、截止到哪个时点 |
| 3 | 居住证是否必须（父母单方/双方） | 居住证、有效居住证、居住登记 | 父母单方/双方是否必须、儿童本人是否需证件 |
| 4 | 各区/街道执行标准差异 | 区教育局、区招生简章、街道执行、积分办法 | 市级原则与区级细则的差异、具体区街道规则 |
| 5 | 房产 vs 租房在入学积分上的差异 | 房产、租房、租赁备案、学位锁定 | 是否都可申请、积分差异、排位差异、附加规则 |
| 6 | 中考是否有社保要求及年限 | 中考报名、社保、学籍、监护人 | 是学籍要求还是监护人社保要求，或多条件并行 |
| 7 | 高考是否有社保要求及年限 | 高考报名、随迁子女高考、社保年限 | 户籍/居住证/学籍/社保年限共同要求 |
| 8 | 是否有上学时间节点要求（如：报名前须已在缴） | 信息采集、报名、截止日期、截至报名月 | 明确节点到某月某日或报名月 |
| 9 | 优质学校对社保年限是否有梯度要求 | 优质学校、学位紧张、排序办法、积分梯度 | 只记录公开积分和排序办法，不推测隐含门槛 |

## 7. 检索词模板

```text
site:{教育局域名} {城市} 义务教育入学 {年份}
site:{教育局域名} {城市} 招生入学 非户籍 {年份}
site:{市政府域名} {城市} 随迁子女 入学 {年份}
{区名} {年份} 学位申请 积分入学 居住证 社保

```

## 8. 输出要求

- 必须区分 `义务教育入学`、`中考报名资格`、`高考报名资格`
- 必须把 `是否要社保` 写清是硬性门槛、积分项还是推荐条件
- 对存在明显区级差异的城市，若未给出区名，也要明确写 `需补区级政策`
- 若页面只写报名流程而未写资格条件，也要保留为有效来源，但备注要写清边界
