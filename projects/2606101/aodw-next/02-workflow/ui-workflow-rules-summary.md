## 核心检查点

### UI 任务必须
- 用户确认后再执行
- 明确任务范围和验收标准
- 记录关键决策和理由

### 代码规范
- 遵循 `03-standards/stacks/` 中的 UI 规范
- 使用对应的 UI Kit 组件
- 遵循无障碍设计原则

### Git 规范
- 遵循 `01-core/git-discipline.md`
- UI 任务使用 feature 分支
