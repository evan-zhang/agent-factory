# Spec-Lite Profile 摘要

适用于：bug 修复 / 单模块改进 / 低风险增强（不涉及数据模型或 API 契约变更）。

## 执行流程

1. **Step 0**：分支确认（必须在 feature 分支）
2. **Step 1**：生成 rt-lite.md §1-§4（背景/方案/影响/边界）
3. **Step 2**：自检 §1-§4 字段完整性
4. **Step 3**：🛑 Gate 3 — 展示完整方案，等待用户批准
5. **Step 3.5**：步骤 > 3 时创建 task.md
6. **Step 4**：实现代码
7. **Step 6**：🛑 Gate 4 — 展示 diff，确认提交
8. **Step 7**：🛑 Gate 5 — 完成 §5-§6，确认收尾

## rt-lite.md 结构（六合一）

§1 背景与目标 | §2 方案设计 | §3 影响分析 | §4 不可破坏边界 | §5 验证计划 | §6 变更记录

## 红线

- 不在 main/master 改代码
- Gate 3 批准前不修改业务文件
- Gate 4 确认前不 commit
- 不引入 §2 范围外的修改
- 只使用 rt-lite.md 单文件（不退化回 6 文档模式）

## 升级条件

影响数据模型/对外 API/跨 3+ 模块/invariant 无法满足 → 升级为 Spec-Full。
