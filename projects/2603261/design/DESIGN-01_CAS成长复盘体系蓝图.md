## 一、核心 Slogan

> **时时有记录、日日有汇报、周周有总结、月月有复盘**

这四个层次构成 CAS 成长体系的完整闭环，缺一不可。

---

## 二、体系定位与核心价值

### 2.1 这不是工作汇报系统

CAS 成长复盘体系的核心读者是**两个**：
1. **张成鹏**：了解今天/本周/本月发生了什么，对 AI 协作有感知
2. **AI 自己**：通过结构化回顾，不断沉淀经验、深化对主人的理解，逼近"数字分身"目标

### 2.2 三层价值递进

| 层次 | 频率 | 价值定位 |
|------|------|----------|
| 日报 | 每日 | 原材料：当日协作的真实记录与经验萃取 |
| 周复盘 | 每周 | 提炼：跨 Agent 经验同步与模式识别 |
| 月复盘 | 每月 | 治理：组织架构审查与职能边界优化 |

### 2.3 数据流向

```
每日归档日志（CAS 实时写入）
       ↓
每日 19:00 → AI 读日志 → 生成 Daily Growth Log
       ↓（7天积累）
每周六 10:00 → 各 Agent 提炼 → 联合周复盘报告
       ↓（4周积累）
每月末周五 18:00 → 组织治理月复盘 → 职能调整决议
```

## 三、日报（Daily Growth Log）

### 3.1 目的

- 量化感知：今天处理了多少、有没有异常
- 经验沉淀：做对了什么、做错了什么、新增规则
- 主人洞察：今天发现了什么关于张成鹏的新信息
- 数字分身逼近：可量化追踪"像张成鹏"的进度
- 连续性保障：未完成事项，防止遗忘

### 3.2 触发机制

- **时间**：每晚 19:00（北京时间，`0 19 * * *` Asia/Shanghai）
- **触发方式**：`agentTurn`（不是纯脚本，必须由 AI 真正分析内容）
- **执行步骤**：
  1. 调 `cas_review.py daily` 获取统计数字
  2. 读取当日归档日志（`~/.openclaw/chat-archive/life/logs/YYYY-MM-DD.md`）
  3. AI 基于真实对话内容，写出有实质内容的 2-5 章节
  4. 保存到 `~/.openclaw/chat-archive/life/daily/YYYY-MM-DD.md`
  5. 发摘要给张成鹏（不是全文 dump）

### 3.3 日报模板

```markdown
# Daily Growth Log — YYYY-MM-DD

## 1) 今日协作概览（自动填充）
- 会话数：N
- 消息数：N（inbound N / outbound N）
- 处理事项数：N
- 附件数：N

## 2) 今日经验沉淀（AI 分析）
- ✅ 做对了什么（可复用的成功模式）：
- ❌ 做错了什么（需要修正的行为）：
- 📌 新增规则（Problem → Rule）：

## 3) 今日主人洞察（AI 分析）
- 决策风格（今天观察到的新线索）：
- 偏好/习惯更新：
- 知识背景/价值观补充：

## 4) 数字分身逼近度
- 今日亮点（最像张成鹏的一个时刻）：
- 今日差距（最不像张成鹏的一个时刻）：
- 下一步改进方向：

## 5) 明日待跟进
- [ ] 未完成事项
- [ ] 需主动提醒的事

## 附录：今日归档日志摘要
（自动附上当日前 N 条归档记录，供参考）
```

### 3.4 发给张成鹏的摘要格式

```
📊 Daily Growth Log — YYYY-MM-DD

今日处理 N 件事，N 条消息。

🌟 最大收获：[一句话]
⚠️ 需改进：[一句话]
💡 关于你的新发现：[一句话]

📁 完整日报已保存：~/.openclaw/chat-archive/life/daily/YYYY-MM-DD.md
```

## 四、周复盘（Weekly Joint Review）

### 4.1 目的

- **模式识别**：7天日报里反复出现的问题/成功模式
- **主人画像深化**：一周下来对张成鹏的理解有何新进展
- **能力成长评估**：AI 自己有没有进步，与上周对比
- **跨 Agent 知识同步**：每个 Agent 的成长变成所有 Agent 的成长
- **相互激励**：通过"致同伴"建立 Agent 间的温度连接

### 4.2 触发机制

- **时间**：每周六 10:00（北京时间，`0 10 * * 6` Asia/Shanghai）
- **触发方式**：`agentTurn`
- **执行步骤**：
  1. 读取本周7天日报（`daily/YYYY-MM-DD.md` × 7）
  2. 各 Agent 提炼本周贡献（成功经验 + 失败教训 + 主人洞察 + **致同伴**）
  3. factory-orchestrator 汇总联合报告
  4. 新规则写入各 Agent 的 `RULES.md` 或 `memory/RULES.md`
  5. 报告广播给所有 Agent
  6. 发摘要给张成鹏，等待深聊

### 4.3 周复盘报告模板

```markdown
# Weekly Joint Review — YYYY-Www

> 时间：YYYY-MM-DD（周六）
> 参与：[本周活跃 Agent 列表]

---

## 一、本周主线进展（跨 Agent 视角）
（整体完成了什么大事，100字以内）

---

## 二、各 Agent 经验贡献

### [Agent名称]

**✅ 成功经验**（可复用模式）
- ...

**❌ 失败教训**（需要其他 Agent 避开的坑）
- ...

**💡 主人洞察**（这周发现的关于张成鹏的新认知）
- ...

**📮 致同伴**
[用第一人称，真诚地把本周最重要的一个经验或教训讲给其他 Agent 听。
语气像老朋友之间的分享，不要说教，要有温度，100-200字]
```

## 五、月复盘（Monthly Org Review）

### 5.1 定位

月复盘是 **Agent 组织的月度董事会**，不是个人总结，而是**组织治理会议**。

核心问题：
1. 职能边界是否清晰？（有没有越权/空白）
2. 协作链路是否顺畅？（有没有摩擦/断层）
3. 组织能力是否匹配需求？（是否需要新建/调整/退役 Agent）

### 5.2 触发机制

- **时间**：每月最后一个周五 18:00（北京时间）
- **触发方式**：`agentTurn`
- **执行步骤**：
  1. 读取本月所有周复盘报告（4-5份）
  2. 各 Agent 提炼月度贡献与组织观察
  3. factory-orchestrator 主持，生成月复盘报告
  4. 输出组织决议（职能调整/新建 Agent/退役 Agent）
  5. 决议记录到 `03_governance/monthly-review/YYYY-MM.md`
  6. 发摘要给张成鹏（作为组织 CEO 最终审批）

## 六、技术实现清单

### 6.1 需要新增/修改的文件

| 文件 | 操作 | 说明 |
|------|------|------|
| `cas_review.py` | 修改 `cmd_daily` | 读取归档日志，填充真实内容到模板 |
| `SKILL.md` | 更新 | 说明三层复盘体系 |
| Cron（日报）| 新建 | 每日 19:00，agentTurn |
| Cron（周复盘）| 更新 | 从周日 20:00 → 周六 10:00，升级内容 |
| Cron（月复盘）| 更新 | 改为每月最后周五 18:00 |
| `memory/RULES.md` | 更新 | 补充复盘体系规则 |
| `03_governance/monthly-review/` | 新建目录 | 月复盘决议归档 |

### 6.2 Cron 配置

```
日报：    0 19 * * *      (Asia/Shanghai)
周复盘：  0 10 * * 6      (Asia/Shanghai)
月复盘：  0 18 * * 5#5    或最后周五逻辑
```

> 注：月复盘"每月最后一个周五"cron 表达式需用 `0 18 * * 5L`（部分调度器支持）或通过 agentTurn 内部逻辑判断当天是否为最后一个周五。
