# 概念结构化模板

## 用途
在直接模式 Step 1 中，将用户提供的抽象概念按此模板结构化，为后续故事生成提供输入。

## 模板结构

```yaml
概念名称: [概念的正式名称]
核心定义: [一句话定义，不超过 30 字]
关键特征:
  - 特征 1: [描述]
  - 特征 2: [描述]
  - 特征 3: [描述]
  - 特征 4: [描述]（可选）
  - 特征 5: [描述]（可选）
日常生活类比:
  - 类比 1: [用日常现象解释]
  - 类比 2: [另一个角度]（可选）
专业领域: [所属学科或领域]
难度等级: [入门 / 进阶 / 研究生]
隐喻潜力: [高 / 中 / 低]（概念本身是否适合隐喻化表达）
```

## 格式示例

```yaml
概念名称: 博弈论中的纳什均衡
核心定义: 所有参与者在他人策略不变时，无人能通过单方面改变策略获益的状态
关键特征:
  - 特征 1: 多方参与，每方都有策略选择
  - 特征 2: 结果取决于所有人的选择组合
  - 特征 3: 均衡状态是"没人想动"的稳定点
  - 特征 4: 可能存在多个均衡点
  - 特征 5: 不一定是最优结果（囚徒困境）
日常生活类比:
  - 类比 1: 几个朋友选餐厅，每人都有偏好，但最终找到一个"谁都不想换"的选择
  - 类比 2: 堵车时每辆车都选了自己认为最快的路，结果所有路一样慢
专业领域: 博弈论 / 经济学
难度等级: 进阶
隐喻潜力: 高
```

## 使用说明

1. 接收到用户概念后，Agent 按此模板填充
2. 核心定义必须精炼，不超过 30 字
3. 关键特征至少 3 个，最多 5 个
4. 难度等级影响故事复杂度：
   - 入门：单一隐喻线索，线性叙事
   - 进阶：多层隐喻线索，非线性叙事
   - 研究生：复杂系统隐喻，多重转折
5. 结构化结果直接传入 story-template.md 作为输入
