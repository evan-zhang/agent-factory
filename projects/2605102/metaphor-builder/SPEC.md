# SPEC.md — Metaphor Builder 完整规格说明

## 1. 概述

Metaphor Builder 是一个将抽象概念转化为隐喻故事的 Skill。它不直接解释概念，而是通过构造完整的世界观和故事结构，让概念"自己浮现"出来。

### 1.1 目标
- 5 分钟让读者理解一个概念
- 故事本身必须成立，即使不知道概念也能读
- 输出为可分享的 HTML 页面

### 1.2 两种运行模式

| 模式 | 触发条件 | 流程 |
|------|----------|------|
| 直接模式 | 用户指定了具体概念 | 概念结构化 → 故事生成 → HTML 页面 |
| 推荐模式 | 用户未指定具体概念 | 概念推荐（3选1）→ 用户选择 → 直接模式 |

## 3. 直接模式详细流程

### 3.1 Step 1：概念结构化
- 读取 `references/concept-template.md`
- 将用户提供的概念按模板结构化：
  - 概念名称
  - 核心定义（一句话）
  - 关键特征（3-5 个）
  - 日常生活类比（1-2 个）
  - 专业领域与难度等级

### 3.2 Step 2：故事生成
- 读取 `references/story-template.md`
- 基于结构化概念，按框架生成 1500-2500 字故事
- 故事必须满足核心提示词中的 6 个要求
- 关键约束：
  - 不要提前暴露概念名
  - 不要写成童话
  - 故事必须有完整的叙事弧（起因→发展→高潮→结局）

### 3.3 Step 3：三层解释
- 读取 `references/interpretation-template.md`
- 在故事结尾处生成三层解释：
  1. 概念本身定义
  2. 隐喻映射（故事元素 ↔ 概念元素）
  3. 理论对应（故事机制 ↔ 现实理论）

### 3.4 Step 4：HTML 页面生成
- 读取 `templates/metaphor-page/` 目录下的设计规范和骨架
- 将故事内容和解释填入骨架 HTML
- 页面结构：
  1. 封面区：标题 + 世界观一句话
  2. 故事区：分段叙事，带场景分隔
  3. 揭晓区：概念名 + 核心定义
  4. 解释区：三层解释卡片
  5. 尾部：来源标注 + 分享提示
- 调用 doc-viewer Skill 上传 HTML，返回可分享链接

## 4. 推荐模式详细流程

### 4.1 方式 A：基于 Obsessions 笔记
- 读取 `config.json` 中的 `vault_path`
- 在指定路径下搜索 Obsessions 笔记
- 提取其中的核心关注点、反复出现的主题
- 筛选出最适合隐喻化的 3 个概念

### 4.2 方式 B：基于 Agent 上下文
- 当 `vault_path` 未配置时自动降级
- 基于当前对话历史、用户画像、近期交互主题
- 推断用户可能感兴趣的概念领域
- 选出 3 个最有隐喻潜力的概念

### 4.3 推荐呈现格式
向用户展示 3 个选项，每个包含：
- 概念名（不剧透隐喻设计）
- 一句话说明为什么选这个
- 难度标签（入门 / 进阶 / 研究生）

用户选择后，进入直接模式。

## 5. 核心提示词

以下提示词注入到每次故事生成过程中：

```
请选择一个【研究生以上难度】的概念。不要直接解释它。先构造一个完整世界观，再通过角色冲突、规则限制、资源交换或系统演化，把这个概念隐藏在故事结构里。要求：
1. 不要提前暴露概念名
2. 读者应能逐渐感受到某种规律存在
3. 在结尾处才揭晓真正概念
4. 最后详细解释：概念本身/隐喻映射/哪些机制对应现实理论
5. 不要写成童话，要像高质量科幻/黑镜/哲学寓言
6. 故事本身必须成立，即使不知道概念也能读
```

## 6. 设计规范

### 6.1 HTML 页面风格
- 基于风格 11（暗色 Linear 系）
- 详见 `templates/metaphor-page/design-token.md`
- 关键设计原则：
  - 暗色背景，高对比度文字
  - 分段叙事，每段有视觉分隔
  - 揭晓区域有特殊动效
  - 解释卡片有层次感

### 6.2 响应式设计
- 移动端优先
- 故事段落最大宽度 680px，居中
- 解释卡片在移动端改为纵向堆叠

## 7. 配置项

| 配置项 | 类型 | 必填 | 默认值 | 说明 |
|--------|------|------|--------|------|
| vault_path | string | 否 | "" | Obsessions 笔记路径，未配置则降级为方式 B |

配置文件位置：`config.json`
初始化方式：`python3 scripts/init_config.py --init`
