1. [严重度：中] CONDITIONAL_PASS 的修复确认流程缺失 → 第6节写"修复后放行（不重评）"，但没说谁来确认修复完成、以什么形式确认，实际执行中 Orchestrator 可能直接跳过验证 → 修复建议：在第6节补充"修复完成由 Orchestrator 对照 CONDITIONAL_PASS 中的必修项逐项勾选确认，记录至 DISCUSSION-LOG.md"

2. [严重度：中] 外部技术引用缺乏可核实路径 → 第8节多处提到"已核实，基于 OpenClaw 本地文档"，但未提供具体文档路径或 API 文档 URL，prompt caching 行为（cache_control 自动附加）无法被后续维护者核实 → 修复建议：补充具体路径（如 `~/.openclaw/...` 或官方文档 URL），或将8节改为"参考实现，需上线前实测核实"
