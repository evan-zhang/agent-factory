# TAROC 方法论核心参考

> 本文件整合自 WSP v8.0，作为 stock-picking-v2 的 Discovery 流程核心方法论。

**核心理念**：吃的不是信息的第一口（太早），也不是最后一口（接盘），而是中间那段——专业人士已确认价值、大众尚未充分消化的窗口期。

**方法来源**：ARK 主题投资 + 专业信息差理论 + 段永平"先用再投资" + 冯柳赔率思维 + Dan Ives 产业阶段 + O'Neil 止损规则

**目标**：月度 5-10% 收益，每笔 10%+，用移动止损 + 信息止损双重保护下限

## Step T: Professional Signal Discovery（专业信号发现）

### T.1 垂直领域扫描（按行业分类）

**AI/科技：**
- Hugging Face Trending、arXiv 热门论文、GitHub Trending
- Semianalysis / Stratechery
- 微信公众号：机器之心、量子位、新智元

**医药/创新药：**
- ASCO / ESMO 摘要发布、FDA 审批日历
- 医药魔方 / 丁香园 Insight / 生物谷

**半导体：** Semiengineering、TrendForce 集邦咨询

**新能源：** 高工锂电、SNE Research

**消费：** 魔镜市场情报、小红书热搜

**金融数据（通用）：**
- 龙虎榜、板块资金流向、融资融券、北向/南向资金、期权异动

### T.2 专业价值判断

对 T.1 抓到的每条信号，AI 扮演行业专家（15年经验）做判断：

1. 这个信息在从业者视角意味着什么？
2. 在行业圈子里算不算"大家都知道的事"？
3. 实质性影响：🔥变革级 / ⚡重要 / 📌一般 / 💤噪音
4. 如果影响≥重要，产业链上谁最受益？

**只有"变革级"或"重要"进入 T.3。**

### T.3 大众认知滞后评估

| 传播阶段 | 特征 | 窗口期 | 评分 |
|---------|------|--------|------|
| Phase 1（专业圈内） | 只在垂直社区讨论 | 5-10天 | ★★★★★ |
| Phase 2（开始扩散） | 券商开始出研报 | 3-7天 | ★★★★ |
| Phase 3（财经媒体） | 财经媒体报道未上头条 | 1-3天 | ★★★ |
| Phase 4（社交媒体） | 上了热搜/头条 | **离场信号** | ✋ |

评估指标：东方财富/雪球讨论量、微信/百度指数、综合媒体头条、券商研报覆盖数

### T.4 三个市场的差异化

**A股（窗口 3-7天）**：
- 入场：行业垂直社区、产业政策前信号、龙虎榜游资、融资融券异动、北向资金行业变化
- 离场：东方财富/同花顺热搜、股吧/雪球暴增、头条/微博热搜

**港股（窗口 1-3天）**：
- 入场：南向资金异动、A+H价差、港交所披露易、卖方首次覆盖
- 离场：富途/雪球港股讨论暴增、基金持仓扎堆

**美股（窗口 1-2天）**：
- 入场：行业大会预期差、期权异动、SEC 13F、细分垂直媒体、非头部券商上调
- 离场：Reddit WSB/Twitter 大面积讨论、CNBC/Bloomberg 头条、散户占比飙升

## Step A: Analysis（赛道分析）

### A.1 产业链地图
```
上游（核心零部件/技术）→ 中游（产品/平台）→ 下游（应用场景）
```
标注：**领导者**（份额>30%，⭐⭐⭐）、**赋能者**（关键零部件，⭐⭐）、**受益者**（间接，⭐）

### A.2 赛道质量检查
1. 成本在下降？→ 是通过；否淘汰
2. 有客户/订单？→ 具体客户通过；PPT淘汰
3. 竞争格局清晰？→ 有龙头加分；混乱减分
4. 政策有利？→ 有支持加分；有打压减分

### A.3 赛道评分（≥60 进 R）
```
总分 = 发展阶段×0.40 + 成本下降×0.25 + 政策×0.20 + 确认度×0.15
```
- 发展阶段：第3-4局=100；第2局=70；第5-6局=50；第7-9局=20
- 成本下降：年降30%+=100；年降15%=70；年降<15%=40
- 政策支持：国家级=100；地方级=60；无=30
- 信息确认度：3+独立源=100；2源=70；1源=40

```bash

```
字段：last, change_percentage, turnover, volume, high, low, prev_close

### R.2 公司质量检查
1. 有具体产品/服务？→ 有通过；概念淘汰
2. 有客户/订单？→ 有通过；PPT淘汰
3. 营收增速 >30%？→ 通过；<30%降权
4. 产业链位置关键？→ 加分
5. 流动性：港股>20亿HKD/日；美股>10亿USD/日

### R.3 新闻催化验证（强制双面搜索）

```
# 正面
web_search: "[名称] [代码] news {年} catalyst"
# 负面（强制，不可跳过）
web_search: "[名称] [代码] 风险 负面 看空 bear case"
```

催化评分：具体订单+15 | 产品量产+12 | 政策利好+10 | 财报超预期+10 | 机构上调+5 | 无新闻0 | 概念炒作-10 | 负面新闻-15

信息源优先级：财报/官方 > 彭博/路透/FT > ARK/雪球/东方财富 > 公众号/论坛

**所有输出标注**：来源 + 置信度 + 备注

## Step O: Odds（赔率评估）

### O.1 估值检查
- PE<30x → 合理，正常仓位
- PE 30-50x + PEG<2 → 成长合理，正常仓位
- PE 50-100x + 赛道极高景气 → 可能合理，仓位减半
- PE>100x或亏损 → 高风险，快通道或不参与

### O.2 入场区间
入场区间 = 当前价 ± 2%。偏离>5% → 等待。

### O.3 赔率计算
```
目标价 = 当前价 × (1 + 行业预期增速 × 时间窗口/12)
赔率 = (目标价 - 当前价) / (当前价 - 止损价)
止损价 = 当前价 × 0.92
通过标准：赔率 > 1.5:1
```

## Step C: Conviction（确信度验证）

### C.1 多空辩论（强制先搜负面）

```
# 强制先搜负面
web_search: "[名称] [代码] bear case risks problems short thesis"
```

三角色辩论：
- **Bull**：核心买入逻辑 + 最乐观场景 + 3个支撑证据（必须引用来源）
- **Bear**：基于搜索到的实际负面 + 最大风险 + 逻辑失败条件
- **评委**：谁更有说服力 + 确信度（★★★/★★/★）+ 理由

**TradingAgents（可选增强）**：
```bash
cd {WORKDIR}/engines/TradingAgents
PYTHONUNBUFFERED=1 .venv/bin/python scripts/run_analysis_stream.py --ticker [代码] --date [日期]
```
不可用或超时 → 回退同LLM三角色辩论。

### C.2 确信度评分
```
总分 = 专业信号传播×0.40 + 赛道×0.20 + 质量×0.15 + 赔率×0.15 + 确信度×0.10
```

| 评分 | 仓位 |
|------|------|
| ≥75 | 标准仓位（100,000 HKD） |
| 60-74 | 快通道（50,000 HKD） |
| <60 | 不建仓 |

集中度：同赛道≤1只 | 港美各≥1只 | 最多3只 | 宁缺毋滥

## 风控规则（不可变）

### 移动止损
- 初始止损 -8%
- 盈利>5% → 保本止损
- 盈利>10% → 止损+5%
- 盈利>15% → 止损+10%
- 盈利>20% → 止损+15%
- 双重确认：第一次触发后等5分钟，第二次仍触发才生成风险事件和人工决策请求

### 信息止损（与价格止损并行）
- Phase 1 入场 → 进入 Phase 3 推送预警
- Phase 2 入场 → 进入 Phase 4 触发止盈/减仓
- 任何持仓 → 综合媒体头条 → 生成 thesis break 风险事件
- 哪个先触发先进入人工决策队列

### 组合保护
- 组合回撤>20% → critical risk event + 冷却期建议（系统性10日/个股5日）
- 波动率>10%（日内）→ 暂停止损
- 隔夜美股暴跌>5% → 次日开盘前通知

### 执行红线
- 买入必须人工确认
- 卖出 v1 禁用自动执行，只能建议或生成 blocked recommendation
- 止损信号不可跳过，但执行必须经过 execution guard
- dry_run默认true
