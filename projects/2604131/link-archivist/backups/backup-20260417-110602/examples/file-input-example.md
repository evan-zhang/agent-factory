# Example 4: File Input — 用户直接发文件

## 输入
用户发送：一个 PDF 文件（某产品白皮书）

## 决策过程
1. 输入类型：文件（非 URL）
2. 调用外部文件解析工具提取文本内容（文件解析不由本 Skill 负责）
3. 获得文本内容后，走标准判断流程
4. 内容涉及技术架构、产品方案 → 关键词命中："架构"、"框架"
5. decide_mode.py → **full**

## 执行过程
1. Agent 调用文件解析工具 → 获得 PDF 全文
2. 基于全文内容生成报告
3. web_search 补充：搜索该公司/产品的最新动态
4. 搜索本地索引 → 关联已有调研
5. 生成完整报告
6. 归档到 `{archive_dir}/2026-04-13/K-260413-004-XX产品白皮书.md`

## 注意
- 文件解析（PDF/Word/PPT/Excel/图片）由外部工具负责
- 本 Skill 只接收已提取的文本内容
- Markdown 文件可直接读取，无需额外解析

## 输出
同 full 模式的标准报告格式。
