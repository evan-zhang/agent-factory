# Link Archivist

## Complete Workflow

```
收到链接
    │
    ├── 【第一步】获取基本信息
    │   └── curl / yt-dlp / web_search
    │
    ├── 【第二步】沟通确认需求 ★ 关键
    │   └── 询问Evan：要简短还是详细？
    │
    ├── 【第三步】执行
    │   ├── 详细 → 完整调研（10-30分钟）
    │   └── 简短 → 直接告知（1-2分钟）
    │
    └── 【第四步】存档 + 发送
```

---

## Step 1: Fetch Basic Info

获取基本信息，判断内容类型。

### 获取方式

| 链接类型 | 获取方式 | 命令 |
|---------|---------|------|
| **今日头条** | curl + Jina | `curl -sL "https://r.jina.ai/<链接>"` |
| **YouTube** | yt-dlp | `yt-dlp --print title --print duration --print uploader <链接>` |
| **其他网页** | curl 或 web_fetch | `curl -sL <URL>` |

### 判断内容类型

| 类型 | 特征 | 处理 |
|------|------|------|
| **开源项目** | GitHub、Star、功能 | 需要详细报告 |
| **新闻/资讯** | 事件、热点 | 简短告知即可 |
| **教程/讲解** | 步骤、用法 | 询问Evan |
| **对比分析** | A vs B | 值得详细报告 |
| **观点评论** | 个人看法 | 简短告知 |

## Step 2: Confirm with User

抓取内容后，**必须先确认需求**，不要直接开始详细调研。

### 汇报模板

**一般内容**：
```

```
| 维度 | 内容 |
| ------ | ---- |
| 项目 | <名称> |
| 功能 | <一句话功能描述> |
| 支持格式 | <格式列表> |

| 特点 | <核心特点> |
```

**核心能力：**
• <能力1>
• <能力2>
• <能力3>

要详细写报告吗？
```

**YouTube视频**：
```
🎬 [标题]
频道：[频道名] | 时长：[X分钟]

要下载转录吗？(y/n/简短)
```

**已调研过的项目更新**：
```
🔄 这是 [项目名] 的更新

[新信息摘要]

要补充到原报告吗？(y/n)
```

### 决策规则

| Evan回复 | 后续动作 |
|---------|---------|
| y / 写 / 详细 | 完整调研流程（10-30分钟） |
| n / 不 | 结束，不归档 |
| 简短 | 简短告知（1-2分钟），不写报告 |

### 好的沟通 vs 坏的沟通

**❌ 坏的沟通（直接开始）：**
> "好的，我现在开始调研..."

**✅ 好的沟通（先确认）：**
> "[标题] - [1句话描述]。要详细报告吗？"

## YouTube 视频额外流程

### 获取信息
```bash
yt-dlp --print title --print duration --print uploader "<链接>"
```

### 时长判断

| 时长 | 决策 |
|------|------|
| < 5分钟 | 简短，不需要转录 |
| 5-15分钟 | 询问用户 |
| > 15分钟 | 通常需要转录 |

### 下载 + 转录
```bash
yt-dlp -x --audio-format mp3 -o "/tmp/video.%(id)s.%(ext)s" "<链接>"
whisper /tmp/video.<id>.mp3 --language Chinese --output_dir /tmp/
rm /tmp/video.<id>.mp3 /tmp/video.<id>.txt
```

## Quality Assessment（5维度）

| 维度 | 值得归档 | 不值得归档 |
|------|---------|-----------|
| **新颖性** | 新项目/新功能 | 已知项目一般更新 |
| **信息量** | 有技术细节、数据 | 只有概述 |
| **相关性** | 和已有调研有联系 | 完全独立无关 |
| **可操作性** | 有实际价值 | 纯概念介绍 |
| **时效性** | 近期（<6个月） | 信息过时 |

## Writing Principles

**原则1：先骨架后血肉**
1. 先确定报告结构
2. 再填入关键信息（表格）
3. 最后补充描述文字

**原则2：信息密度优先**
- 每句话都要有信息量
- 避免废话和重复
- 用表格代替大段文字

**原则3：联系已有调研** ★
- 查LEARNING_INDEX.md找相关项目
- 做对比分析
- 指出可借鉴之处

**原则4：数据说话** ★
- 引用具体数字（Star、融资、用户数）
- 引用具体功能
- 格式覆盖要详细（15+种 vs 10种）
