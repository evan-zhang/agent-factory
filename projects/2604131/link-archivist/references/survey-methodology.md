# 调研方法论：执行调研（full 模式）

> 当 Phase 3 触发时，AI 读取此文件并按 checklist 逐步执行。
> 版本：v1.5.0 | 更新：2026-04-26

---

## Checklist：内容分析

- [ ] **判断内容类型**，确定分析侧重方向：
  - 开源项目介绍 → 重点：架构、技术栈、团队背景
  - 新闻/资讯 → 重点：关键信息提取
  - 教程/讲解 → 重点：步骤提取
  - 对比分析 → 重点：对比表格
  - 观点/评论 → 重点：观点记录

**输出**：内容类型判定结果 + 分析方向。

- [ ] **检测内容中的 GitHub 项目引用**（显式 URL 或疑似项目名）

**输出**：疑似项目名列表（可为空）。

---

## Checklist：信息补充与交叉验证

### GitHub 项目发现（条件触发）

> **触发条件**：内容中提到项目名 + "GitHub"/"开源"等关键词 → MUST 执行。
> 无关键词 → 跳过此节。

- [ ] 运行 `scripts/github_discovery.py` 提取显式 GitHub URL
- [ ] 提取疑似项目名（CamelCase、引号包裹等）
- [ ] 对每个疑似项目名，GitHub API 搜索 Top 3
- [ ] 核验 `description` 是否与原描述相关
- [ ] 匹配则触发 GitHub 深度调研

**输出**：`隐含项目发现` 表格（项目名 | Star 数 | 更新时间 | 与原内容关联）。无发现则跳过。

### GitHub 项目深度调研（条件触发）

> 仅当发现 GitHub 项目时执行。

- [ ] GitHub API 获取仓库信息（Star、License、更新时间）
- [ ] 读取 README 前 3000 字符
- [ ] 提取技术栈、核心功能描述

**输出**：每个项目的结构化信息摘要。

### Web Search 交叉验证 [MUST]

- [ ] 按策略执行搜索：
  - 开源项目 → `项目名 GitHub 功能关键词 2026`
  - 新闻事件 → `项目名 最新 2026`
  - 融资/公司 → `公司名 融资 投资 2026`
- [ ] 验证必须核实的数据：Star 数、用户数、融资额、投资方、创始人背景

**输出**：验证数据记录，用于 Claim 验证。

### Claim 验证（批判性论证）[MUST]

> **强制步骤**：所有关键 claim 必须联网验证，不接受"作者说什么就是什么"。

- [ ] 提取内容中所有可验证的 claim（数字、时间、功能描述、第三方背书）
- [ ] 逐条对照下表进行验证：

| Claim 类型 | 验证方式 | 结论判定 |
|-----------|---------|---------|
| 精确数字（Star/用户/融资） | GitHub API / 官网 | 一致 ✅ / 不一致 ❌ |
| 时间性描述（更新日期/发布日期） | GitHub API / PR 历史 | 一致 ✅ / 过时 ⚠️ |
| 功能/特性描述 | README / 官方文档 | 一致 ✅ / 夸大 ⚠️ |
| 第三方背书 | Web Search | 找到 ✅ / 找不到 ❌ |
| 性能指标 | Web Search + benchmark | 可信度评估 |

**输出**：Claim 验证表（序号 | 作者原话 | 声称数据 | 实际核查 | 结论）。
