### 改造 P3：One-pager 终局五问

**新增方法**（SKILL.md Step 5 的 One-pager 段落强化）：

```
【终局五问标准答法】（One-pager 强制包含 5 问 + 5 答）

1. 峰值销售额：基准X亿元（来源：第三章 3.6 双轨模型合计）
2. 承接业务主体：XXX（来源：第三章 互斥校验结果）
3. 假设错误一半是否可接受：可接受/不可接受
   - 可接受：保守情景下 IRR 仍 ≥ 门槛
   - 不可接受：保守情景下 IRR < 门槛，标注"不立项"
4. 最坏情况止损机制：触发条件 + 止损手段
   - 注册失败 → 立即停止
   - 峰值 < X 千万 → 商业化放弃
   - 触发一票否决项 → 自动叫停
5. 先发制人价值：高/中/低（不引进的后果评估）
   - 高：竞品 1-2 年内进入 → 必须引进
```

## 三、暂不改 P2 的原因

**P2 院外市场深度分析** 是 A-1 模板 v1.1 的特色章节，CMS 报告里做院外深度分析是必要的。

**但 Epioxa/Fabior 都缺数据**：
- 院外市场需要：零售药店销售数据、电商渠道数据、消费者偏好调研
- 当前数据源：仅公开流行病学、医保数据、IQVIA 报告（无 PDB 订阅）
- **强行加进模板 = AI 会编"院外占比 30%"这种 D 级数据**

**正确做法**：
- 暂在 SKILL.md 标注"P2 待接入 IQVIA/PDB 数据源后启用"
- 数据源接入后，单独出 v0.6.0 启用此章节
- v0.5.0 模板不强制要求院外分析（Gate 3.6 双轨模型的"院外渠道说明"行仍保留作为可选项）
