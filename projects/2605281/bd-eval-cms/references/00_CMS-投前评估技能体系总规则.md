## 1. 体系概述

### 1.1 体系规模与构成

本体系由 **20个独立技能** 与 **1个战略合作治理增效工具包**（非独立挂载）共同构成，覆盖CMS集团全业务条线的投前评估全流程。

```
总体结构
├── D群 × 4   —— 基座与工作流（横向底座，支撑所有技能）
├── A群 × 9   —— 中国市场药品评估（核心业务主干）
├── B群 × 3   —— 消费属性产品评估（医美+消费健康）
├── E群 × 1   —— 股权投资评估
├── C群 × 3   —— 国际市场评估（单市场 / 多市场排序 / 组合策略）
└── 战略合作治理增效工具包（非独立）
```

### 1.2 三框架融合原则

所有20个独立技能均在三框架融合的底层架构上运行，不允许任何技能绕过任一层：

| 框架层级 | 名称 | 核心功能 |
|---|---|---|
| **骨架层** | 终审模板 v6.2 | 定义输出结构、必输字段、一票否决项、Gate结论格式 |
| **内容层** | 投前尽调14维度 | 规定评估维度的覆盖完整性与深度标准 |
| **方法论层** | 麦肯锡8步法 | 问题结构化→假设树→数据驱动→叙事线→交付物 |

> **强制规则**：三层框架缺一不可。任何单技能如仅满足其中一层或两层，视为不合规输出，须在当轮标注"框架缺失项"并终止推进。

### 1.3 体系设计原则

1. **以终为始**：每项评估必须先定义成功终局（One-pager），再倒推验证路径。
2. **Gate即止损**：任一Gate未达标，当轮停止推进，不允许"带着疑点往下走"。
3. **财务硬编码**：第5节财务内控标准为硬编码基准，不可在单技能层面自行降低门槛。
4. **模块化串接**：每个技能暴露标准串接接口，支持技能链组合调用。
5. **输出一致性**：全体系统一采用第10节输出规范，不允许各技能自定义风格。
6. **临床阶段分层**：临床阶段项目不得笼统写作"I/II/III期"或"临床阶段品种"后直接套用成熟资产口径；必须按I期、IIa/PoC、IIb/EOP2前、EOP2后、III期、NDA/BLA前后分层，分别确定证据成熟度、PoS/rNPV、交易付款和止损节点。
7. **在研产品证据链完整性**：在研产品必须同时核科学假设、非临床、安全窗、BA/PK/PD/免疫原性、临床设计、统计、CMC、注册沟通、数据完整性、商业与交易止损，不得只用"临床阶段较靠后"替代完整技术评估。
8. **注册加速与早期可及边界**：真实世界研究/RWE、海南乐城、大湾区港澳药械通、北京天竺临床急需进口或罕见病保障、突破性治疗、附条件批准、优先审评、MRCT中国同步开发等路径，均可作为临床医学与注册策略的加速工具评估；但早期可及不等同于NMPA上市批准，RWE不自动替代关键注册证据，未获官方资格和专业责任人确认前不得进入基准上市时间、基准销量或基准财务模型。

### 1.4 临床阶段项目统一分层原则（强制）

凡处于I期至NDA/BLA前后的国内或海外临床阶段药品，必须触发D-1《临床阶段分层评估卡》，并在D-3 One-pager中列明当前阶段、下一证据拐点、PoS/rNPV口径和交易止损节点。

| 阶段 | 体系口径 | 不得出现的错误用法 |
|---|---|---|
| I期 / PoC前 | 主要验证安全窗、BA/PK/PD、剂量探索和转化医学基础；交易上偏期权价值 | 用早期疗效信号支撑确定性峰值销售或高首付款 |
| IIa / PoC后 | 验证PoC是否与目标适应症、人群、终点、BA/PK/PD和商业主张一致 | 只写"II期阳性"而不解释效应量、剂量-反应、终点、暴露-效应和SOC |
| IIb / EOP2前 | 判断是否足以进入EOP2或关键III期设计沟通 | 把EOP2前项目按EOP2后或III期成熟资产估值 |
| EOP2后 / 关键III期准备 | 监管沟通和III期方案成熟度提高，但仍需核终点、统计、CMC和执行风险 | 写成"疗效已确证"或取消临床风险折现 |
| III期进行中 | 重点核研究执行、入组/事件、数据完整性、统计显著性、CMC商业化准备 | 仅因进入III期就默认获批或默认标签范围 |
| NDA/BLA前后 | 重点核CSR/ISS/ISE、审评问题、标签、检查、药物警戒和上市准备 | 忽略发补、标签收窄、检查失败或补充研究风险 |

临床阶段项目的财务模型必须与阶段分层一致：EOP2前后须分列PoS/rNPV；III期进行中须列读出失败、延迟和补充研究压力情景；NDA/BLA前后须列标签收窄、审评延迟和CMC/检查压力情景。

凡PK/PD、剂量选择、暴露-效应、免疫原性、MRCT/桥接、BE或生物类似性影响项目结论，必须触发D-1《BA/PK/PD/免疫原性验收卡》。BA不可采信时，不得支撑PoC成立、剂量选择、桥接外推、BE通过、标签主张或rNPV上调。

海外在研或国际多中心临床（MRCT）项目，若中国权益需要使用海外临床数据、全球MRCT数据包、CSR/ISS/ISE、原始数据或统计分析成果，必须单列"海外临床数据包权益与成本"。不得默认海外数据随授权免费取得；必须确认数据访问权、使用权、再提交权、审计权、数据更新权、监管问询支持义务、费用口径和付款节点。若合作方按中国市场生命周期量或价值占全球约10%（如内部亦称MSC，须与最低销售承诺MSC区分）等口径要求分摊全球临床数据成本，该比例只能作为谈判参考和压力情景，不得自动视为合理成本进入基准模型；若数据费用导致IRR/rNPV/回收期或交易倍数不达标，必须重谈、设置上限、改为里程碑后置或停止推进。

国内临床阶段项目必须与海外临床阶段项目分开判断：国内合作方或CMS自主持有的在研品种，应额外核Sponsor/IND申请人、CDE受理或60日默示许可、临床试验登记公示、伦理批件、GCP机构与PI能力、SUSAR/SAE/DSUR安全性报告、方案变更、HGR（人类遗传资源）审批/备案、原始数据可审计性、中心入组质量和交易数据访问权。不得把"国内已进II/III期"直接等同于注册路径清晰或商业化授权成熟；EOP2后项目也必须有CDE沟通纪要、书面反馈或等效监管证据。

### 1.6 药品BD高标准漏审清单（Final前强制）

> 本清单适用于所有药品类投前项目（A群药品、C群国际药品、E-1涉及药品管线或商业化权益的项目）。B群如涉及药品、药械组合或医疗器械注册风险时参照执行。清单不是新增章节堆叠，而是Final前的横向复核：任何一项为“未评估/无证据/无责任人/无交易保护”，均不得输出无条件通过。

### 2.2 适应症细分归口与边界判断规则

> 深康、德镁、维盛均为院内经营主体，原则上按细分治疗领域、**主评估适应症**、核心处方科室和主要商业化资源确定**一家院内主承接主体**。三者不是常态"主辅"结构；专业会签仅在明确触发条件下发生，不等同于共同承接、资源共享或P&L拆分。

#### 多适应症项目：主评估适应症选择规则（强制）

多适应症项目不得直接按"首发适应症"或"覆盖多个科室"写成多主体协同。必须先建立适应症机会账，选择一个**主评估适应症**作为主报告、主财务模型和主承接主体的锚点。

| 判断项 | 选择要求 |
|---|---|
| 主评估适应症 | 原则上选择可被证据支撑的商业潜力最大、战略价值最高、且CMS可承接能力最强的适应症；通常以未来3-5年峰值销售/NPV/利润贡献最高者为先 |
| 首发适应症 | 只代表注册或上市路径的先后，不自动决定主承接主体；仅当首发适应症也是近期主要收入贡献或战略入口时，才可作为主评估适应症 |
| 潜力适应症 | 若证据尚不支持标签、指南、注册或销售预测，只能列为"潜力适应症/上行情景"，不得作为基准财务模型和主承接主体的唯一依据 |
| 主承接主体 | 由主评估适应症对应的核心处方科室、患者路径、销售资源、P&L归属和承接能力决定，深康/德镁/维盛原则上只选一家 |
| 条件会签 | 次要适应症预计贡献显著、影响医学策略/准入路径/销售预测/既有组合冲突时触发；否则填"无" |

**模糊边界处理：**
- 路由结论必须输出 `主评估适应症 / 首发或当前注册适应症 / 潜力适应症 / 主承接主体 / 专业会签主体 / 专业输入或资料支持主体 / 院外叠加层 / 国际叠加层 / 信息隔离状态`；其中专业会签主体可以为"无"，不得为了完整性而强行列会签。
- 深康、德镁、维盛之间原则上只确定一个院内主承接主体。多适应症项目应先完成"主评估适应症校准卡"，再按主评估适应症、近期销售预测贡献、核心处方科室、P&L归属和销售资源可承接性判定主责，不得默认写成"深康/德镁/维盛协同"。
- 专业会签的触发标准是：另一主体的治疗领域、医生网络、医学策略、准入路径、既有组合或销售资源会实质改变项目结论；否则专业会签主体填"无"。
- 组织层级、代表覆盖、KOL网络、费用率和爬坡速度不得跨主体套用；深康、德镁、维盛的商业化假设须分别取数和验证。
- 如存在已签竞品限制、排他、优先权、同机制/同API/同渠道冲突或NDA信息隔离义务，优先执行§4.2A，不得用主体归口判断覆盖合规约束。

### 2.3 六大业务主体组合约束规则（强制执行）

**规则一：院内直营 vs 代理，在同一产品上互斥**
- 深康、德镁、维盛为院内直营模式（自有医药代表）；天津康哲为代理/经销商模式。
- 两类模式在院内推广属性上根本不同，不得同时承接同一产品的院内推广。
- 任何评估报告的"承接主体方案"不得出现"直营 + 天津康哲协同"的非法组合。专业会签不得被用来绕过该互斥规则；如仅需非承接性的资料、注册或市场意见支持，应写入"专业输入/资料支持"，不得写入承接主体方案。

**规则二：院外业务中心可叠加于任意一个院内主体**
- 院外业务中心不属于院内业务，可独立承接DTP药房、零售药店、互联网医院、电商等渠道。
- 合法叠加组合为：`深康 + 院外`、`德镁 + 院外`、`维盛 + 院外`、`天津康哲 + 院外`。
- 院外是渠道层，不改变院内主承接主体，也不消除院内直营与代理互斥。

**规则三：承接主体方案须先从合法组合中二选一**
- 先选择院内路径：`直营路径（深康/德镁/维盛三选一）` 或 `天津康哲代理/经销路径`。
- 再判断是否叠加院外业务中心；不得先写"协同"再倒推合法性。
- 每份评估报告的"承接主体比较"章节必须先列出合法方案边界，再在合法方案内部比较，不得以"协同"名义绕过互斥规则。

### 2.4 技能群与业务主体交叉使用规则

- 同一产品原则上只形成一份主承接报告，由一个院内主承接主体负责商业化终局。仅当存在清晰独立的渠道层、国际市场层、权利边界或财务模型时，才叠加对应报告或章节。
- 跨主体协同为例外，不是默认结构。确需协同时，须在报告末尾附"协同说明"章节，说明分工界面、权益划分、P&L归属和资源调度机制。
- 天津康哲可按日常业务习惯简称为"天康"；康哲维盛可简称为"维盛"。首次出现建议标注全称，后文可使用简称。

## 3. Gate底层逻辑（以终为始·Gate 0~5 + Final框架）

### 3.1 框架来源

以D-1/D-3 Stage-Gate主线为跨模板主键，并吸收A-8等成熟模板的门控经验，抽象为体系级通用Gate逻辑。A/B/C/E主评估模板及A-0/A-7/B-2/C-2/C-3/D-0/D-2等筛选、组合或工具类技能可使用等价门控结构，但必须映射到 `Gate 0~5 + Final`。

### 3.2 评估起点：One-pager（终局先立）

在任何Gate启动前，必须完成One-pager，明确以下五项：

| 字段 | 内容要求 |
|---|---|
| 成功终局定义 | 上市后第X年峰值销售额、市占率目标、对集团贡献 |
| 核心价值主张 | 凭什么在目标市场胜出（1-2句话，不允许罗列） |
| 最大风险单项 | 若只有一个风险能击垮本项目，是什么 |
| 止损边界 | 在何种条件下应主动退出或终止 |
| 决策人与时限 | 谁来拍板，最晚何时须给出结论 |

### 3.3 体系Gate主键详解（Gate 0~5 + Final）

> **Gate编号规则**：D-1与D-3采用 `Gate 0~5 + Final` 作为体系级主键。各技能可设置场景化别名或本地Gate，但必须映射回以下主键；不得把本地Gate编号当作跨模板主键，也不得因模板节奏不同漏掉任一审查功能。

#### Gate 0 — 准入/路由门
**核心问题**：商机是否值得投入资源，路径是否选对？  
**必审项**：五必填、D-0路由、业务主体/适应症归口、硬排除、内部竞品/信息隔离、初步财务量级。  
**输出**：立项 / 补料 / Watch / 否决。  
**未达处理**：不得进入立项或阶段B。

---

#### Gate 1 — 前提门（Premise Gate）
**核心问题**：标的是否具备可评估、可授权、可注册的基本前提？  
**必审项**：合作方主体资质、权属/授权链、产品法律可行性、注册路径、CMS现有管线/已签竞品限制/排他条款/优先权条款/信息隔离义务、NDA状态、主体合法组合。  
**输出**：前提通过 / 条件通过 / 前提未达。  
**未达处理**：当轮终止，不得用商业潜力补足前提缺口。

---

#### Gate 2 — 证据/技术门（Evidence & Technical Gate）
**核心问题**：科学、临床、注册、CMC和执行证据是否支撑目标主张？  
**必审项**：MoA、非临床/临床数据质量、BA/PK/PD、统计、CDE/注册沟通、CMC/GMP、数据完整性、国内/海外临床专项卡、RWE可采信性。  
**输出**：证据强度评级 + 技术风险评级 + 置信度 + 需补证据Top 5。  
**未达处理**：证据不可采信时阻断或降级；阶段A公开信息不足只能标注待验证，不得写成通过。

---

#### Gate 3 — 商业/准入门（Commercial & Access Gate）
**核心问题**：市场、竞争、适应症、承接主体和支付逻辑是否成立？  
**必审项**：患者池、指南/SOC、竞品格局、KOL、准入路径、价格走廊、渠道/院内院外、承接主体资源、国际市场HTA/支付体系（如适用）。  
**输出**：商业可行性结论 + 市场容量/峰值销售假设 + 准入/支付结论 + 置信度。  
**未达处理**：商业假设未验证时只能条件推进或Watch，不得直接给确定性峰值销售。

---

#### Gate 4 — 财务/合规价值门（Value & Compliance Gate）
**核心问题**：经济回报和法务/IP/合规风险是否可接受？  
**必审项**：2506AARFY硬门槛、IRR/rNPV/回收期/倍数、COGS/SG&A、IP/FTO、数据保护/市场独占、合规红旗、信息隔离、汇率敞口（非人民币市场）。  
**输出**：财务达标/未达/待确认 + 合规/IP红旗 + 基准/压力情景。  
**未达处理**：财务硬门槛不达或高风险未关闭时不得终审通过。

---

#### Gate 5 — 交易/执行门（Dealability & Execution Gate）
**核心问题**：TS、合同、供应链和投后承接是否可执行？  
**必审项**：交易结构、首付款/里程碑/分成、止损条款、退出权、数据访问权、知识产权归属、供应/技转/质量协议、交割条件与时间表、投后承接人。  
**输出**：可做 / 条件可做 / 不可做 + TS关键条款 + 红旗关闭状态。  
**未达处理**：未关闭不得签约；D-3只能出v4待确认版。

---

#### Final — 终审门
**核心问题**：是否可提交投委会形成批准/条件批准/否决？  
**必审项**：D-3 Final、Gate 0~5全部结论、专业模块签署、红旗台账V2、未关闭事项与条件批准清单。  
**输出**：批准 / 条件批准 / 否决 / 再内审。  
**未达处理**：任一体系Gate无结论、无证据或无责任人时，不得生成无条件批准。

### 3.7 置信度判定口径

| 置信度 | 判定标准 | 使用限制 |
|--------|----------|----------|
| 高 | 至少一个官方/内部锁定来源，或两个以上独立来源相互印证，且无重大冲突 | 可作为Gate通过或Final结论依据 |
| 中 | 有单一主来源或多来源但独立性不足，关键假设仍需验证 | 可用于阶段A/B判断，但须列补证路径 |
| 低 | 仅有合作方口头陈述、单一专家观点、未经核验估算或公开信息缺失 | 只能作为待验证假设，不得单独支撑终审结论 |

## 4. 投前评估三阶段流程模型

### 4.1 阶段A — 商机发现与立项决策

```
商机捕获
    └─► Gate 0 初筛（A-0技能：商机雷达与初筛）
            └─► 具体产品初审评估报告（A-1至A-8 / B-1 / B-3 / C-1 / E-1 按产品类型选取）
                    └─► 立项会决策（Go / No-Go / 附条件Go）
```

**阶段A关键节点说明**：
- Gate 0初筛须在72小时内完成，输出"初筛一页纸"（不超过500字）。
- 初审评估报告覆盖M-01~M-12（第6节公司通用模块，含内部竞品约定与信息隔离状态、法规/IP/保护期触发初筛）+ Gate 1~Gate 3。
- 立项会须有投前负责人、对应业务主体负责人、财务代表三方同时在场。

### 4.2 NDA弹性签署节点

> **重要**：NDA（保密协议）签署节点**不固定于特定阶段**，遵循以下弹性规则：

| 情形 | NDA签署时机 |
|---|---|
| 合作方主动提供完整数据包 | 数据交换前签署 |
| 合作方有内部数据库可授权访问 | 签署NDA后即可获取，无需等待阶段B |
| 非竞争性公开数据 | 可先行研究，NDA在进入实质谈判前签署 |
| 公开上市公司信息 | 无需NDA |

签署NDA并不代表项目正式立项，亦不构成对合作的任何承诺。

### 4.2A 潜在竞争项目与信息隔离机制

> 本节为体系级前置合规约束，参考《【请示】投前引进保密协议执行及潜在竞争项目信息隔离机制-初稿建议 [2606AAMRQ]》所提示的问题。任何项目在签署NDA、接收保密资料、进入阶段B深化评估或生成D-3 Final前，均须核查是否与CMS现有组合、在谈项目、已签协议或已接收保密信息存在竞争限制或信息隔离义务。

**触发场景**：

| 场景 | 示例 | 处理原则 |
|------|------|----------|
| 现有产品组合重叠 | 同适应症、同机制、同API、同渠道、同目标医生/KOL资源 | 先做内部组合冲突判断，再决定是否继续评估 |
| 已签协议限制 | 排他、非竞争、不接触竞品、优先谈判权、优先购买权、最惠条款、区域/渠道限制 | 法务核查条款边界；若明确禁止且无法豁免，停止评估 |
| 在谈项目冲突 | 已签NDA或已进入数据室的竞品项目 | 建立项目隔离和Clean Team，不得交叉使用保密信息 |
| 人员交叉风险 | 同一评估人员同时接触两个潜在竞品的保密资料 | 调整人员名单、权限和资料访问边界 |
| 信息输出风险 | 报告中混用不同项目的非公开数据、假设或条款信息 | 分项目归档，来源标注，禁止跨项目引用保密信息 |

**隔离状态判定**：

| 状态 | 定义 | 处理 |
|------|------|------|
| Green | 无内部组合重叠，无已签协议限制，无保密信息交叉 | 可正常推进 |
| Yellow | 存在潜在竞争或人员交叉，但可通过Clean Team、权限隔离、来源标注解决 | 条件通过；完成隔离方案前不得接收新增保密资料 |
| Red | 已签协议明确禁止评估/接触竞品，或无法建立有效信息隔离 | 停止评估或提交法务/投前委员会书面豁免 |

**最低输出要求**：
- 项目启动时输出《竞品约定与信息隔离核查卡》。
- NDA签署前确认对方NDA是否含不竞争、不接触竞品、使用限制、返还/销毁、人员范围限制等条款。
- 阶段B前确认Clean Team名单、资料权限、文件夹/数据室隔离、报告来源标注规则。
- D-3 Final前确认所有Yellow项已关闭；Red项未获得书面豁免时不得生成Final。

**《竞品约定与信息隔离核查卡》模板**：

| 字段 | 填写要求 |
|------|----------|
| 标的项目 | 产品名/合作方/适应症/机制/API/渠道 |
| 内部重叠项 | CMS现有产品、在谈项目、已签协议、已接收保密资料中可能重叠的项目 |
| 重叠类型 | 同适应症/同机制/同API/同渠道/同KOL资源/同地域权利 |
| 已签条款限制 | 排他、不竞争、不接触竞品、优先谈判/购买、最惠、区域/渠道限制、信息使用限制 |
| NDA状态 | 未签/拟签/已签；是否含竞品接触或信息使用限制 |
| 隔离状态 | Green / Yellow / Red |
| Clean Team方案 | 人员名单、权限边界、数据室/文件夹隔离、会议隔离、报告来源标注 |
| 法务结论 | 可推进/条件推进/停止/需书面豁免 |
| 下一步动作 | 责任人、截止时间、关闭证据 |

### 4.3 阶段B — 立项后双轨并行

```
立项批准
    ├─► 技术深化评估线（Gate 3~Gate 5）
    │       ├── 临床与注册深度尽调
    │       ├── 生产与供应链验证
    │       └── 市场与准入精细化建模
    │
    └─► 商务谈判线（TS/LOI草案拟定）
            ├── 财务进场节点：TS阶段（不早于TS草案出具）
            ├── 法务进场节点：LOI签署前
            └── 双线互动：技术评估结论实时反哺谈判条款
```

**双轨并行原则**：
- 两条线独立推进，但须在每周例会中同步进展。
- 技术线重大发现（如临床数据造假、专利障碍）须立即暂停谈判线，等待技术结论更新。
- 商务线重大让步（如独家条款放弃、里程碑大幅降低）须触发技术线重新评估财务模型。

### 4.4 阶段C — 终审决策

```
双线收敛
    └─► D-3技能：生成投委会决策包
            └─► Final 投委会审议
                    ├─► 批准：进入协议谈判与签署
                    ├─► 条件批准：列明附加条件，设定条件达成时限
                    └─► 否决：归档，说明否决理由，可设置"重新提交窗口期"
```

**投委会决策包必备模块**：
1. One-pager终局摘要
2. Gate 0~5 + Final结论卡汇总
3. 财务回报敏感性分析（基准/乐观/悲观三情景）
4. 一票否决项最终核查表
5. 交易结构草案
6. 推荐意见（Go / No-Go / 有条件Go）

## 5. 财务评估内控标准

> **硬编码声明**：本节所有财务门槛以《投前评估关键要素与问题引导清单(供参考)_财务附件.xlsx》（对应汇报 `投前评估关键要素-财务维度 [2506AARFY]`）为准，不可在单技能层面自行调整。如确有特殊情形需要豁免，须经集团CFO及投前委员会联合书面审批，并在评估报告中明确标注豁免原因。

> **财务缩写注释规范**（适用全文件）：IRR（内部收益率）/ EBITDA（税息折旧摊销前利润）/ FCF（自由现金流）/ CAPEX（资本支出）/ NPV（净现值）/ rNPV（风险调整后净现值）/ BOM（物料清单）/ SG&A（销售费用及管理费用）/ ROI（投资回报率）/ ROIC（投入资本回报率）/ VBP（带量采购）/ HTA（卫生技术评估）/ SOC（标准治疗方案）/ EPS（每股收益）/ DSO（应收账款周转天数）/ TS（条款清单）/ LOI（意向书）/ NDA（保密协议）/ CDE（国家药品审评中心）。

|---|---|---|---|---|---|---|---|---|
| ≤3亿元 | 上市≤3年 | ≥15% | ≤5年 | 2倍 | 4倍 | ≥60% | 10年均≥20% | 上市后3年累计 |
| ≤3亿元 | 上市3-5年 | ≥20% | ≤5年 | 2.5倍 | 6倍 | ≥60% | 10年均≥20% | 上市后3年累计 |

| >3亿元 | 上市3-5年 | ≥22% | ≤5年 | 2.7倍 | 7倍 | ≥60% | 10年均≥20% | 上市后3年累计 |
| >3亿元 | 上市5年以上 | ≥25% | ≤5年 | 3.1倍 | 9倍 | ≥60% | 10年均≥20% | 上市后3年累计 |

**创新药补充说明**：
- "产品风险投资额"按2506AARFY定义：不考虑资金时间价值，在产品研究III期临床结束前支付的投资金额，包含首付款、III期临床阳性结果之前支付的注册里程碑和公司承担的临床试验费用。
- 如产品投资同时对拥有产品权利的企业进行了股权投资，且被投资企业核心产品为该单一产品，则该股权投资视同为产品风险投资；如被投资企业拥有多种品种，按战投部预估的股权投资亏损额计入产品风险投资额。
- 上市剩余年限从协议签署日起算至预计获批上市日。
- 投资回收期从产品上市销售首年开始计算，至项目累计现金流量现值为0止；产品上市前研究阶段每年投入资金按复利5%计算到产品上市前终值后纳入投资总额。
- IRR（内部收益率）为考虑时间价值下投资回报收益率，标准参考CMS、中国生物制药、恒瑞医药等近5年平均年净资产收益率20%。
- 毛利率按源表定义：`1 - （分摊后的首付款及注册里程碑 + 销售分成Royalty + 供货成本 + 其他如有）/ 净销售额`。

### 5.2 仿制药财务内控标准

| 品种类型 | IRR（内部收益率）| 回收期 | 5年倍数 | 净利润率 | 盈亏平衡 |
|---|---|---|---|---|---|
| 一般化学仿制药 | ≥50% | ≤3年 | 3倍 | 5年均≥20% | 上市首年盈利 |
| 短期不纳入集采品种 | ≥35% | ≤3.5年 | 4倍 | 5年均≥20% | — |

**仿制药补充说明**：
- 预测周期按源表：`集采前时长 + 第一轮集采周期时长`。
- "产品风险投资额（仿制药）"按2506AARFY定义：不考虑资金时间价值，产品上市前所有投资金额均视为风险投资额，包含首付款至获得生产批件承担的费用。
- VBP（带量采购）风险必须在Gate 4中单独量化，建议敏感性测试覆盖"当期不集采/3年内集采/上市即集采"三个情景。
- 短期不集采品种须附专家论证意见，说明判断依据及结论有效期（通常不超过3年）。
- 生物类似药、改良型新药未在2506AARFY投资标准表中单列固定门槛；正式投委会包必须由财务部门确认其适用"创新药"或"仿制药"哪一类标准，不得在技能层面自设替代门槛。

|---|---|---|---|---|---|
| ≤1亿元 | ≤2年 | 5年 | ≥50% | ≤2年 | 15倍 |
| ≤1亿元 | >2年 | 5年 | ≥55% | ≤2年 | 20倍 |

**医美产品补充说明**：

### 5.4 财务模型通用规则

1. **双情景基准**：所有财务模型须至少输出"基准情景"和"压力情景"；压力情景假设可由财务部门或项目评审会确定，不得替代2506AARFY硬门槛。
2. **汇率处理**：非人民币市场交易须采用当日中间价作为基准汇率，敏感性测试覆盖±10%汇率波动。
3. **折现率**：折现率作为模型假设须由财务模型注明；投资回收期按2506AARFY要求，对上市前研究阶段投入按复利5%计算至产品上市前终值。
4. **税务假设**：中国大陆项目默认适用高新技术企业15%所得税税率；国际项目须在报告中明确当地税率假设。
5. **rNPV（风险调整后净现值）适用范围**：临床阶段项目（I期、IIa/PoC、IIb/EOP2前、EOP2后、III期、NDA/BLA前后）必须采用rNPV模型，不允许直接使用NPV（净现值）；EOP2前后须分列PoS和rNPV口径。

### 5.5 产品系数评分规则

产品系数评分用于解释"引进数量/产品系数"等管理台账口径，不等同于简单产品个数。

| 项目 | 规则 |
|---|---|
| 初始分数① | 预测周期累计NPV（净现值）≥3亿元时，初始分数=1.0；累计NPV<3亿元时，初始分数=累计NPV/3亿元 |
| 调节系数② | 2-3亿元=0.8；1.5-2亿元=0.7；1-1.5亿元=0.6；0.5-1亿元=0.5；<0.5亿元=0.4 |
| 业务特点调节系数③ | 常规业务=1.0；全新业务且不确定性高=0.5，待后续跟进，5年看判断达到峰值的可能性后回顾调整 |
| 最终得分 | `最终得分 = 初始分数① × 调节系数② × 业务特点调节系数③` |

## 8. 战略合作治理增效工具包（非独立技能）

### 8.1 定位说明

本工具包**非独立技能**，不可单独触发，必须通过"签约后治理链"挂载于任一主技能之上。适用于协议签署后合作执行期的全生命周期治理。

### 8.2 五大工具详述

#### 工具1：Issue Tree（问题树）
- **用途**：结构化拆解合作中的核心问题，确保团队对问题的认知一致
- **格式要求**：MECE原则，3层以上，叶节点可赋责任人
- **触发场景**：合作执行期出现方向性争议、目标偏差或跨部门协同障碍时

#### 工具2：IP归属三维矩阵
- **用途**：厘清合作中产生的知识产权（IP）归属
- **三维度**：资金出资方 × 技术贡献方 × 使用权归属
- **输出格式**：3×3矩阵表 + 合同语言建议
- **触发场景**：联合开发项目、合作研究协议（SRA）签署前后

#### 工具3：JSC-JPT治理框架
- **JSC**（联合指导委员会）：季度战略层级，成员含双方VP级以上
- **JPT**（联合项目团队）：月度执行层级，成员含双方项目负责人
- **输出内容**：会议章程模板 / 决策权限矩阵 / 升级路径 / KPI仪表盘
- **触发场景**：独家合作协议签署后，商业化合作正式启动时

#### 工具4：退出机制模板
- **触发条件列表**（以下任一条件成立即可触发退出）：
  - 连续2个里程碑未达成
  - 合作方关键人员（CEO/CSO）离职且90天内未补充
  - 监管机构发出重大合规警告
  - 双方对下一阶段开发方向存在无法调和的实质分歧
- **退出方式**：全面退出 / 缩减合作范围 / 转让合作权益
- **输出内容**：退出通知模板 / 资产分割方案 / 保密延续义务条款

#### 工具5：投委会审议决策包
- **适用场景**：阶段C终审、追加投资审批、重大合同条款变更审议
- **必备内容**：One-pager摘要 / Gate结论卡汇总 / 三情景财务模型 / 一票否决核查表 / 推荐意见
- **格式**：不超过20页PPT或等效HTML报告
- **时限要求**：投委会召开前48小时完成并送达所有委员

## 10. 输出规范（全局）

### 10.1 文件格式

- **标准格式**：HTML（所有正式评估报告）
- **辅助格式**：Markdown（内部讨论草稿）、PDF（归档存档）
- **禁用格式**：纯TXT、Word .doc（旧格式）、未加密的在线共享链接

### 10.2 视觉规范

| 规范项 | 要求 |
|---|---|
| 主品牌色 | 深蓝 `#1a3a5c` |
| 辅助色 | 允许使用白色、浅灰背景色（`#f5f7fa`），但**正文文字禁止使用灰色** |
| 字体 | 中文：思源黑体或等效无衬线字体；英文：Arial或等效 |
| 风格基调 | 麦肯锡风格：简洁、数据驱动、无装饰性元素、无多余图片 |
| 禁用元素 | 灰色字体（任何位置）、元描述框（"本报告旨在…"类开场）、修订说明段落 |

### 10.3 财务缩写标注规范

- 所有财务英文缩写在**全文首次出现时**必须在括号内标注中文全称
- 再次出现时可直接使用缩写，无需重复标注
- 参考标注示例：`IRR（内部收益率）`、`EBITDA（税息折旧摊销前利润）`、`FCF（自由现金流）`
- 中文缩写无需额外标注（如"集采"无需标注"带量采购"，但首次使用VBP时须注"VBP（带量采购）"）

### 10.4 多币种标注规范

- 非人民币市场的所有金额须**双币标注**：`当地货币金额（约合人民币XXX万元/亿元）`
- 汇率须在报告首部声明基准汇率及取值日期
- 示例：`港币5,000万元（约合人民币4,580万元，汇率基准：2026-05-28）`
- 国际市场评估报告（C群）须额外附汇率敏感性分析

### 10.5 章节顺序规范

正式报告章节必须按以下顺序排列，不允许调换：
```
第1章 → 第2章 → ... → 第N章 → 附录 → 参考文献
```
- **附录和参考文献必须置于正文所有章节之后**，不允许将附录插入正文章节之间
- 参考文献须使用统一引用格式（Vancouver格式或GB/T 7714），并提供可核查的来源链接

### 10.6 禁止内容清单

以下内容**严禁出现**在任何正式输出报告中：
1. 任何灰色字体（十六进制色值 `#888888` 以上区间）
2. 元描述性语言（"本报告旨在…" / "以下是…的分析" / "根据您的要求…"）
3. 修订说明段落（"第X版修订了…"类表述，版本信息统一在文件头部元信息中记录）
4. 未经标注的推测性财务数据（所有财务数字须注明来源或标注"基于假设"）
5. 来源不明的市场数据（须标注数据来源及获取时间）
6. 使用未经业务确认或容易误导的主体简称；天津康哲可简称"天康"，康哲维盛可简称"维盛"，首次出现建议标注全称

## 附录B：技能调用决策树（快速检索）

```
用户输入 → D-0 路由调度器判断
    ├── 中国市场 + 药品 + 海外已上市未在中国上市 → A-1
    ├── 中国市场 + 药品 + 国内Biotech/药企成熟品种合作 → A-2 (± E-1)
    ├── 中国市场 + 自主研发意向 → A-3 (± E-1)
    ├── 中国市场 + 早期/临床前 → A-4
    ├── 中国市场 + 药品 + 已上市产品推广权/代理权接管 → A-5
    ├── 中国市场 + 已有院内处方基础 + 院外延伸 → A-6
    ├── 中国市场 + OTC/消费健康/DTC → B-3
    ├── 多产品同时比较 → A-7
    ├── 中国市场 + 仿制药/生物类似药 → A-8
    ├── 医美 + 单品 → B-1
    ├── 医美 + 组合复盘 → B-2
    ├── 消费健康（非医疗器械/药品）→ B-3
    ├── 商业化授权 + 国内未上市Biotech + 含股权条款 → A-2 + E-1
    ├── 纯股权投资（无商业化授权） → 集团投资流程 / E-1仅作参考附件
    ├── 国际市场 + 单品单市场 → C-1
    ├── 国际市场 + 单品多市场排序 → C-2
    ├── 国际市场 + 组合策略 → C-3
    └── 协议签署后治理 → 任意主技能 + 治理工具包
```
