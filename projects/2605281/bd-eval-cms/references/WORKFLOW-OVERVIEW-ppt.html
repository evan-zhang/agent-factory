03
两入口设计

规范层 vs 执行层

同一个 skill，两个不同的"读者"

SKILL.md

规范层

给 Agent 读 · 决定"做什么"
  • 触发词 + 11 步必做（Step 1~11）
  • D-0 路由（20 个技能之一）
  • 一票否决清单（8 条）
  • Gate 0~5 + Final 评估框架
  • 业务主体互斥规则
  • 财务硬门槛（3 类产品）
  • One-pager 终局先立
  • 报告骨架（HTML 麦肯锡风格）
EXECUTION.md

执行层

给 sub-agent 读 · 决定"怎么做"
  • Phase 1DISCOVERY · 5 次联网搜索
  • Phase 2D-0 路由 + 技能确认 Battle
  • Phase 3逐 Gate 深度评估（6-Gate）
  • Phase 4Gate Battle 对抗审查
  • Phase 5报告合并 + 质量终检
  • Phase 5.5HTML 生成 + 上传归档
  • 断点续跑（state.json 状态机）
  • 心跳超时 → 标记僵尸
04

Q2是否国际市场（港澳台/东南亚/GCC/澳新/拉美）？→ C-1 / C-2 / C-3

Q4是否为医美产品（注射/器械/能量源设备）？→ B-1 / B-2
Q5是否为消费健康/OTC？→ B-3
Q6是否为国内 Biotech（股权+商业化双引擎）？→ A-2 + E-1
Q7化学药品类型（3/4 类仿制 / 创新 / 代理 / 生物类似 / 已上市 / Rx→OTC）？→ A-1 / A-2 / A-4 / A-5 / A-6 / A-8
