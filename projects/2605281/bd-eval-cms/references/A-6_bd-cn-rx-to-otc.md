# A-6 院内驱动流向院外（处方药院内立足后院外延伸评估）

---

## 模块一：技能元信息

| 字段 | 内容 |
|------|------|
| **技能编号** | A-6 |
| **技能名称** | 院内驱动流向院外（Rx-to-OTC/院外延伸 投前评估） |
| **版本** | v1.0 |
| **描述** | 针对已在院内（医疗机构）建立稳定处方基础的处方药（Rx），评估向院外（院外DTP（直接面向患者）药店/零售药店/互联网医院/院外特药渠道）延伸的可行性与商业价值。核心评估逻辑：院内处方驱动是院外销售的根本基础；院外延伸是「增量」而非「替代」，必须保证院内基础不受损。涵盖院内/院外双渠道财务模型，评估互联网医院政策风险、DTP渠道可行性、零售终端适配性及院外增量成本结构。 |
| **触发条件** | 标的品种已在中国上市且具备院内处方基础；业务方向为向院外渠道（DTP/零售药店/互联网医院）延伸；用户说「院外延伸」「院外业务」「DTP渠道」「处方外流」「互联网医院」「零售药店」「OTC延伸」「院外增量」时触发 |
| **适用业务主体** | 院外业务（依托深康/德镁/维盛/天津康哲院内品种）；深康/德镁/维盛现有管线中适合院外延伸的品种 |
| **继承来源** | pharma-bd-due-diligence 14维度框架（渠道/准入维度）；finance-report-format-standard 财务缩写规范 |
| **商机接入路径** | 深康/德镁/维盛现有品种院外延伸评估 → A-6；A-2代理权评估发现院外机会 → A-6并行；A-0/A-7分拨院外类标的 → A-6 |

## 模块三：评估阶段映射（三阶段模型）

```
阶段A（公开信息）：One-pager院外终局先立 + Gate 1 + Gate 2
  └─ 核心问题：院内资质过关吗？院外政策合规吗？
  └─ 信息来源：NMPA批文/说明书、医保局政策、互联网医院政策、行业报告
  └─ 输出物：院外延伸可行性初判报告

阶段B（双轨并行）：Gate 3 + Gate 4 + Gate 5
  └─ 核心问题：院外患者需求有多大？渠道怎么铺？院外成本多少？
  └─ 双轨：渠道建设轨（DTP/零售接洽）+ 市场调研轨（患者需求/竞品院外现状）
  └─ 输出物：院外市场评估报告

阶段C（终审）：Gate 6
  └─ 核心问题：双渠道合并财务回报达标吗？院外增量NPV（净现值）>0吗？
  └─ 输出物：院外延伸决策建议书（双渠道财务模型）
```

### ▶ Gate 1：院内品种资质评估（前提门）

**1.1 品种基本条件核查**
- 是否具备处方药院外流转所需的基础资质（非管制药品/非特殊管理药品）？
- 说明书是否支持患者自管用药（无需医生实时监测的品种更适合院外）？
- 用药剂量是否固定且稳定（适合自我管理的患者群体）？
- 治疗周期是否为长期用药（慢性病患者院外便利性需求强）？

**1.2 院内处方稳定性评估**
- 院内处方量近12个月趋势（增长/稳定/下降）？
- 活跃处方医生数量及区域分布？
- 院内医保报销地位（甲类/乙类/自费/限定报销条件）？
- 是否有医院限制外带处方的政策风险？

**1.3 患者用药依从性特征**
- 患者是否具备自我管理能力（文化程度/年龄/病情复杂度）？
- 用药中断风险（不经医生提醒能否坚持用药）？
- 是否有患者随访/患者教育需求（影响院外服务模式设计）？

## 模块六：一票否决清单

1. **政策合规封闭**：品种属于互联网诊疗明确禁止品类且无其他合法院外渠道
2. **院内基础极度脆弱**：院内处方量持续下降（近12个月降幅≥20%），院外延伸无处方驱动基础
3. **患者自管能力缺失**：适应症患者无法进行自我管理用药（如需持续医疗监测的高风险品种）
4. **院外增量NPV（净现值）为负**：即便乐观情景下院外增量NPV（净现值）仍为负
5. **院内患者外流不可控**：评估显示院外延伸将导致院内收入损失>20%，合并后整体价值下降
6. **渠道费率侵蚀殆尽**：院外渠道费率+运营成本导致院外增量毛利率（毛利润/营业收入）<30%
7. **执行团队能力根本性缺失**：集团无院外业务团队且短期内无法组建
