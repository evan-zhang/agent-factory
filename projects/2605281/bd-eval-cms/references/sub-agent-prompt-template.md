# CMS 投前评估 — 子Agent Prompt 模板

> **用途**：Phase 3 逐 Gate 深度评估时，作为 sessions_spawn 的 task 参数基础。
> **版本**：v0.1

---

## 模板使用说明

本模板为 CMS 投前评估体系的 Gate 章节撰写子Agent 提供统一的 prompt 结构。

**使用方式**：
1. 复制下方模板
2. 替换 `{变量}` 部分为实际值
3. 通过 sessions_spawn 发送给子Agent

---

```
## 本 Gate 评估要点

{从技能定义文件中提取的本Gate评估要点，列出核心评估维度}

## 撰写规范

### 1. 结论卡（必须，放在章节开头）

```
【Gate {N}：{名称}】
结论：通过 / 条件通过 / 停止
置信度：高 / 中 / 低
关键支撑证据：
  1. ...
  2. ...
  3. ...
需补充证据 Top 5：
  1. ...（预计获取难度：高/中/低）
  2. ...
红旗事项：（如有）
下一步行动：
```

### 2. 置信度标注

对关键数据（市场规模、患者人数、竞品销售额、峰值销售预测等）：

| 等级 | 定义 | 标注方式 |
|------|------|---------|
| A级 | ≥2个独立来源交叉验证 | 无需标注 |
| B级 | 单一权威来源 | 标注来源名称，如（来源：IQVIA 2025）|
| C级 | 仅内部/合作方提供 | `[C级-待验证]` |
| D级 | 基于假设推算 | `[D级-基于假设]` |

**关键财务数据使用 C/D 级时**，必须在章节末尾添加风险提示段。

### 3. 信息来源标注

每段实质性内容必须标注来源：

- 正文内引用标注 `[REF-XXX]`，指向 `REFERENCES.md` 中的参考文献条目
- `来源：内部评估——[文件名]（日期）`（非公开来源时使用）
- `需进一步与[对象]沟通/确认`
- **禁止**：不允许出现“外网核查”“分析推断”等无具体 REF 编号的标注

**必须进行外网独立核查**，至少覆盖：权威指南、竞品注册状态、合作方合规历史、医保/准入政策、市场数据交叉验证。所有核查结果必须存入 REFERENCES.md 并在正文中引用。

### 4. 阶段标签

- 基于公开信息的内容标注 `[阶段A]`
- 基于深化数据的内容标注 `[阶段B]`
- 同一章节内两种内容都有时，分别标注

### 5. 财务缩写规范

首次出现必须标注中文全称：
- IRR（内部收益率）
- NPV（净现值）
- COGS（销售商品成本）
- rNPV（风险调整净现值）
- WACC（加权平均资本成本）
- EBITDA（税息折旧摊销前利润）
- VBP（带量采购）

### 6. 信息冲突处理

当内部评估与外网核查存在冲突时：
1. 两者并列呈现
2. 给出独立判断及依据
3. 以外网核查为基准
4. 添加冲突标记

### 7. 禁止事项

- 禁止灰色字体
- 禁止元描述（"本章节旨在..."）
- 禁止修订说明
- 禁止未标注来源的市场数据

## 搜索要求 + 参考文献采集

- 至少进行 3 次定向搜索（覆盖本 Gate 核心评估维度）
- **搜索前**：先读取现有参考文献文件 `{品种目录}/references/REFERENCES.md`，了解已抓取的 URL，避免重复
- **搜索后**：对每个有价值结果（含具体数据/结论）执行 `web_fetch` 抓取内容
- 新的参考文献追加到 `{品种目录}/references/REFERENCES.md`，编号续接现有最大编号
- 参考文献格式：
  ```
  ## [REF-XXX] 标题
  - **URL**: https://...
  - **抓取时间**: YYYY-MM-DD HH:MM TZ
  - **来源类型**: 官方|学术|行业媒体|公司公告|监管数据库
  - **内容摘要**: （200-500字，保留关键数据和原文表述）
  - **关键数据点**: （列出可用于报告的具体数字/结论）
  ```
- **正文引用**：每个数据点后标注 `[REF-XXX]`，示例：全球PBC患者约50万人[REF-012]
- **禁止**：不允许出现“外网核查”“分析推断”等无具体 REF 编号的标注
- **报告开头**：列出本 Gate 使用的参考文献编号清单，如：`本 Gate 引用：[REF-003] [REF-007] [REF-012]`
- 搜索来源记录在章节末尾
- 优先使用权威来源（NMPA/FDA/ClinicalTrials/IQVIA/学术文献）

## 输出

将完整章节内容写入：{输出文件路径}

文件命名格式：`Gate-{N}-{Gate英文名}.md`
示例：`Gate-1-premise.md`、`Gate-4-payment.md`

## 约束

- 全文简体中文
- 财务缩写首次标注全称
- 关键数据置信度标注完整
- 不超过 3000 字（不含参考文献）
```

## 验证子Agent Prompt 模板

```
你是 CMS 投前评估体系的交叉验证专家。

## 任务

验证品种「{品种名}」以下 Gate 章节的质量和一致性。

## 待验证章节

{列出本批次已完成的Gate章节文件路径}

## 验证检查项

1. **结论卡格式**：每个 Gate 是否有标准结论卡
2. **Gate 间一致性**：不同 Gate 引用的同一数据是否一致
3. **置信度合理性**：标注等级是否与数据来源匹配
4. **参考文献引用**：每个数据点是否有 `[REF-XXX]` 引用，是否有“外网核查”等无编号标注
5. **交叉引用**：Gate 6 是否正确引用了 Gate 1-5 的结论
6. **财务数据一致性**：不同章节的同一财务指标是否一致
7. **冲突标记**：内外部信息不一致处是否标记
8. **参考文献完整性**：正文引用的 REF 编号是否都在 REFERENCES.md 中存在

## 输出

列出发现的问题，按严重程度排序：
- 🔴 必须修正（数据矛盾/结论冲突）
- 🟡 建议修正（置信度不合理/来源缺失）
- 🟢 可选优化（表述优化/格式调整）

写入：{输出文件路径}
```

## One-pager 子Agent Prompt 模板

```
你是 CMS 投前评估体系的战略分析师。

## 任务

为品种「{品种名}」撰写 One-pager 终局先立文件。

## 背景

{品种基本信息摘要}

## 搜索要求 + 参考文献采集

- 至少进行 2 次定向搜索（覆盖品种核心信息维度）
- **搜索前**：先读取现有参考文献文件 `{品种目录}/references/REFERENCES.md`，了解已抓取的 URL，避免重复
- **搜索后**：对有价值结果执行 `web_fetch` 抓取内容，追加到 REFERENCES.md
- 正文引用数据点时标注 `[REF-XXX]`
- **报告开头**：列出 One-pager 使用的参考文献编号清单

## 终局五问（必须逐一回答）

1. **成功终局定义**：上市后第X年峰值销售额、市占率目标、对集团贡献
2. **核心价值主张**：凭什么在目标市场胜出（1-2句话，不允许罗列）
3. **最大风险单项**：若只有一个风险能击垮本项目，是什么
4. **止损边界**：在何种条件下应主动退出或终止
5. **决策人与时限**：谁来拍板，最晚何时须给出结论

## 输出

写入：{输出文件路径}/One-pager.md
```
