## 模块一：技能元信息

### 1.1 技能定位

A-8是CMS医药投前评估体系A群组（科学与商业双轨评估）的核心进阶技能，专门用于对以下三类中国境内产品引进标的进行**6-Gate门控全流程评估**：

1. **化学仿制药**（含5.2类、5.1类改良，3类境内首仿境外已上市）
2. **改良型新药**（4类：新适应症、新剂型、新复方等）
3. **生物类似药**（含单抗类似药、重组蛋白类似药）

A-8覆盖从科学可行性（Gate 1-3，阶段A）到商业可行性验证（Gate 4-6，阶段B）的完整评估闭环。

### 1.2 技能边界

| 适用场景 | 不适用场景 |
|---|---|
| 国内仿制药引进（已有或潜在BE数据） | 全球首创新药（NME）——转A-6 |
| 改良型新药（4类）引进 | 海外已上市/中国未上市创新药（5.1类）——转A-7 |
| 生物类似药引进 | 纯代理权引进（无注册权）——转A-3 |
| 境内首仿引进 | 院外/消费健康OTC品种——转A-5 |
| 同API不同剂型/规格/途径的改良引进 | 战略级平台合作——转A-9 |

### 1.3 默认输出模式（v2.0修订）

**v2.0核心规则：默认输出为阶段A+B综合评估报告（Gate 1-6全流程一份大报告）。**

- 首次应用A-8技能：**必须**输出完整版（A+B综合报告）
- 后续迭代修订：可按单Gate章节更新，但结论框架保持完整
- 仅当用户明确指令"只做阶段A快评"或"Gate 1-3初步评估"时，才输出阶段A单独报告
- 阶段A单独报告须在封面显著标注：`⚠️ 本报告为阶段A快评（Gate 1-3），阶段B商业可行性验证尚未完成，不构成引进决策依据`

### 1.4 技能触发规则

以下任一条件满足即触发A-8：

- 用户提及仿制药引进、BE研究引进、首仿引进、改良型新药引进、生物类似药引进
- 用户提及5.2类/3类/4类/生物类似药的投前评估
- 用户说"帮我评估这个仿制药"、"这个改良型新药值不值得引进"、"生物类似药投前报告"
- 涉及CMS业务主体下的非创新药产品引进决策

## 模块二：通用前置章节规范（M-01 ~ M-10）

> 以下M编号章节为CMS 投前评估体系的通用规范，在A-8中全部适用，并在本模块记录A-8专属扩展规则。

### M-01 信息来源标注规范（v2.0新增来源三分法）

**强制规则：报告中每一段实质性内容必须标注信息来源类型。**

#### 来源三分法

| 标注符 | 含义 | 使用规则 |
|---|---|---|
| `[内部评估]` | 来自CMS内部评估文件、投前访谈、合作方提供资料 | 必须标注文件名+日期，如：`[内部评估: 投前初评报告_2026-04-10]` |
| `[外网核查]` | 来自公开数据库、权威指南、官方政策文件、文献 | 必须标注引用编号[N]，参考文献列于附录C，如：`[外网核查][3]` |
| `[待确认]` | 尚未经外部核查，需与合作方沟通或需CDE确认 | 标注待确认路径，如：`[待确认: 需合作方提供BE批件原件]` |

#### 置信度分级（与来源标注联动）

| 级别 | 定义 | 标注方式 |
|---|---|---|
| A级 | ≥2个独立来源交叉验证 | 无需特殊标注（默认最高可信度） |
| B级 | 单一权威来源（官方数据库、CDE公示、权威指南） | 标注来源即可 |
| C级 | 仅内部调研/投前访谈，无外部佐证 | 正文标注 `[C级-待验证]` |
| D级 | 推算/估算，基于假设或模型输出 | 正文标注 `[D级-基于假设]` |

**关键规则：当关键财务假设（如市场规模、峰值销售额、集采降幅假设）依赖C级或D级数据时，必须在执行摘要的风险框中明确标出。**

### M-02 信息冲突管理规范（v2.0新增）

#### 冲突识别与标记

```

│  ⚠️ [冲突标记] 本段涉及信息冲突，详见附录C冲突序号#N  │
│  内部口径：XXXX                                      │
│  外网核查：YYYY                                      │
│  本报告采纳立场：ZZZZ（附理由）                       │
└─────────────────────────────────────────────────────┘
```
（HTML渲染时使用黄色边框 conflict-box 组件，边框色 #f9a825，背景色 #fffde7）

#### 附录C信息冲突汇总表（必填）

| 冲突序号 | 冲突涉及内容 | 内部口径 | 外网核查结论 | 本报告采纳立场 | 理由 |
|---|---|---|---|---|---|
| #1 | （填写） | （填写） | （填写） | （填写） | （填写） |

**无冲突时，附录C填写"本报告评估过程中，内部评估与外网核查未发现实质性信息冲突。"**

### M-03 外网独立核查强制要求（v2.0新增中立审查规则）

**强制规则：A-8评估过程中必须进行外网独立核查，至少覆盖以下5个维度：**

1. **指南推荐维度**：国际/国内权威临床指南对本品适应症的推荐级别（GRADE证据等级）
2. **竞品注册状态维度**：已受理/已上市/撤市的同适应症竞品注册状态（CDE药品数据库查询）
3. **合规历史维度**：标的原研方/供应商的警告信、召回记录、FDA/EMA审查历史
4. **医保政策维度**：现行医保目录收录状态、历史国谈价格走势、集采状态
5. **市场数据维度**：至少一个独立第三方市场数据来源（如PDB、IMS/IQVIA、米内网）

**中立审查框（neutral-box）组件规范：**
```
╔═══════════════════════════════════════════════════════╗
║  🔍 中立审查结论                                      ║
║  外网核查已覆盖上述5个维度                            ║
║  与内部评估一致之处：XXXX                             ║
║  与内部评估存在差异之处：YYYY                         ║
║  中立结论：ZZZZ                                       ║
╚═══════════════════════════════════════════════════════╝
```
（HTML渲染时使用深蓝色边框 neutral-box 组件，边框色 #1a3a5c，背景色 #e8eaf6）

**禁止性规定：报告结论部分禁止出现"与投前初审一致"、"与合作方材料吻合"、"内外部评估结果相符"等暗示立场预设或确认偏误的描述。结论必须基于独立核查后的中立判断。**

### M-04 一票否决核查规范

一票否决清单见模块六，Gate 8（第8章）为固定核查节点，任何一票否决项触发即终止评估并出具否决报告。

### M-05 财务内控标准（硬编码）

详见模块五。财务内控标准不因商务谈判或内部偏好而调整。

### M-06 止损三件套规范

|---|---|---|
| ①首付款上限 | 基于最差情景财务模型确定的可承受首付上限（元/USD） | （填写） |
| ②里程碑付款与退出条款 | 里程碑未达成时的退款机制或合同解除条款 | （填写） |
| ③销量对赌条款 | 若3年累计销售额低于阈值X的处理机制 | （填写） |

### M-07 紧急提醒横幅规范（v2.0新增）

**触发条件：存在≤6个月内必须完成的关键前置动作时，强制在封面三开关下方或执行摘要前显示紧急提醒横幅。无紧急事项时不显示。**

典型触发场景：

- 竞品注册窗口（竞品预计获批时间节点前6个月内）
- 合作方报价有效期
- 国谈申报窗口截止日期

### M-08 报告Three-Switch（三开关）规范

Three-Switch是报告封面的决策信号灯，是执行层阅读报告的第一视觉入口：

| 开关 | 选项 | 说明 |
|---|---|---|
| 一票否决项 | ✅ 无触发 / ❌ 触发（列明） | 触发则报告直接输出否决结论 |
| 止损三件套 | ✅ 已全部谈定 / 🟡 谈判中（列明缺项） / ❌ 未启动 | |
| 财务达标 | ✅ 全部达标 / 🟡 部分达标（列明未达标项） / ❌ 不达标 | 对应模块五财务内控标准 |

### M-09 数据置信度与来源标注联动规则

- 报告中所有市场规模、峰值销售额、定价假设、集采降幅假设数字，必须在首次出现时标注置信度级别
- 财务模型中的关键输入参数，必须在附录A的参数说明列中逐项标注来源和置信度
- C级/D级假设必须同时进行敏感性测试（详见M-10）

### M-10 敏感性测试规范

关键财务假设的敏感性测试为**强制要求**（非可选），最低要求覆盖：

1. **市场规模±20%** 对峰值销售额的影响
2. **集采/国谈降幅+10%** 对净利润率（净利润率=净利润/营业收入×100%）的影响
3. **首付款±X00万** 对回收期的影响

定价敏感性测试详见Gate 4（第5章）子模块4.3。

## 模块三：评估阶段映射（v2.0修订）

### 3.1 默认综合输出模式

```
┌──────────────────────────────────────────────────────────────────┐
│                    A-8 v2.0 评估流程总览                          │
│                   【默认：阶段A+B综合输出】                       │
├─────────────────────────┬────────────────────────────────────────┤
│  阶段A：基础可行性评估   │  阶段B：商业可行性深化验证             │
│  信息来源：公开+快速内评 │  信息来源：深化数据+谈判文件           │
├─────────────────────────┼────────────────────────────────────────┤
│  Gate 1 注册路径 [阶段A] │  Gate 4 价格走廊·准入策略 [阶段B]     │
│  Gate 2 IP·四层壁垒 [A]  │  Gate 5 成本走廊·底价探明 [阶段B]     │
│  Gate 3 医学价值·竞争[A] │  Gate 6 交易结构·止损条款 [阶段B]     │
│  第4章 同API专项对比 [A/B]│  第8章 一票否决·综合结论              │
└─────────────────────────┴────────────────────────────────────────┘
```

### 3.2 阶段A与阶段B的信息深度差异

| 维度 | 阶段A（Gate 1-3） | 阶段B（Gate 4-6） |
|---|---|---|
| 主要信息来源 | CDE公开数据库、文献、指南、合作方初始材料 | BE报告/注册文件、供应商报价、谈判条款草案 |
| 市场数据深度 | 公开市场报告+米内网/PDB摘要 | 医院流向数据、实地调研、KOL访谈 |
| 财务模型精度 | 范围估算（±30%） | 基准模型（±15%） |
| 决策用途 | 立项决策（进入/放弃Gate 4-6） | 引进决策（签约/不签约） |
| 典型耗时 | 5-10个工作日 | 10-25个工作日 |

### 3.3 阶段A单独快评输出规则（非默认）

仅在以下情形输出阶段A单独报告：
- 用户明确指令：**"只做阶段A"、"Gate 1-3快评"、"初步筛查报告"**
- 快评报告必须在封面和执行摘要显示以下警示框：

```
⚠️ 本报告为A-8阶段A快评（Gate 1-3），阶段B商业可行性验证尚未完成。
本报告仅供立项决策参考，不构成最终引进决策依据。
若立项决策通过，须在Gate 4-6完成后输出完整A+B综合报告。
```

## 模块五：财务内控对照表（硬编码，不可修改）

> 以下标准为CMS 投前投委会决策硬门槛，任何条线的评估报告均不得修改或豁免。

| 品种类型 | IRR（内部收益率）| 回收期 | 净利润率（净利润率=净利润/营业收入×100%）| 毛利率 | 首年盈利要求 | 峰值销售额 |
|---|---|---|---|---|---|---|
| 一般化学仿制药（集采品种） | ≥50% | ≤3年 | 5年均≥20% | ≥40%（集采后） | 必须实现 | ≥1亿元 |
| 短期不集采品种 | ≥35% | ≤3.5年 | 5年均≥20% | ≥55%（非集采状态） | 必须实现 | ≥5000万元 |
| 改良型新药（4类） | ≥30% | ≤4年 | 5年均≥20% | ≥60% | 上市2年内实现 | — |
| 生物类似药 | ≥25% | ≤5年 | 10年均≥20% | ≥60% | 上市3年累计盈利 | — |

**财务缩写说明（本表首次出现，后文不再重复注释）：**
- IRR：Internal Rate of Return（内部收益率）
- 净利润率：净利润/营业收入×100%
- 毛利率：（营业收入-营业成本）/营业收入×100%
- NPV：Net Present Value（净现值）
- rNPV：risk-adjusted Net Present Value（风险调整后净现值）
- EBITDA：Earnings Before Interest, Taxes, Depreciation and Amortization（息税折旧摊销前利润）
- CAPEX：Capital Expenditure（资本性支出）

**财务内控标准适用规则：**
1. 品种类型判断以**提交投委会时的注册状态**为准
2. 若同一产品兼有仿制和改良属性，按更严格的标准执行
3. 境内首仿获批后3年数据保护期内按"短期不集采"标准；保护期届满后按"集采品种"标准建模
4. 生物类似药的集采情景需单独建模，不可与化学仿制药合并使用同一模板

| 保护层 | 机制 | 期限 | 对仿制药进入的影响 |

## 版本修订记录

### v1.0 → v2.0 变更摘要

**变更日期：** 2026-05-28
**变更主导：** CMS医药投前评估体系技能架构组

---

#### 核心修订（6项）

| 修订项 | v1.0 | v2.0 |
|---|---|---|
| **核心修订1：输出模式** | 阶段A和阶段B分别输出，默认输出阶段A报告 | **默认输出阶段A+B综合报告**；首次应用必须输出完整版；仅用户明确指令时输出阶段A快评 |
| **核心修订2：同API专项对比** | 无独立章节，散落于竞品分析 | **新增第4章**（Gate 3.5升为独立章节，编号为第4章，Gate 4-6顺延为第5-7章）；固定4节结构：基本差异→竞争维度→医保准入→综合定位；同API不同剂型/规格/途径必填 |
| **核心修订3：来源标注与冲突管理** | 无来源标注规范，无冲突管理 | **来源三分法**（[内部评估]/[外网核查]/[待确认]）全报告强制标注；**附录C信息冲突汇总表**必填；冲突标记 conflict-box 视觉组件 |
| **核心修订4：中立客观审查** | 无强制外网核查要求 | **强制外网独立核查5个维度**；**neutral-box**中立审查框每Gate必填；禁止"与投前初审一致"等立场预设描述 |
| **核心修订5：执行摘要结构** | 结构不固定，可能含财务详表 | **固定五部分结构**（总评结论→基本情况→关键风险→待确认事项→止损三件套）；**禁止在执行摘要放财务详表**（财务详表移至附录A） |
| **核心修订6：Gate 3竞争格局** | 简版竞品分析 | 成熟品种**扩写为3.3节六子模块**（竞品全景图/同API深度对比/替代疗法威胁/市场容量验证/竞争者时间表/中立SWOT）；触发条件明确 |
