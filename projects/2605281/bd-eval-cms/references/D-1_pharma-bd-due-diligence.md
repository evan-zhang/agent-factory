## 模块4：评估章节结构（Gate 编排）

### 总体编排原则

```
One-pager 终局先立（锚定门）
    ↓
Gate 0：商机申报与快扫（准入门）—— 能否立项？
    ↓
Gate 1：关键假设前提验证（前提门）—— 核心假设是否成立？
    ↓
Gate 2：科学与技术可行性（技术门）—— 技术风险是否可控？
    ↓
Gate 3：商业化可行性（商业门）—— 市场与推广逻辑是否成立？
    ↓
Gate 4：财务与合规可行性（价值门）—— 财务回报是否达标？
    ↓
Gate 5：合同条款与交割可行性（可做门）—— 交易是否可以执行？
```

> D-1的Gate 0~5为体系主门控。A/B/C/E模板如使用本地Gate名称，必须在报告中映射回D-1 Gate 0~5；未映射的本地章节不得视为已完成体系审查。

### Gate 0：商机申报与快扫（准入门）

**目标：** 判断是否值得投入资源进行深度评估。

| 评估项 | 权重 | 评估方法 |
|--------|------|---------|
| 战略契合度（与集团/业务主体战略方向匹配度） | 高 | 对照年度 投前 优先清单 |
| 产品差异化潜力（vs 现有竞品） | 高 | 公开竞品数据比较 |
| 合作方可信度初判 | 中 | 公开信息/监管数据库核查 |
| 财务规模初步可行性（峰值销售 vs 预期首付款比例） | 中 | 行业经验类比 |
| 明显一票否决项检查 | 高 | 对照否决清单 |

**Gate 0 结论格式：**
- 结论：**立项 / 暂缓 / 否决**
- 置信度：高（>80%）/ 中（50-80%）/ 低（<50%）
- 需补证据 Top 5：（具体列出5条）

### Gate 1：关键假设前提验证（前提门）

**目标：** 验证支撑商业逻辑的3~5个核心假设是否成立。

| 典型关键假设类型 | 验证方法 | 数据来源 |
|----------------|---------|---------|
| 目标患者人群规模假设 | 流行病学文献 + KOL 访谈 | D-2 市场全景 |
| 监管路径假设（注册时间/类别） | NMPA 公开数据 + 注册顾问意见 | M-10 + 注册维度 |
| 竞品格局假设（进入时机窗口） | 竞品研究 + 医保谈判时间线 | D-2 + 公开数据库 |
| 定价假设（参照系及可及性） | 同类产品医保价 + 国际价格比较 | 公开药品价格数据库 |
| 合作方履约能力假设 | 已有合作伙伴访谈 + 公开案例 | M-03 + M-07 |

### Gate 2：科学与技术可行性（技术门）

**评估范围：** 维度1（科学与风险）+ 维度2（科学与商业）+ 维度7（注册策略）+ 维度9（生产策略）

| 子评估项 | 核心问题 |
|---------|---------|
| 作用机制合理性 | MoA 是否有充分临床前/临床证据支持？ |
| 临床数据质量 | 设计、统计、终点是否符合当前监管标准？ |
| 安全性档案 | SAE 发生率、重要提示/禁忌是否可接受？ |
| 注册可行性 | NMPA 审评路径是否明确？时间线是否可预测？ |
| CMC 状态 | 生产工艺是否成熟？供应链是否稳定？ |

### Gate 3：商业化可行性（商业门）

**评估范围：** 维度3（市场调查）+ 维度4（市场预测）+ 维度5（推广规划）+ 维度11（准入策略）

| 子评估项 | 核心问题 |
|---------|---------|
| 市场规模验证 | 目标患者人群是否经过第一手调研验证？ |
| 市场份额可得性 | 竞品格局下，可实现份额假设是否合理？ |
| 渠道与推广策略 | 与集团现有商业化能力是否匹配？ |
| 医保准入路径 | 医保纳入时间线、谈判价格区间是否合理？ |
| 上市后医学策略 | RWE/HEOR 规划是否支持市场准入论据？ |

### Gate 4：财务与合规可行性（价值门）

**评估范围：** 维度6（财务）+ 维度12（法律）+ 维度13（专利）+ 维度14（商标）

| 子评估项 | 核心问题 |
|---------|---------|
| NPV/rNPV（净现值/风险调整净现值） | 项目价值与首付款、里程碑及风险投资额是否匹配？ |
| IRR（内部收益率） | 是否达到2506AARFY对应产品类型和上市年限门槛？ |
| 回收期 | 投资回收期是否达到2506AARFY对应上限？ |
| 合同结构合规性 | 排他条款、回购条款、竞业限制是否有法律风险？ |
| 专利悬崖风险 | 核心专利到期时间与销售峰值的时序关系？ |

### Gate 5：合同条款与交割可行性（可做门）

**评估范围：** 维度8（医学研究策略）+ 维度10（供应链策略）+ TS 条款审查

| 子评估项 | 核心问题 |
|---------|---------|
| TS 关键条款可谈判性 | 首付款、里程碑、分成比例是否在合理范围？ |
| 独占权范围 | 地域独占、适应症独占是否覆盖商业化需求？ |
| 供应链交割 | API（活性药物成分）/制剂供应协议是否稳定？ |
| 临床/注册里程碑合理性 | 里程碑支付时间点是否与监管进度挂钩？ |
| 退出机制 | 如项目失败，终止条款是否对集团公平？ |

## 模块5：财务内控对照表

> D-1只负责尽调核查和证据闭环，不自行设定或调整集团财务硬门槛。
> 财务硬门槛以《投前评估关键要素与问题引导清单(供参考)_财务附件.xlsx》（`投前评估关键要素-财务维度 [2506AARFY]`）及总规则第5节为准；源表未单列的品类，须由财务部门确认参照口径后方可形成D-3 Final结论。

### 财务内控核查项

| 核查项 | 要求 | 输出 |
|--------|------|------|
| 产品类型归类 | 明确创新药/仿制药/医美/源表未单列品类 | 财务参照口径 |
| 风险投资额 | 按2506AARFY定义拆分首付款、里程碑、临床/注册/上市前投入 | 风险投资额测算表 |
| IRR（内部收益率） | 对照2506AARFY对应产品类型和上市年限 | 达标/不达标/待财务确认 |
| 投资回收期 | 按2506AARFY口径，从首个销售年度至累计折现现金流为0 | 回收期核查 |
| 5年/10年风险投资回报倍数 | 创新药按5年、10年倍数核查；仿制药/医美按源表对应预测周期核查 | 倍数核查 |
| 毛利率/净利润率/盈亏平衡期 | 仅按源表列明产品类型适用；未列明不得自行新增硬门槛 | 经营质量核查 |
| 产品系数/综合财务等级 | 按总规则§5.5/§5.6补充评分 | A+/A/B/C/D等级或待评 |

### 财务模型关键假设标准化

| 假设参数 | 标准化方法 | 敏感性测试范围 |
|---------|-----------|--------------|
| 折现率（WACC，加权平均资本成本） | 集团统一 WACC 基准 ±2% 调整 | ±3% |
| 峰值销售预测 | 基于 TAM（目标可及市场）× 份额假设 | -30% / +20% |
| 专利保护期 | 以最窄保护专利到期年计算 | -2年悬崖情景 |
| 医保纳入时间 | 基准情景+1年、悲观情景+3年 | +1/+3年 |
| 里程碑成功率折现 | 按临床阶段分层、EOP2/监管沟通状态、TA基准和证据质量校准PoS；不得用单一PhII→PhIII均值替代项目判断 | 各阶段±10%，并列EOP2前/后敏感性 |

## 模块6：一票否决清单

> 以下任何一条成立，项目立即终止评估，进入否决记录。
> 否决结论须在红旗风险台账中注明具体触发条款及证据来源。

### 合规与法律类（硬性否决）

| 编号 | 否决条件 | 核查方法 |
|------|---------|---------|
| VK-01 | 合作方在近5年内收到 FDA / NMPA / EMA 警告信（Warning Letter）且未完成整改 | 监管机构公开数据库 |
| VK-02 | 产品或合作方涉及未解决的刑事诉讼或重大民事欺诈指控 | 律所尽调报告 |
| VK-03 | 合作方实际控制人或核心管理层在制裁名单中（OFAC/BIS/其他） | OFAC、BIS 数据库核查 |
| VK-04 | 知识产权归属存在争议且合作方无法提供明确权属证明 | 专利局数据库 + 律所意见 |
| VK-05 | 合同中含有强制仲裁条款且仲裁地不在集团可接受司法管辖范围 | 法务审查 |
| VK-06 | CMS已签协议/NDA明确禁止接触或评估同适应症/同机制/同API/同渠道竞品，且无法取得书面豁免 | CMS合同台账 + 法务审查 |

### 科学与安全类（硬性否决）

| 编号 | 否决条件 | 核查方法 |
|------|---------|---------|
| VK-07 | 关键临床试验存在数据造假指控（经监管机构认定或可信第三方报告） | 监管公告 + 文献核查 |
| VK-08 | 产品存在无法通过说明书管理的黑框警告（Black Box Warning）且涉及致死性 SAE | FDA/NMPA 说明书核查 |
| VK-09 | 关键适应症已被更安全/有效竞品占据>70%市场份额且无差异化路径 | 竞品市场分析（D-2） |
| VK-10 | 生物类似药参照药已在评估市场内失去专利保护且市场价格崩塌 | 市场价格数据 |

### 财务类（硬性否决）

| 编号 | 否决条件 | 核查方法 |
|------|---------|---------|
| VK-11 | 基准情景下 rNPV（风险调整净现值）为负值 | 财务模型 |
| VK-12 | 首付款超过集团单项目预算上限且无法通过联合融资覆盖 | 财务部门确认 |
| VK-13 | 合作方要求付款后提供数据，且拒绝任何形式的里程碑退款保障机制 | TS 条款审查 |

### 战略类（软性否决，需高管层确认豁免）

| 编号 | 否决条件 | 豁免条件 |
|------|---------|---------|
| VS-01 | 产品适应症与集团当前战略重点完全无关联 | CEO/COO 书面豁免 |
| VS-02 | 合同要求集团提供超过现有能力的临床/注册支持且无合理过渡期 | 投前委员会豁免 |

| 资料类型 | 阶段A（DRL V0） | 阶段B（DRL V1） |

### 临床阶段分层评估底座（强制）

> 凡处于I期至NDA/BLA前后的国内或海外临床阶段药品，不得只写"临床阶段"或"II/III期积极"。必须先完成阶段分层，再决定医学、注册、CMC、财务和交易口径。EOP2后代表关键III期方案、统计计划和CMC准备度获得较高确定性讨论基础，但不等同于疗效已经确证。

#### C. 分层使用规则

- I期、IIa和EOP2前项目原则上不得用"上市前成熟资产"财务口径；除非已有同机制强验证、突破性资格或监管书面反馈，仍须列为上行情景。
- EOP2后项目必须取得监管沟通纪要、书面反馈或等效证据，且III期终点、对照、人群、样本量、统计分析和CMC准备度均可解释，方可在基准情景提高PoS。
- III期进行中项目的主要风险从"科学假设"转向"研究执行、数据完整性、入组/事件、统计显著性、CMC商业化和标签边界"，报告必须相应调整颗粒度。
- 多适应症项目须先用总规则§2.2确定主评估适应症；未被证据支持的潜力适应症只能列为上行情景，不得支撑主承接主体或基准财务模型。
- 海外临床数据进入中国评估时，必须叠加ICH E17/MRCT、ICH E5外推、CDE沟通、桥接研究和中国SOC/准入差异判断；不能只因海外进入III期或EOP2后就默认中国可注册。
- 国内合作方或CMS自主持有的临床阶段项目，必须叠加《国内临床项目注册与执行核查卡》；不能只因国内已进入II期、III期或NDA前就默认注册路径、数据质量或商业化授权成熟。

- 关键PK、PD、ADA/NAb或BE结论来自未验证、验证不完整或不适用于样本基质/浓度范围的方法。

### HTML 报告格式规范（D-1 深化报告阶段适用）

**视觉规范：**
- 主色调：深蓝 `#1a3a5c`（标题栏、表头、分隔线）
- 辅助色：浅蓝 `#e8f0f8`（交替行背景）、橙黄 `#f5a623`（警示/红旗标记）
- 字体：正文 14px，标题 18-22px，表头加粗
- 禁止使用灰色字体（`#999999` 及以下均不可用）
- 禁止生成元描述段落（如"本报告旨在……"等开头套话）

**内容规范：**
- 所有财务缩写须在首次出现时标注中文全称，例如：IRR（内部收益率）、NPV（净现值）、rNPV（风险调整净现值）、WACC（加权平均资本成本）、MOIC（投资回报倍数）、FCF（自由现金流）、EBITDA（息税折旧摊销前利润）、CAPEX（资本支出）、DSO（应收账款周转天数）、ROIC（投入资本回报率）

- 每个 Gate 结论须包含：结论陈述 + 置信度（%）+ 置信度来源说明 + 需补证据 Top 5
- 红旗项须用橙黄色警示背景标记，一票否决项须用红色（`#c0392b`）标记
