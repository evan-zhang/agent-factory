## 模块3：评估阶段映射

### 三阶段模型概述

```
阶段A：商机发现 → Gate 0 初筛 → 具体产品初审评估报告 → 立项会决策（公开信息域）
          ↓
     NDA 弹性签署节点（不固定阶段，由合作方保密要求驱动；签署前须完成竞品约定与信息隔离初筛）
          ↓
阶段B：立项后双轨并行
        技术深化评估线 ‖ 商务谈判线（财务在 TS 阶段进场）
          ↓
阶段C：D-3 Final投委会终审 → 批准 / 条件批准 / 否决
```

### D-0 介入阶段

| 阶段 | D-0 角色 | 输出深度 |
|------|----------|---------|
| **阶段A（Gate 0 之前）** | 主要介入阶段。接收商机信息，完成路由决策，输出推荐技能+串接方案 | 路由决策单（1~2页） |
| **阶段B** | 如评估过程中产品类型或业务主体发生变化，可重新触发路由 | 更新版路由决策单 |
| **阶段C** | 不介入（已由 D-3 承接终审包） | 不适用 |

### 第一层：产品类型识别

```

  │     ├─ 含股权条款 + 国内未上市Biotech → A-2 + E-1
  │     └─ 不含股权或仅商业化合作 → A-2
  │

  │     ├─ 海外已上市、拟中国注册/商业化 → A-1
  │     └─ 海外I期/IIa/IIb/EOP2前/EOP2后/III期/NDA-BLA前后，拟引进中国权益 → A-1 + D-1临床阶段分层评估卡 + 在研产品综合评估卡（涉及PK/PD、桥接或免疫原性时加BA验收卡）

  │     ├─ 化学仿制药/改良仿制/高壁垒仿制 → A-8（天津康哲为主）
  │     └─ 治疗用生物制品3.3类生物类似药 → A-4
  │
  ├─ 医美产品
  │     ├─ 单标的评估 → B-1
  │     └─ 组合复盘 → B-2
  │
  ├─ 消费健康 / 院外品种
  │     ├─ 纯OTC/消费健康/无院内基础 → B-3
  │     ├─ 已有院内基础，向院外/DTP/互联网医院延伸 → A-6
  │     └─ 体系内成熟已上市品种做院外再激活/轻投入试点 → 主轴A（做/不做速判，按产品类型财务源表） + 辅轴A-6/B-3（院外执行与单位经济模型）
  │
  ├─ 多标的横向比较 / BIO展会批量筛选
  │     └─ → A-7（必要时先A-0扫描）
  │
  ├─ 国际市场（港澳台/东南亚/GCC/澳新/拉美）

  │     └─ 组合策略 → C-3
```

### A-2 vs A-5 边界判断
- 产品是否已在中国上市（有NMPA批文+销售历史）？
  - 是 → A-5（已上市产品推广权引进）
  - 否（在研/NDA阶段/未上市）→ A-2（国内合作·成熟阶段产品）
- 合作方是国内企业还是海外企业？
  - 国内 → A-2 或 A-5（按上市状态区分）
  - 海外 → A-1（海外产品License-in）或 A-5（进口产品已在中国上市）

### 第二层：业务主体匹配

D-0不得只输出泛化"业务主体"。路由结论必须拆成：
`主评估适应症 / 首发或当前注册适应症 / 潜力适应症 / 主承接主体 / 专业会签主体（可为无） / 专业输入或资料支持主体（如适用，非承接） / 渠道叠加主体 / 国际叠加主体 / 归口置信度 / 争议点`。

深康、德镁、维盛均为院内经营主体，默认按细分治疗领域、主评估适应症、核心处方科室和主要商业化资源确定一家院内主承接主体，不默认形成主辅协同。

#### 多适应症主评估适应症校准（强制）

多适应症项目必须先做"适应症机会账"，再做主体归口。D-0不得因一个产品覆盖多个适应症而输出多个院内主承接主体。

| 校准项 | D-0最低输出 |
|---|---|
| 候选适应症清单 | 每个适应症的TA、主要处方科室、当前证据/注册状态、预计上市顺序、市场潜力、竞争格局和准入路径 |
| 主评估适应症 | 选择可被证据支撑的商业潜力最大、战略价值最高、且CMS可承接能力最强的适应症 |
| 首发/当前注册适应症 | 仅说明路径，不自动决定主承接主体；若与主评估适应症不同，必须列"双轨说明" |
| 潜力适应症 | 证据不足时只可作为上行情景或待验证项，不得进入基准模型 |
| 主承接主体 | 按主评估适应症的核心处方科室、患者路径、P&L归属和销售资源承接能力确定 |
| 会签触发 | 次要适应症显著影响医学、准入、销售预测、既有组合或资源配置时触发；否则填"无" |

**主体组合合法性先验校验（强制）：**
- 院内路径必须先二选一：`直营路径（深康/德镁/维盛三选一）` 或 `天津康哲代理/经销路径`。
- 深康、德镁、维盛不得与天津康哲同时承接同一产品的院内推广；不得输出"直营 + 天津康哲协同"。
- 院外业务中心是渠道叠加层，可叠加于深康、德镁、维盛或天津康哲中的任意一个院内主体。
- 如需其他主体提供非承接性的资料、医学、注册或市场输入，应记录为"专业输入/资料支持"，不得写入承接主体方案。

| 适应症/产品领域 | 默认主承接主体 | 条件会签/叠加触发 | 判断口径 |
|----------------|-------------|-------------|-------------|
| 心肾代谢 / 消化肝病 / CNS与精神 / 综合内科慢病 | 深康 | 眼底/耳鼻喉并发症成为关键场景时可会签维盛；皮肤科入口或皮肤安全性管理足以影响结论时可会签德镁 | 以内科处方、医院慢病管理和深康商业平台为主 |
| 皮肤健康药品 / 皮肤科处方药 / 外用局部治疗 | 德镁 | 主评估适应症为皮肤时默认德镁主承接；若仅首发适应症为皮肤但主要商业潜力转向系统性免疫/全身慢病，须按主评估适应症重新判断并可会签深康；院外复购/DTC成规模时可叠加院外业务 | 以皮肤科处方、皮肤健康组合和德镁药品线为主 |
| 医美（注射/器械/能量源/耗材） | 德镁医美 | 机构渠道、DTC或消费健康打法占比较高时可叠加院外业务；组合风险复盘走B-2 | 医美走B-1/B-2，不并入A群药品 |
| 眼科 / 耳鼻喉专科药械 | 维盛 | 系统性疾病共管显著影响医学策略、准入路径或销售预测时可会签深康；院外复购/续方成规模时可叠加院外业务 | 以专科医生、专科渠道和维盛覆盖能力为主 |
| 高壁垒仿制 / 改良仿制 / 生物类似药 | 天康为项目平台或代理主体 | 对应TA主体仅可提供非承接性专业输入；若直营TA主体实际承接院内推广，则天津康哲不得同时作为院内承接主体 | 产品属性决定A-8/A-4；商业化模式需另行确认 |
| 已有院内处方基础，向DTP/零售/互联网医院延伸 | 原院内主承接主体不变 | 院外业务作为渠道叠加层；是否叠加取决于院外场景是否构成关键增量 | A-6是渠道叠加，不替代TA归口 |
| OTC/DTC/消费健康，无明确院内处方基础 | 院外业务 | 仅当皮肤健康、专科医学背书或既有院内处方基础显著影响打法时，可请对应院内主体会签 | 走B-3，不走A-6 |
| 国际市场（HK/SEA/GCC/澳新/拉美） | 康联达 Rxilient | 国内对应TA主体仅在中国资料、医学判断或供应协同会影响国际评估时会签 | 国际是市场层，不替代国内TA判断 |

**归口优先级：**
1. 监管身份和产品属性先决定主技能（A/B/C/E）。
2. 主评估适应症、主要处方科室和患者路径决定国内TA主承接主体；深康、德镁、维盛之间默认只选一个院内主承接主体。
3. 先完成主体组合合法性核查：直营院内主体与天津康哲不得在同一产品院内推广上并列承接。
4. 渠道只作为叠加层；院外、DTC、国际市场不得覆盖TA归口。
5. 如存在CMS既有组合、已签竞品限制、排他/优先权或NDA信息隔离义务，先输出Green/Yellow/Red，再决定是否继续路由。
6. 多适应症或模糊项目必须先判断主评估适应症、首发/当前注册适应症、近期收入贡献、核心处方科室、P&L归属和销售资源可承接性；未触发实质影响时，专业会签主体填"无"，不得写成泛化的"深康/德镁/维盛协同"。

### 第三层：评估阶段判断

| 用户提供信息量 | 判断为阶段 | D-0 行动 |
|---------------|-----------|---------|
| 仅产品名+适应症 | 阶段A（Gate 0前） | 输出 Gate 0 快扫模板 + 推荐技能 |

| 有 TS 草案 + 财务模型 | 阶段C | 输出 D-3 投委会包启动指令 |

## 模块5：财务内控对照表

> **D-0 路由器不执行财务评估，无财务内控门槛。**
> 财务内控标准由下游技能（D-1 基座财务维度及对应A/B/C/E主技能）承接。
> D-0 仅记录用户提供的初步财务规模（M-09）用于辅助路由判断（大型平台项目 vs 单品种引进）。

---

## 模块6：一票否决清单

> **D-0 路由器不执行否决判断，无一票否决条件。**
> 一票否决清单由 D-1 基座（红旗风险台账）及各主技能执行。
> D-0 的职责是识别"路由到哪个技能"，而非"是否应该否决"。

## 模块7：Gate 阶段说明

| Gate 编号 | 阶段名称 | D-0 在此 Gate 的角色 | 输出深度要求 |
|-----------|---------|---------------------|-------------|

## 模块9：输出规范

### 路由决策单格式规范

D-0 输出为**结构化文本/Markdown 表格**，不生成 HTML 报告。

**必含字段：**
1. 路由编号（自动生成：D0-YYYY-NNN）
2. 标的基本识别信息（M-01 ~ M-06 摘要）
3. 产品类型判断结论 + 置信度（高/中/低）
4. 多适应症主评估适应症校准（主评估适应症 + 首发/当前注册适应症 + 潜力适应症 + 选择理由）
5. 主承接主体 + 专业会签主体 + 专业输入/资料支持主体 + 渠道/国际叠加层 + 理由
6. 主体组合合法性核查（合法/非法/待确认 + 合法组合边界）
7. 法规/IP/保护期触发初筛（None/Yellow/Red + 需由D-1关闭的问题）
8. 评估阶段判断 + 依据
9. 推荐主技能（单选）
10. 推荐串接链路（按执行顺序列出）
11. 双轨场景识别结论（是/否 + 描述）
12. 当前数据缺口（Top 3~5）
13. 下一步行动（具体、可执行）

**语言规范：**
- 全部简体中文
- 财务缩写须标注中文全称，如 IRR（内部收益率）、NPV（净现值）、TS（条款清单）
- 禁止使用灰色字体
- 禁止生成元描述（如"本报告旨在……"等套话开头）
- 置信度须给出判断依据，不得仅写"高/中/低"
