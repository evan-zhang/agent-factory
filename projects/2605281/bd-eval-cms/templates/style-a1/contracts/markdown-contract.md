# Markdown 输出契约 v0.9.1

- 版本：0.9.1
- 日期：2026-06-13
- 适用范围：所有使用 Style A1 渲染模板的 CMS BD 报告生成端
- 目标：固定"报告生成端"和"HTML 渲染端"的接口协议

## 1. 契约目标

本契约不是写作指南，而是上游报告生成端（如 bd-eval-cms Skill）与 Style A1 renderer 之间的接口协议。

目标是让上游输出稳定结构，避免 renderer 依靠模糊文本猜测关键组件是否存在。

## 2. 通用必选结构

所有 Style A1 profile 的 Markdown 至少应包含以下结构：

### 2.1 报告标识

- **报告标题**：包含技能代码或通过 state.json 提供 skill_code
- **密级标识**：包含保密级别信息（内部/机密等）
- **呈报对象**：明确报告呈报对象

### 2.2 核心组件（One-pager 或等价终局先立章节）

- **One-pager 结构**：所有报告必须包含"终局先立"章节
  - 推荐格式：一级标题"执行摘要"或"One-pager"
  - 必须包含：核心结论、投资建议、关键风险
  - 位置：报告前部（前3个章节内）

### 2.3 结论框（必选）

所有报告至少包含以下结论框之一：

- **`::: conclusion-box`**：主要结论框，包含核心投资结论
- **`::: risk-box`**：风险评估框，包含主要风险因素
- **`::: veto-box`**：一票否决框，包含关键否决因素
- **`::: conflict-box`**：冲突框，包含利益冲突说明

**最低要求**：至少1个 `conclusion-box` 和1个 `risk-box`

### 2.4 置信度与阶段标识（必选）

所有报告必须包含：

- **置信度标识**：`[置信度:A]` / `[置信度:B]` / `[置信度:C]` / `[置信度:D]`
- **阶段标签**：`[阶段A]` / `[阶段B]` / `[阶段C]`

**格式要求**：使用标准格式，不得自创变体

### 2.5 参考文献章节（必选）

所有报告必须包含：

- **参考文献章节**或 **`::: references-shell`**
- 如信息不足，应明确标注"参考文献待补充"而非省略整个章节

### 2.6 术语表（必选）

所有报告必须包含：

- **术语表或 glossary table**
- 包含报告中的专业术语和缩写

## 3. A-1 特定要求

A-1 profile（海外已上市/中国未上市投前评估）额外要求：

### 3.1 业务主体互斥

- **`::: exclusion-box`**：业务主体互斥规则约束框
- 必须包含：主体互斥规则说明、适用场景、例外情况

### 3.2 Gate 结构

- **Gate 1/2 至少存在**：必须包含 Gate 1 和 Gate 2 结论
- 推荐格式：使用 `## Gate 1`、`## Gate 2` 等二级标题
- 每个 Gate 应包含：通过状态、关键结论、风险因素

### 3.3 Battle 结构

- **Battle 审查层与执行层结构**：
  - 审查层：战略视角、风险控制视角
  - 执行层：运营视角、落地可行性视角
  - 争议点：审查层与执行层的分歧点

## 4. 信息不足时的表达规范

**禁止上游因为信息不足而省略结构**。

### 6.1 正确做法

保留结构，并明确写明：

- **信息不足**：明确标注哪些信息缺失
- **当前置信度**：给出基于现有信息的置信度评估
- **需要补充的数据**：列出需要补充的具体数据项
- **对结论的影响**：说明信息不足对结论的影响程度

### 6.2 示例

❌ **错误做法**：
```markdown
## 结论
（因信息不足而省略）
```

✅ **正确做法**：
```markdown
## 结论
**当前信息状态**：信息不足，待补充关键财务数据

**当前置信度**：[置信度:C]

**需要补充的数据**：
- 目标公司近3年财务报表
- 市场份额数据
- 竞争对手分析

**对结论的影响**：
- 无法确定投资回报率区间
- 市场风险评估存在较大不确定性

**基于现有信息的初步结论**：
（在数据不完整情况下的谨慎结论）
```

## 7. 模板变量规范

上游应避免输出未替换的模板变量。

### 7.1 禁止的残留变量

以下模板变量不得出现在最终 Markdown 中：

- `{{ }}` 格式的未替换变量
- `{% %}` 格式的模板控制语句
- `{{ variable_name }}` 样式的占位符

### 7.2 默认值处理

对于可能缺失的字段，应提供合理的默认值，而非模板变量：

- 日期：使用"待定"或"YYYY-MM-DD"
- 数值：使用"TBD"或合理估算值
- 文本：使用"待补充"而非模板变量

## 8. 契约执行与校验

### 8.1 Renderer 校验职责

Style A1 renderer 负责校验：

1. **必选结构存在性**：检查所有必选组件是否存在
2. **模板变量残留**：检查是否有未替换的模板变量
3. **置信度与阶段标识**：检查格式是否正确
4. **profile 特定要求**：检查 profile 特定组件是否存在

### 8.2 失败策略

- **必选结构缺失**：fail-fast，拒绝生成 HTML
- **模板变量残留**：fail-fast，拒绝生成 HTML
- **可选结构缺失**：warning，允许生成 HTML 但记录警告
- **信息不足表达不当**：warning，要求明确标注信息不足状态

### 8.3 校验报告

Renderer 应输出结构化校验报告：

```json
{
  "contract_version": "0.9.1",
  "profile": "A-1",
  "status": "pass",
  "required_components": {
    "present": ["conclusion-box", "risk-box", "confidence-badge", "stage-tag"],
    "missing": []
  },
  "template_tokens": {
    "unreplaced": []
  },
  "warnings": [],
  "errors": []
}
```

## 10. 附录：必选组件清单

### 10.1 通用必选组件

所有 profile 必选：

1. 报告标题（含 skill_code）
2. One-pager 或等价终局先立章节
3. 至少1个 conclusion-box
4. 至少1个 risk-box
5. 置信度标识（[置信度:A/B/C/D]）
6. 阶段标签（[阶段A/B/C]）
7. 术语表或 glossary table（必须置于附录或正文末尾，不得占用前三章核心业务章节）
8. 参考文献章节或 references-shell

### 10.2 A-1 特定必选组件

1. exclusion-box
2. Gate 1 内容
3. Gate 2 内容
4. Battle 审查层结构
5. Battle 执行层结构
