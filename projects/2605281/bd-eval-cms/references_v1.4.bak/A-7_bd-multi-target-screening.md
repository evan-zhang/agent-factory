# A-7 多标的横向筛选（Multi-Target Comparative Screening）

---

## 模块一：技能元信息

| 字段 | 内容 |
|------|------|
| **技能编号** | A-7 |
| **技能名称** | 多标的横向筛选（Multi-Target Comparative Screening） |
| **版本** | v1.0 |
| **描述** | 对已知标的池（来自A-0输出、会议标的、投前团队主动收集）进行批量初筛→七维评分→优先级排序→分拨的横向比选流程。解决「多个候选标的资源有限，应优先推进哪几个」的决策问题。与A-0的核心区别：A-0「从无到有发现标的」；A-7「在已知标的池内排序决策」。 |
| **触发条件** | 用户有≥2个候选标的需要横向比较；BIO展会/CPHI等会议后批量标的筛选；季度投前管道（Pipeline）复盘；用户说「帮我比较这几个产品」「哪个标的更值得推进」「多标的排序」「投前管道优化」「批量筛选」时触发 |
| **适用业务主体** | 全集团（深康/德镁/维盛/院外业务/天津康哲/康联达Rxilient），根据提交标的的业务主体自动匹配财务门槛 |
| **继承来源** | pharma-bd-report-generator 模板7；A-0输出的A/B/C分级结果可直接导入 |
| **商机接入路径** | A-0 A级多标的输出 → A-7；会议后标的池 → A-7；投前团队季度复盘 → A-7 |

> **阶段深度**：A-7为横向筛选，信息颗粒度要求统一、可比，采用「最小公分母原则」——每个标的须填写相同字段。

| 编号 | 字段 | 必填/选填 | 说明 |
|------|------|-----------|------|
| M-01 | 标的编号与名称 | 必填 | 每个标的须有唯一编号（便于矩阵对比） |
| M-02 | 治疗领域（TA） | 必填 | 心肾代谢/消化肝病/CNS/皮肤健康/眼科/仿制药/医美 |
| M-03 | 产品阶段 | 必填 | 临床前/I期/II期/III期/已上市 |
| M-04 | 来源（接入方式） | 必填 | A-0分拨/会议/原研方主动接触/内部推荐 |
| M-05 | 发起业务主体 | 必填 | 深康/德镁/维盛/院外/天津康哲/康联达 |
| M-06 | 初步交易结构 | 选填 | License-in/代理权/共同开发/收购 |
| M-07 | 峰值销售额（PeakSales）粗估 | 必填 | 乐观/基准/保守（可为粗估区间） |
| M-08 | 已知财务条款 | 选填 | 首付款（前期预付款）意向、提成范围 |
| M-09 | A-0分级（如有） | 选填 | A/B/C/Watch（来自A-0输出） |
| M-10 | 评估截止日期 | 必填 | 明确排序结论需在何时输出 |

## 模块三：评估阶段映射（三阶段模型）

```
Stage 1（一票否决快扫）：对应「前提筛」
  └─ 目的：剔除不可能推进的标的，节约后续评分资源
  └─ 输入：M-01~M-07基本信息 + 六项否决条件
  └─ 输出：通过标的清单（进入Stage 2）+ 关闭标的清单（注明原因）

Stage 2（七维评分矩阵）：对应「价值筛」
  └─ 目的：对通过Stage 1的标的进行结构化评分
  └─ 输入：公开可获取信息 + 内部评估意见
  └─ 输出：每个标的的七维评分表 + 加权总分排名

Stage 3（优先级排序与分拨）：对应「决策输出」
  └─ 目的：将标的分配至A/B/C/Watch/关闭五个优先级等级，并分拨至对应评估技能
  └─ 输入：Stage 2评分结果 + 集团资源约束
  └─ 输出：投前优先级矩阵 + 各标的推荐下游技能 + 资源分配建议
```

## 模块四：评估章节结构（3-Stage）

---

### ▶ Stage 1：一票否决快扫（前提筛）

**一票否决七条（任一命中即关闭）**：

| 编号 | 否决项 | 判断标准（快扫，≤5分钟/标的） |
|------|--------|-------------------------------|
| K-1 | TA完全不匹配 | 与集团六大TA及医美无任何交集 |
| K-2 | 中国注册路径完全封闭 | NMPA明确禁止该品类或历史无先例且无路径可探 |
| K-3 | 独家竞争封锁 | 同适应症中国市场已被强竞对手独家锁定 |
| K-4 | 峰值销售额（PeakSales）过低 | 乐观情景下峰值销售额（PeakSales）<1亿元人民币 |
| K-5 | IP障碍不可绕过 | 核心专利保护期≥10年且无授权意向 |
| K-6 | 严重安全信号 | 不可克服的致死性安全事件或FDA/EMA黑框警告 |
| K-7 | 资源冲突极端情况 | 与集团现有管线存在直接内部竞食且无法区隔 |

**Stage 1输出表格**：
```
标的编号 | 标的名称 | K1~K7状态 | 通过/关闭 | 关闭原因
```

### ▶ Stage 2：七维评分矩阵（价值筛）

**评分说明**：每维度1~5分（1=极差/5=极优），各维度有明确评分锚点，避免主观偏差。

#### 维度1：管线协同度（权重20%）

| 分值 | 描述 |
|------|------|
| 5 | 与现有管线高度协同（同TA同科室，商业化团队100%复用，患者群完全重叠） |
| 4 | 较强协同（同TA不同机制，商业化团队>70%复用） |
| 3 | 中等协同（相邻TA，商业化团队50%~70%复用） |
| 2 | 弱协同（需新建部分团队能力） |
| 1 | 无协同（全新TA，需从零建设商业化能力） |

#### 维度2：临床价值（权重20%）

| 分值 | 描述 |
|------|------|
| 5 | III期阳性数据+突破性疗效改善+优异安全窗口 |
| 4 | III期完整数据+中等疗效改善+安全可控 |
| 3 | II期阳性数据+差异化潜力，安全数据有限 |
| 2 | I期/早期数据，临床价值待验证 |
| 1 | 临床前/概念验证阶段，不确定性极高 |

#### 维度3：注册确定性（权重15%）

| 分值 | 描述 |
|------|------|
| 5 | 5.1类/6类直接申报，海外数据充分可桥接，无需新临床 |
| 4 | 需小型桥接研究（<50人，<1年），路径清晰 |
| 3 | 需中型桥接研究（50~200人，1~2年） |
| 2 | 需大型补充临床（>200人，2~4年），高不确定性 |
| 1 | 需全新中国III期临床（>4年），或注册路径不明 |

#### 维度4：市场规模（权重15%）

| 分值 | 描述 |
|------|------|
| 5 | 基准峰值销售额（PeakSales）>20亿元人民币 |
| 4 | 基准峰值销售额（PeakSales）10~20亿元人民币 |
| 3 | 基准峰值销售额（PeakSales）5~10亿元人民币 |
| 2 | 基准峰值销售额（PeakSales）2~5亿元人民币 |
| 1 | 基准峰值销售额（PeakSales）<2亿元人民币 |

#### 维度5：竞争格局（权重10%）

| 分值 | 描述 |
|------|------|
| 5 | 中国市场无同类竞品，蓝海进入 |
| 4 | ≤1个同类竞品，差异化空间大 |
| 3 | 2~3个同类竞品，有差异化卡位点 |
| 2 | 4~6个同类竞品，市场竞争激烈 |
| 1 | >6个同类竞品，赛道红海，难以建立市场地位 |

#### 维度6：交易可行性（权重10%）

| 分值 | 描述 |
|------|------|
| 5 | 原研方明确有意向授权，首付款（前期预付款）意向在集团接受范围内，已进入实质谈判 |
| 4 | 原研方有初步意向，条款预期基本合理 |
| 3 | 原研方态度中立，需进一步接触建立关系 |
| 2 | 原研方意向不明确，或已与多方并行谈判 |
| 1 | 原研方已有其他优先谈判方，或条款预期严重超出集团范围 |

#### 维度7：财务回报（权重10%）

| 分值 | 描述 |
|------|------|
| 5 | 预估IRR（内部收益率）≥25%，回收期≤3年 |
| 4 | 预估IRR（内部收益率）20%~25%，回收期≤4年 |
| 3 | 预估IRR（内部收益率）15%~20%，回收期≤5年 |
| 2 | 预估IRR（内部收益率）10%~15%，回收期5~7年 |
| 1 | 预估IRR（内部收益率）<10%，或回收期>7年 |

### ▶ Stage 3：优先级排序与分拨（决策输出）

**优先级分级规则**：

| 等级 | 加权总分区间 | 定义 | 行动指令 | 资源分配 |
|------|-------------|------|----------|---------|
| **A级** | ≥3.5分 | 强优先推进，建议0~30天内正式立项 | 分拨至对应单标的评估技能（A-1/A-2/A-3/A-4/A-6/A-8），启动阶段A评估 | 完整投前团队资源投入 |
| **B级** | 3.0~3.4分 | 次优先，纳入季度跟踪，设置明确触发条件 | 指定负责人持续跟踪，触发条件达成后升级至A级 | 单一投前成员跟踪维护 |
| **C级** | 2.5~2.9分 | 潜力标的，纳入年度回顾，关注赛道变化 | 年度复盘时重新评估，赛道重大变化时可提前激活 | 最低资源（数据库监测） |
| **Watch** | 2.0~2.4分 | 早期/平台型，持续低强度监测 | 纳入行业监测清单，无近期推进计划 | 被动监测 |
| **关闭** | <2.0分或Stage 1否决 | 当前不可行，正式关闭 | 存档，记录关闭原因及可重开条件 | 无资源投入 |

**资源约束分配矩阵**（当A级标的超过团队承接能力时）：

```
团队投前承接容量：[N个并行项目]
当前A级标的数量：[M个]
如果M > N，按以下原则再降优先级：
  1. 管线协同度最高的优先（团队资源复用最大化）
  2. 注册确定性最高的优先（最快产生价值）
  3. 交易可行性最高的优先（最快推进谈判）
```

## 模块六：一票否决清单

> A-7阶段一票否决直接对应Stage 1的K-1~K-7。

1. **K-1**：TA完全不匹配集团六大业务
2. **K-2**：中国注册路径完全封闭（监管层面）
3. **K-3**：独家竞争封锁（市场层面）
4. **K-4**：乐观峰值销售额（PeakSales）<1亿元人民币
5. **K-5**：不可绕过的IP障碍（≥10年专利，无授权意向）
6. **K-6**：严重不可克服安全信号
7. **K-7**：与集团现有重点管线存在极端直接竞食

## 模块九：输出规范

- **主报告格式**：HTML（深蓝 `#1a3a5c`，白底，无灰色字）+ 可导出评分矩阵Excel
- **核心可视化**：气泡图（X轴=市场规模，Y轴=加权总分，气泡大小=交易可行性）+ 热力矩阵（标的×维度得分）
- **财务缩写**：IRR（内部收益率）、NPV（净现值）、CAGR（复合年均增长率）、PeakSales（峰值销售额）等均须括号标注中文全称
- **双币标注**：非人民币金额同步标注人民币等值（注明汇率换算日期）
- **报告章节顺序**：
```
封面
执行摘要（筛选总结 + A级推荐清单）
第1章 参与筛选标的概览
第2章 Stage 1一票否决快扫结果
第3章 Stage 2七维评分矩阵详述
第4章 优先级排序与分拨建议（Stage 3）
第5章 资源分配建议
附录A 各标的完整评分卡
附录B 关闭标的归档记录
参考文献
```
