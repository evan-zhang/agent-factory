# D-0 投前评估路由调度器 (投前 Evaluation Router)

---

## 模块1：技能元信息

| 字段 | 内容 |
|------|------|
| **技能编号** | D-0 |
| **技能名称** | 投前评估路由调度器 |
| **版本** | v1.0 |
| **最后更新** | 2025-06 |
| **描述** | 集团投前评估体系统一入口。接收任意标的/商机信息，自动识别产品类型、业务主体归属、评估阶段，输出推荐技能编号、串接方案及后续评估行动计划。本技能不直接产出评估报告，只输出路由决策与任务编排。 |
| **触发条件** | 任何投前评估请求的第一入口。用户提及"投前项目"、"产品引进"、"商机扫描"、"立项评估"、"标的评估"、"合作评估"、"引进分析"、"投前调度"，或直接描述一个标的但未指定具体评估框架时，均应触发 D-0。 |
| **适用业务主体** | 全集团六大业务主体（深康 / 德镁 / 维盛 / 院外业务 / 天津康哲 / 康联达 Rxilient） |
| **继承来源** | 无（D-0 为顶层路由器，不继承具体评估框架） |
| **商机接入路径** | 主动扫描 / 合作方来访 / 展会/学术会议 / 内部推荐 / 自研转化 |
| **输出物** | 路由决策单（纯文本/表格，非 HTML 报告） |

## 模块2：通用前置章节（公司基本信息 M-01 ~ M-10）

> D-0 作为路由器，前置章节以「快扫」深度采集，仅需核心识别字段，不做深度分析。
> 详细尽调信息由下游具体技能承接。

| 编号 | 字段名称 | 路由阶段采集要求 | 示例/说明 |
|------|----------|-----------------|-----------|
| M-01 | 标的名称 | 必填 | 通用名 / 商品名 / 项目代号 |
| M-02 | 合作方名称 | 必填 | 许可方 / 代理方 / 合并目标 |
| M-03 | 产品类型 | 必填 | 详见路由决策树，共7大类 |
| M-04 | 主治疾病/适应症 | 必填 | ICD-11 编码或文字描述 |
| M-05 | 目标市场 | 必填 | 中国大陆 / 香港 / 东南亚 / 其他国际 |
| M-06 | 评估阶段 | 必填 | A（公开信息）/ B（立项后）/ C（投委会） |
| M-07 | 合作方式 | 选填 | Licensing-in / 代理 / 合资 / 股权 / 自研转化 |
| M-08 | 开发阶段 | 必填 | 临床前 / I期 / II期 / III期 / 已上市 |
| M-09 | 初步财务规模 | 选填 | 预期首付款/里程碑/销售峰值（如有） |
| M-10 | 内部发起部门 | 必填 | 发起 投前 请求的业务单元或职能 |

## 模块3：评估阶段映射

### 三阶段模型概述

```
阶段A：商机发现 → Gate 0 初筛 → 具体产品初审评估报告 → 立项会决策（公开信息域）
          ↓
     NDA 弹性签署节点（不固定阶段，由合作方保密要求驱动）
          ↓
阶段B：立项后双轨并行
        技术深化评估线 ‖ 商务谈判线（财务在 TS 阶段进场）
          ↓
阶段C：Gate 3 投委会终审 → 批准 / 条件批准 / 否决
```

### D-0 介入阶段

| 阶段 | D-0 角色 | 输出深度 |
|------|----------|---------|
| **阶段A（Gate 0 之前）** | 主要介入阶段。接收商机信息，完成路由决策，输出推荐技能+串接方案 | 路由决策单（1~2页） |
| **阶段B** | 如评估过程中产品类型或业务主体发生变化，可重新触发路由 | 更新版路由决策单 |
| **阶段C** | 不介入（已由 D-3 承接终审包） | 不适用 |

## 模块4：路由决策树

### 第一层：产品类型识别

```
标的输入
  ├─ 创新药（first-in-class / best-in-class / 改良型）
  │     ├─ 中国大陆市场 → 匹配 D-1（基座）+ D-2（市场）+ D-3（一页纸）
  │     │                 → 主技能：pharma-bd-report-generator（5.1类/6类路径）
  │     └─ 国际市场（HK/SEA/其他） → 匹配 D-1 + D-2 + 康联达专属模板
  │
  ├─ 仿制药 / 生物类似药
  │     ├─ 化学仿制药 → 天津康哲路径 → pharma-bd-report-generator（仿制药框架）
  │     └─ 生物类似药 → 深康/维盛路径 → pharma-bd-report-generator（生物类似药框架）
  │
  ├─ 医美产品
  │     ├─ 单标的评估 → medical-aesthetics-product-evaluator
  │     └─ 组合复盘 → medical-aesthetics-portfolio-audit
  │
  ├─ 消费健康 / 院外品种
  │     └─ 院外业务路径 → pharma-bd-report-generator（院外/消费健康框架）
  │
  ├─ 平台型战略合作（4.0模式）
  │     └─ → pharma-bd-report-generator（战略级平台框架）+ 股权评估附加模块
  │
  ├─ 多标的横向比较 / BIO展会批量筛选
  │     └─ → D-1（基座）+ D-3（批量一页纸）+ pharma-bd-report-generator（多标的比较框架）
  │
  ├─ 路由细分判断
  │     ├─ 中国市场 + 国内Biotech/药企 + 成熟阶段品种（NDA已提交/即将提交/III期完成）+ 商业化授权 → A-2 (± E-1)
  │     └─ 中国市场 + 药品 + 已在中国上市（有NMPA批文+销售历史）+ 推广权/代理权转让 → A-5
  │
  └─ 市场全景 / 纯市场研究需求
        └─ → D-2（市场全景报告）直接路由
```

### A-2 vs A-5 边界判断
- 产品是否已在中国上市（有NMPA批文+销售历史）？
  - 是 → A-5（已上市产品推广权引进）
  - 否（在研/NDA阶段/未上市）→ A-2（国内合作·成熟阶段产品）
- 合作方是国内企业还是海外企业？
  - 国内 → A-2 或 A-5（按上市状态区分）
  - 海外 → A-1（海外产品License-in）或 A-5（进口产品已在中国上市）


### 第二层：业务主体匹配

| 适应症/产品领域 | 优先业务主体 | 备选业务主体 |
|----------------|-------------|-------------|
| 心肾代谢 / 消化肝病 / CNS | 深康 | 维盛（眼底代谢合并症） |
| 皮肤健康药品 | 德镁 | 深康（系统性皮肤病） |
| 医美（注射/器械/耗材） | 德镁 | 院外业务 |
| 眼科（处方药/器械） | 维盛 | 深康（眼底新生血管） |
| 院外/OTC/消费健康 | 院外业务 | 天津康哲 |
| 化学仿制药（首仿/一致性评价） | 天津康哲 | 深康/德镁 |
| 国际市场（HK/SEA/欧美） | 康联达 Rxilient | 各业务主体国际部 |

### 第三层：评估阶段判断

| 用户提供信息量 | 判断为阶段 | D-0 行动 |
|---------------|-----------|---------|
| 仅产品名+适应症 | 阶段A（Gate 0前） | 输出 Gate 0 快扫模板 + 推荐技能 |
| 有公开数据+初步财务意向 | 阶段A（Gate 0后立项前） | 输出完整路由决策单 + D-1 启动指引 |
| 有 NDA 保护数据 | 阶段B | 输出 B 阶段双轨任务分配表 |
| 有 TS 草案 + 财务模型 | 阶段C | 输出 D-3 投委会包启动指令 |

## 模块5：财务内控对照表

> **D-0 路由器不执行财务评估，无财务内控门槛。**
> 财务内控标准由下游技能（D-1 基座财务维度、pharma-bd-report-generator）承接。
> D-0 仅记录用户提供的初步财务规模（M-09）用于辅助路由判断（大型平台项目 vs 单品种引进）。

---

## 模块6：一票否决清单

> **D-0 路由器不执行否决判断，无一票否决条件。**
> 一票否决清单由 D-1 基座（红旗风险台账）及各主技能执行。
> D-0 的职责是识别"路由到哪个技能"，而非"是否应该否决"。

## 模块7：Gate 阶段说明

| Gate 编号 | 阶段名称 | D-0 在此 Gate 的角色 | 输出深度要求 |
|-----------|---------|---------------------|-------------|
| **Gate 0** | 商机申报与快扫（准入门） | **核心介入点**。D-0 在 Gate 0 前完成路由，确保商机进入正确技能通道 | 路由决策单 + Gate 0 快扫模板指引 |
| Gate 1 | 前提门（关键假设验证） | 被动触发：若前提门评估中发现产品类型误判，重新路由 | 更新版路由决策单 |
| Gate 2 | 商业化可行性门 | 不主动介入 | — |
| Gate 3 | 投委会终审门 | 不介入（D-3 承接） | — |

### 双轨串接场景自动识别

| 串接场景 | 识别信号 | 推荐组合 |
|---------|---------|---------|
| 院内 + 院外双渠道 | 同一产品同时涉及医院渠道和院外/OTC渠道 | 深康/德镁/维盛（院内）‖ 院外业务（院外） |
| 国内 + 国际双市场 | 合作协议包含大陆+港澳/东南亚 | 各业务主体（国内）‖ 康联达 Rxilient（国际） |
| 商业化 + 股权双结构 | TS 草案中含股权条款 | pharma-bd-report-generator + 股权评估附加模块 |
| 多适应症跨业务主体 | 单一产品覆盖2个以上业务主体适应症领域 | 主责业务主体牵头 + 参与业务主体协同评估 |

## 模块9：输出规范

### 路由决策单格式规范

D-0 输出为**结构化文本/Markdown 表格**，不生成 HTML 报告。

**必含字段：**
1. 路由编号（自动生成：D0-YYYY-NNN）
2. 标的基本识别信息（M-01 ~ M-06 摘要）
3. 产品类型判断结论 + 置信度（高/中/低）
4. 业务主体匹配结论 + 理由
5. 评估阶段判断 + 依据
6. 推荐主技能（单选）
7. 推荐串接链路（按执行顺序列出）
8. 双轨场景识别结论（是/否 + 描述）
9. 当前数据缺口（Top 3~5）
10. 下一步行动（具体、可执行）

**语言规范：**
- 全部简体中文
- 财务缩写须标注中文全称，如 IRR（内部收益率）、NPV（净现值）、TS（条款清单）
- 禁止使用灰色字体
- 禁止生成元描述（如"本报告旨在……"等套话开头）
- 置信度须给出判断依据，不得仅写"高/中/低"

**示例输出片段：**

```
路由编号：D0-2025-042
标的：XX注射液（通用名）
产品类型：创新药（5.1类，境外已上市）→ 置信度：高（合作方已提供 NDA 资料目录）
业务主体：深康（心肾代谢适应症，慢性肾病）
评估阶段：阶段B（立项后，NDA已签）
推荐主技能：pharma-bd-report-generator（5.1类/6类路径）
串接链路：D-1（基座）→ D-2（慢性肾病市场全景）→ D-3（更新立项后一页纸）
双轨识别：否
数据缺口：① III期中国桥接数据 ② 竞品医保谈判价格 ③ CMC本地化生产方案
下一步：启动 D-1 第二阶段深化（阶段B），同步触发 D-2 肾病市场全景分析
```

## 附录B：常见错误路由纠正指南

| 错误场景 | 错误路由 | 正确路由 | 纠正逻辑 |
|---------|---------|---------|---------|
| 用户说"评估一个皮肤病药物"，直接路由到医美 | medical-aesthetics-product-evaluator | pharma-bd-report-generator（德镁药品路径） | 皮肤病药物≠医美，须区分处方药属性 |
| 用户说"评估眼科产品"，路由到深康 | 深康 | 维盛 | 眼科为维盛主责业务主体 |
| 多适应症产品，仅路由到一个业务主体 | 单一路由 | 双轨路由（主责+协同） | 需明确主责业务主体并注明协同单元 |
| 已有 TS 草案但路由为阶段A | 阶段A | 阶段C | TS 草案存在 = 已进入商务谈判线，判定阶段B/C |
