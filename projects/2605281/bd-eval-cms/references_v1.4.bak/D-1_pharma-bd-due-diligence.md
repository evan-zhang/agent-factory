## 模块3：评估阶段映射

### D-1 在三阶段模型中的介入深度

| 阶段 | D-1 角色 | 评估深度 | 主要输出物 |
|------|----------|---------|---------|
| **阶段A（公开信息域）** | 基础框架提供者。基于公开信息完成14维度初步扫描，识别明显红旗，输出 Gate 0 快扫结论 | 快扫级（约10-15页） | Gate 0 评估摘要 + DRL V0（初始资料清单） |

## 模块4：评估章节结构（Gate 编排）

### 总体编排原则

```
One-pager 终局先立（锚定门）
    ↓
Gate 0：商机申报与快扫（准入门）—— 能否立项？
    ↓
Gate 1：关键假设前提验证（前提门）—— 核心假设是否成立？
    ↓
Gate 2：科学与技术可行性（技术门）—— 技术风险是否可控？
    ↓
Gate 3：商业化可行性（商业门）—— 市场与推广逻辑是否成立？
    ↓
Gate 4：财务与合规可行性（价值门）—— 财务回报是否达标？
    ↓
Gate 5：合同条款与交割可行性（可做门）—— 交易是否可以执行？
```

### Gate 0：商机申报与快扫（准入门）

**目标：** 判断是否值得投入资源进行深度评估。

| 评估项 | 权重 | 评估方法 |
|--------|------|---------|
| 战略契合度（与集团/业务主体战略方向匹配度） | 高 | 对照年度 投前 优先清单 |
| 产品差异化潜力（vs 现有竞品） | 高 | 公开竞品数据比较 |
| 合作方可信度初判 | 中 | 公开信息/监管数据库核查 |
| 财务规模初步可行性（峰值销售 vs 预期首付款比例） | 中 | 行业经验类比 |
| 明显一票否决项检查 | 高 | 对照否决清单 |

**Gate 0 结论格式：**
- 结论：**立项 / 暂缓 / 否决**
- 置信度：高（>80%）/ 中（50-80%）/ 低（<50%）
- 需补证据 Top 5：（具体列出5条）

### Gate 1：关键假设前提验证（前提门）

**目标：** 验证支撑商业逻辑的3~5个核心假设是否成立。

| 典型关键假设类型 | 验证方法 | 数据来源 |
|----------------|---------|---------|
| 目标患者人群规模假设 | 流行病学文献 + KOL 访谈 | D-2 市场全景 |
| 监管路径假设（注册时间/类别） | NMPA 公开数据 + 注册顾问意见 | M-10 + 注册维度 |
| 竞品格局假设（进入时机窗口） | 竞品研究 + 医保谈判时间线 | D-2 + 公开数据库 |
| 定价假设（参照系及可及性） | 同类产品医保价 + 国际价格比较 | 公开药品价格数据库 |
| 合作方履约能力假设 | 已有合作伙伴访谈 + 公开案例 | M-03 + M-07 |

### Gate 2：科学与技术可行性（技术门）

**评估范围：** 维度1（科学与风险）+ 维度2（科学与商业）+ 维度7（注册策略）+ 维度9（生产策略）

| 子评估项 | 核心问题 |
|---------|---------|
| 作用机制合理性 | MoA 是否有充分临床前/临床证据支持？ |
| 临床数据质量 | 设计、统计、终点是否符合当前监管标准？ |
| 安全性档案 | SAE 发生率、重要提示/禁忌是否可接受？ |
| 注册可行性 | NMPA 审评路径是否明确？时间线是否可预测？ |
| CMC 状态 | 生产工艺是否成熟？供应链是否稳定？ |

### Gate 3：商业化可行性（商业门）

**评估范围：** 维度3（市场调查）+ 维度4（市场预测）+ 维度5（推广规划）+ 维度11（准入策略）

| 子评估项 | 核心问题 |
|---------|---------|
| 市场规模验证 | 目标患者人群是否经过第一手调研验证？ |
| 市场份额可得性 | 竞品格局下，可实现份额假设是否合理？ |
| 渠道与推广策略 | 与集团现有商业化能力是否匹配？ |
| 医保准入路径 | 医保纳入时间线、谈判价格区间是否合理？ |
| 上市后医学策略 | RWE/HEOR 规划是否支持市场准入论据？ |

### Gate 4：财务与合规可行性（价值门）

**评估范围：** 维度6（财务）+ 维度12（法律）+ 维度13（专利）+ 维度14（商标）

| 子评估项 | 核心问题 |
|---------|---------|
| rNPV（风险调整净现值） | rNPV 是否覆盖首付款及里程碑总承诺？ |
| IRR（内部收益率） | IRR 是否超过集团门槛（见模块5）？ |
| 回收期 | 投资回收期是否在集团可接受范围内？ |
| 合同结构合规性 | 排他条款、回购条款、竞业限制是否有法律风险？ |
| 专利悬崖风险 | 核心专利到期时间与销售峰值的时序关系？ |

### Gate 5：合同条款与交割可行性（可做门）

**评估范围：** 维度8（医学研究策略）+ 维度10（供应链策略）+ TS 条款审查

| 子评估项 | 核心问题 |
|---------|---------|
| TS 关键条款可谈判性 | 首付款、里程碑、分成比例是否在合理范围？ |
| 独占权范围 | 地域独占、适应症独占是否覆盖商业化需求？ |
| 供应链交割 | API（活性药物成分）/制剂供应协议是否稳定？ |
| 临床/注册里程碑合理性 | 里程碑支付时间点是否与监管进度挂钩？ |
| 退出机制 | 如项目失败，终止条款是否对集团公平？ |

## 模块5：财务内控对照表

> 以下门槛值为集团参考基准，各业务主体可根据细分市场特征在±20%范围内调整。
> 所有财务缩写均标注中文全称。

### 按产品类型分类财务内控门槛

| 产品类型 | IRR（内部收益率）门槛 | NPV（净现值）最低要求 | 回收期上限 | 回报倍数（MOIC）参考 | 毛利率（Gross Margin）参考 | 净利润率（Net Margin）参考 |
|---------|--------------------|--------------------|----------|--------------------|--------------------------|-----------------------------|
| 创新药（进口5.1类） | ≥18% | ≥首付款3倍 | ≤8年 | ≥3x | ≥65% | ≥15%（成熟期） |
| 创新药（6类境外新药） | ≥20% | ≥首付款4倍 | ≤7年 | ≥3.5x | ≥70% | ≥18%（成熟期） |
| 化学仿制药（首仿） | ≥25% | ≥首付款2倍 | ≤5年 | ≥2x | ≥55% | ≥12% |
| 生物类似药 | ≥22% | ≥首付款2.5倍 | ≤6年 | ≥2.5x | ≥60% | ≥14% |
| 医美产品（注射类） | ≥30% | ≥首付款3倍 | ≤4年 | ≥3x | ≥70% | ≥20% |
| 消费健康/院外品种 | ≥28% | ≥首付款2倍 | ≤4年 | ≥2.5x | ≥50% | ≥15% |
| 平台型战略合作（4.0模式） | ≥15%（组合口径） | 正 NPV | ≤10年 | ≥2x（组合） | 依产品线 | 依产品线 |

### 财务模型关键假设标准化

| 假设参数 | 标准化方法 | 敏感性测试范围 |
|---------|-----------|--------------|
| 折现率（WACC，加权平均资本成本） | 集团统一 WACC 基准 ±2% 调整 | ±3% |
| 峰值销售预测 | 基于 TAM（目标可及市场）× 份额假设 | -30% / +20% |
| 专利保护期 | 以最窄保护专利到期年计算 | -2年悬崖情景 |
| 医保纳入时间 | 基准情景+1年、悲观情景+3年 | +1/+3年 |
| 里程碑成功率折现 | 按临床阶段标准成功率（PhII→PhIII: 40-50%） | 各阶段±10% |

## 模块6：一票否决清单

> 以下任何一条成立，项目立即终止评估，进入否决记录。
> 否决结论须在红旗风险台账中注明具体触发条款及证据来源。

### 合规与法律类（硬性否决）

| 编号 | 否决条件 | 核查方法 |
|------|---------|---------|
| VK-01 | 合作方在近5年内收到 FDA / NMPA / EMA 警告信（Warning Letter）且未完成整改 | 监管机构公开数据库 |
| VK-02 | 产品或合作方涉及未解决的刑事诉讼或重大民事欺诈指控 | 律所尽调报告 |
| VK-03 | 合作方实际控制人或核心管理层在制裁名单中（OFAC/BIS/其他） | OFAC、BIS 数据库核查 |
| VK-04 | 知识产权归属存在争议且合作方无法提供明确权属证明 | 专利局数据库 + 律所意见 |
| VK-05 | 合同中含有强制仲裁条款且仲裁地不在集团可接受司法管辖范围 | 法务审查 |

### 科学与安全类（硬性否决）

| 编号 | 否决条件 | 核查方法 |
|------|---------|---------|
| VK-06 | 关键临床试验存在数据造假指控（经监管机构认定或可信第三方报告） | 监管公告 + 文献核查 |
| VK-07 | 产品存在无法通过说明书管理的黑框警告（Black Box Warning）且涉及致死性 SAE | FDA/NMPA 说明书核查 |
| VK-08 | 关键适应症已被更安全/有效竞品占据>70%市场份额且无差异化路径 | 竞品市场分析（D-2） |
| VK-09 | 生物类似药参照药已在评估市场内失去专利保护且市场价格崩塌 | 市场价格数据 |

### 财务类（硬性否决）

| 编号 | 否决条件 | 核查方法 |
|------|---------|---------|
| VK-10 | 基准情景下 rNPV（风险调整净现值）为负值 | 财务模型 |
| VK-11 | 首付款超过集团单项目预算上限且无法通过联合融资覆盖 | 财务部门确认 |
| VK-12 | 合作方要求付款后提供数据，且拒绝任何形式的里程碑退款保障机制 | TS 条款审查 |

### 战略类（软性否决，需高管层确认豁免）

| 编号 | 否决条件 | 豁免条件 |
|------|---------|---------|
| VS-01 | 产品适应症与集团当前战略重点完全无关联 | CEO/COO 书面豁免 |
| VS-02 | 合同要求集团提供超过现有能力的临床/注册支持且无合理过渡期 | 投前委员会豁免 |

## 模块7：Gate 阶段说明

| Gate 编号 | 阶段名称 | 介入评估阶段 | D-1 工作内容 | 输出深度要求 |
|-----------|---------|------------|------------|------------|
| **Gate 0** | 商机申报与快扫（准入门） | 阶段A | 完成14维度快扫，输出快扫结论，生成 DRL V0 | 1~2页结论摘要 + DRL V0 |
| **Gate 1** | 关键假设前提验证（前提门） | 阶段A→B转折 | 验证Top 3~5关键假设，结论与置信度 | 假设验证矩阵（表格） |
| **Gate 2** | 科学与技术可行性（技术门） | 阶段B（技术线） | 科学/注册/CMC深化评估，输出技术风险评级 | 技术评估章节（10~15页） |
| **Gate 3** | 商业化可行性（商业门） | 阶段B（技术线+商务线） | 市场/推广/准入深化，输入来自 D-2 市场全景 | 商业评估章节（10~15页） |
| **Gate 4** | 财务与合规可行性（价值门） | 阶段B（财务进场后） | 财务模型审核、法律/专利/商标深化，出具红旗台账V1 | 财务章节+红旗台账V1（10页） |
| **Gate 5** | 合同条款与交割可行性（可做门） | 阶段B→C转折 | TS条款审查，供应链/医学策略可行性，红旗台账升级至V2 | TS审查摘要+红旗台账V2（5~10页） |

## 模块8：串接接口

### DRL（尽调资料需求清单）标准化7大模块

#### DRL-1：科学模块（Science）

| 资料类型 | 阶段A（DRL V0） | 阶段B（DRL V1） |
|---------|--------------|--------------|
| 临床前数据摘要 | 公开文献即可 | 完整研究报告 |
| 所有临床试验 CSR（临床研究报告） | 摘要/公开部分 | 完整 CSR |
| 安全性数据库（PSUR/IB） | 最新版 IB | 最新 PSUR + IB |
| KOL 意见（如有） | 选填 | 必填（≥3名） |

#### DRL-2：注册模块（Regulatory）

| 资料类型 | 阶段A | 阶段B |
|---------|-------|-------|
| 各市场注册状态 | 公开查询 | 完整注册档案目录 |
| 历史监管沟通记录 | 不适用 | 完整会议纪要 |
| 拟定说明书（各市场版本） | 已批准版本 | 拟定中国说明书草案 |

#### DRL-3：CMC模块（Chemistry, Manufacturing & Controls，化学生产控制）

| 资料类型 | 阶段A | 阶段B |
|---------|-------|-------|
| 生产工艺概要 | 公开信息 | 完整工艺描述 |
| 供应商清单 | 不适用 | API 及制剂生产商清单 |
| 质量体系认证（GMP，药品生产质量管理规范） | 证书有效期 | 最近一次 GMP 审计报告 |

#### DRL-4：市场模块（Market）

| 资料类型 | 阶段A | 阶段B |
|---------|-------|-------|
| 目标市场销售数据（如已上市） | 公开第三方数据 | 合作方内部销售报告 |
| 竞品分析报告 | 内部快扫 | D-2 市场全景报告 |
| 患者调研数据 | 不适用 | 合作方已有患者调研 |

#### DRL-5：财务模块（Finance）

| 资料类型 | 阶段A | 阶段B |

## 模块9：输出规范

### HTML 报告格式规范（D-1 深化报告阶段适用）

**视觉规范：**
- 主色调：深蓝 `#1a3a5c`（标题栏、表头、分隔线）
- 辅助色：浅蓝 `#e8f0f8`（交替行背景）、橙黄 `#f5a623`（警示/红旗标记）
- 字体：正文 14px，标题 18-22px，表头加粗
- 禁止使用灰色字体（`#999999` 及以下均不可用）
- 禁止生成元描述段落（如"本报告旨在……"等开头套话）

**内容规范：**
- 所有财务缩写须在首次出现时标注中文全称，例如：IRR（内部收益率）、NPV（净现值）、rNPV（风险调整净现值）、WACC（加权平均资本成本）、MOIC（投资回报倍数）、FCF（自由现金流）、EBITDA（息税折旧摊销前利润）、CAPEX（资本支出）、DSO（应收账款周转天数）、ROIC（投入资本回报率）
- 非人民币市场须双币标注（如：USD 100M / RMB 约7.2亿元，按报告发布日汇率）
- 每个 Gate 结论须包含：结论陈述 + 置信度（%）+ 置信度来源说明 + 需补证据 Top 5
- 红旗项须用橙黄色警示背景标记，一票否决项须用红色（`#c0392b`）标记

**章节顺序规范（适用 D-1 深化报告）：**
```
第1章 执行摘要（One-pager 级）
第2章 公司基本信息（M-01 ~ M-10）
第3章 Gate 0：快扫结论
第4章 Gate 1：关键假设验证
第5章 维度深化：科学与技术（Gate 2）
第6章 维度深化：商业化可行性（Gate 3）
第7章 维度深化：财务与合规（Gate 4）
第8章 维度深化：合同条款（Gate 5）
第9章 红旗风险台账（动态版）
附录A：DRL 清单（当前版本）
附录B：财务模型假设明细
附录C：参考文献与数据来源
```

> 附录须排列于全部正文章节之后，不得将附录插入正文章节之间。
