```

### 🆕 信息冲突处理原则（v1.1新增 · P0）

> 当评估过程中出现多来源信息不一致时，按以下原则处理：

1. **时间性原则**——同一事实的多个来源以最新日期/版本为准
2. **事实性核查**——当内部材料与公开可验证信息（如FDA官网、ClinicalTrials.gov、公司公告）存在事实性冲突（日期、数据、状态），以公开可验证来源为准，并在报告中以脚注标注差异
3. **判断性分歧**——当内部评估意见与外部核查结论存在判断性分歧（如市场规模估算），两者并列呈现，由评估方给出独立结论并注明依据
4. **目标市场数据优先**——中国目标市场一手数据的权威性高于海外文献推算（详见Gate 3数据使用原则）

### ▶ Gate 1：注册路径可行性（前提门）

**核心评估问题**：
- 海外数据是否满足NMPA（国家药品监督管理局）5.1类/6类申报要求（关键临床试验完整性、数据质量、种族差异）？
- 是否需要开展桥接试验（Bridging Study）或中国补充临床研究？
- 若需桥接，预计入组人数/试验周期/增量成本？
- NMPA是否已有类似品种受理先例？
- 优先审评/突破性治疗品种（BT）资格可能性？
- 🆕 **多层保护机制层级分析**：标的产品的知识产权与市场保护机制的层级关系和实际防护效力排序？

**🆕 多层保护机制层级分析要求（v1.1新增 · P1）**：

> 对标的产品的知识产权与市场保护机制，须完成层级分析：

| 分析步骤 | 要求 |
|---------|------|
| ① 列出所有保护机制 | 物质专利、制备方法专利、用途专利、数据保护期、市场独占期、行政保护等 |
| ② 标注各机制到期时间 | 从预计中国获批日期起算 |
| ③ 排列实际防护效力等级 | ★★★（主要壁垒）/ ★★（有效补充）/ ★（补充层） |
| ④ 识别主要壁垒vs补充层 | 主要壁垒=到期最晚且法律效力最强的机制；补充层=已完全落入主要壁垒保护区间内的机制 |

> **判定规则**：若某保护机制的到期时间早于另一更强机制的到期时间，则前者为"补充层"而非"主要防线"，报告中不得将补充层误判为主要壁垒。

**评估工具与信息来源**：
- NMPA公告系统（已受理/已批准同类品种检索）
- FDA/EMA审评报告（EPAR/SBA）中的种族亚组数据
- CDE（药品审评中心）技术指南（桥接试验相关）
- 内部注册专家访谈

**Gate 1结论模板**：
```

  1. CDE是否已与原研方进行过沟通记录
  2. 中国患者种族差异数据（如PK（药代动力学）桥接）
  3. 国内已批/在审竞品的注册路径参考案例

```

### ▶ Gate 3：市场规模与竞争格局（支付门）

---

#### 🆕 Gate 3 数据使用原则（v1.1新增 · P0）

> 本原则适用于Gate 3所有市场规模估算与竞争格局分析：

1. **中国目标市场一手调研数据优先于海外文献外推**——当中国本土调研数据（如深康/德镁/维盛市场部调研）可用时，以其作为基准情景输入
2. **海外流行病学/市场数据不可直接用于中国患者规模或市场规模估算**——美国、全球等海外数据仅可作为参考上限或方法论借鉴，不可作为基准情景输入
3. **地域差异说明**——当海外数据与中国本土数据存在差异时，报告中须以"地域差异说明"形式呈现，不得表述为"数据冲突"
4. **数据地域标注**——所有引用的流行病学数据须明确标注数据来源地域（如"美国PMC 2024"、"中国深康调研2026"），禁止不标注地域直接引用

**核心评估问题**：
- 中国目标适应症患者总数及诊断/治疗渗透率？

- 主要竞争者（已上市+在研）的市占率分布？
- 产品进入市场时间窗（是否已过赛道红海期）？

## 模块六：一票否决清单

> 以下任一条件命中，立即终止评估，无需进入后续Gate。

1. **注册路径完全封闭**：NMPA明确拒绝该品类5.1类/6类申报，或历史先例显示中国注册不可行
2. **桥接试验不可逾越**：需全新开展中国III期临床（耗时>5年），且原研方不愿承担费用或数据
3. **竞品已实质性封锁市场**：≥2个同类产品已进入NRDL（国家基本药物目录），且无差异化路径
4. **无法通过GMP（药品生产质量管理规范）检查**：原研生产基地在过去3年内有NMPA/FDA重大GMP缺陷警告
5. **IRR（内部收益率）不达标**：基准情景IRR（内部收益率）<15%且无敏感性情景可达标
6. **毛利率（毛利润/营业收入）<60%**：COGS（销售商品成本）结构导致毛利率（毛利润/营业收入）不可能超过60%
7. **独家性丧失**：原研方已就相同适应症在中国授权竞争方，或独家条款存在重大法律瑕疵
8. **峰值销售额（PeakSales）低于底线**：即便乐观情景下峰值销售额（PeakSales）仍低于对应适应症规模的乐观情景底线（大适应症<8亿/中适应症<5亿/罕见病<2.5亿）

## 模块九：输出规范

- **主报告格式**：HTML（深蓝 `#1a3a5c` 主色，白底，无灰色字，无元描述）
- **麦肯锡风格**：每章节顶部为结论句（Action Title），数据支撑置于正文
- **财务缩写标注**：IRR（内部收益率）、NPV（净现值）、CAGR（复合年均增长率）、COGS（销售商品成本）、DCF（折现现金流）、EBITDA（息税折旧摊销前利润）、PeakSales（峰值销售额）、RCT（随机对照试验）等均须括号标注中文全称
- **双币标注**：美元/欧元金额须同步标注人民币（注明汇率换算日期）
- **置信度等级**：高（主要结论有公开数据支撑）/ 中（有部分依据但存在待验证假设）/ 低（结论主要基于推断）

### 🆕 阶段标识呈现规则（v1.1新增 · P2）

> 规定报告中阶段归属的呈现方式：

| 场景 | 呈现规则 |
|------|---------|
| 首次整体评估（Gate 1~6全流程） | 正文章节标题不显示阶段标签；目录须以分组标签标明各章节所属阶段 |
| 分组标签色块 | 深蓝=阶段A（公开信息评估）/ 中蓝=阶段B（立项后深化评估）/ 棕色=综合决策层 |
| 信息来源标注 | 各分组标签下须注明该阶段所需信息来源类型 |
| 单阶段评估 | 仅阶段A或仅阶段B时，无需分组标签 |

```
