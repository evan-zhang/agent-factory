# TPR-20260327-001 交付说明

## 给 Skill 工厂的话

**项目背景**：

这是 **bp-reporting-templates** 的复刻项目。原 Skill 创建于 2026-03-25，根据 BP（Business Plan）文档自动生成月报/季报/半年报/年报填写规范。

原 Skill 只有一个 SKILL.md 文件和几个参考模板，没有代码实现，完全依赖 AI Agent 理解指令执行。这次复刻的目标是将其标准化为可执行的代码实现。

**核心功能**：
1. 解析用户指令（组织名/个人名 + 生成类型）
2. 从玄关开发者平台 API 获取 BP 数据
3. 并行生成多套填写规范（支持"四套"/"季报"/"月报和年报"等）
4. 审查机制（检查编码对齐、数字锚点、颜色预警）

**验收标准**：
- [ ] 能正确解析用户指令（"生成四套"/"只做季报"等）
- [ ] 能从 API 获取 BP 数据
- [ ] BP 编码正确填充到模板
- [ ] 衡量标准、数字锚点来自 BP 原文（不得编造）
- [ ] 颜色预警按规则判断（红>5%/黄3-5%/绿<3%）
- [ ] 输出格式为 Markdown

**注意事项**：
- API 4.13 (getMarkdown) 可能返回 401，需使用 4.4+4.5 Fallback
- 零容忍项：重大合规事故、BP签约率<100%、奖金发放delay
- 模板填充原则：**BP 原文怎么写，模板就怎么填**

## 核心逻辑说明

### 数据流

```
用户输入 → input_handler → [组织名, 个人名, 生成类型]
                ↓
         api_client → 从玄关平台获取 BP 数据
                ↓
           parser → 解析为 BPData 结构
                ↓
    template_manager → 加载目标模板
                ↓
           filler → 填充模板（重点是第2章）
                ↓
          reviewer → 审查（编码对齐、数字锚点、颜色预警）
                ↓
           输出 Markdown 文件
```

### 第2章填充规则（核心）

```markdown
### 2.{n} [{BP维度名称}]

**对标BP：** {P系列编码}（个人）/ {A系列编码}（组织）

**{时间维度}承接重点：**
- {引用BP原文的举措}

**当前状态：**
- 量化指标：{引用BP原文的数字锚点}
- 偏离判断：{红/黄/绿}

**是否偏离预期：**
- {是/否}，{偏离率}
```

### 颜色预警规则

| 指标类型 | 🔴 红 | 🟡 黄 | 🟢 绿 |
|---------|------|------|------|
| 财务类 | >5% | 3-5% | <3% |
| 得分类 | >10% | 5-10% | <5% |
| 节点类 | delay>2周 | delay 1-2周 | 正常 |

**零容忍项**（任何偏离即红）：
- 重大合规事故
- BP签约率<100%
- 奖金发放delay
