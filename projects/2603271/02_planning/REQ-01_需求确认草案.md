# REQ-01 需求确认草案（bp-reporting-templates 复刻）

项目：AF-20260327-001  
阶段：S2 REQUIREMENTS（待用户确认）

## 1. 目标
将 bp-reporting-templates 从“纯模板+指令理解”升级为“可执行代码化 Skill”，实现稳定、可复核、可测试交付。

## 2. 范围（In Scope）
1. 解析用户指令：识别 组织/个人 + 生成类型（四套/子集）
2. 周期前置选择：先列出可用 BP 周期并由用户选择 period_id
3. 数据获取：优先 API 拉取 BP 数据，失败时支持文件回退
4. 模板类型前置选择：先列出月报/季报/半年报/年报并由用户选择
5. 模板生成：按已选模板并行生成
4. 审查机制：编码对齐、数字锚点、衡量标准、颜色预警、零容忍项
5. 输出格式：Markdown

## 3. 非范围（Out of Scope）
1. 不直接代写业务内容（仅输出填写规范）
2. 不做 docx/pdf 输出（本期仅 Markdown）
3. 不做复杂多租户权限系统

## 4. 关键规则
1. BP 原文怎么写，模板就怎么填（禁止编造）
2. 生成前必须完成“双选择”：周期 + 模板类型
3. 零容忍项任一触发即红色预警
4. 审查失败需回报具体错误项，不允许静默通过

## 5. 默认实现决策（建议）
1. API 主路径 + 文件回退（双通道）
2. 并行按用户请求子集执行（不是固定四套全跑）
3. 审查失败自动重试 2 次，仍失败转人工确认

## 6. 验收标准（可测试）
- [ ] 能正确解析“生成四套 / 只做季报 / 月报和年报”等指令
- [ ] API 拉取成功时能生成目标模板
- [ ] API 401/失败时能自动走回退路径
- [ ] 输出中的编码/数字锚点均可追溯到 BP 原文
- [ ] 颜色预警符合阈值规则（红>5%/黄3-5%/绿<3%，含零容忍）
- [ ] 输出为 Markdown，结构章节完整

## 7. 用户确认结果（2026-03-27）
用户授权“由调度员代确认关键问题”，确认结果如下：
1) API双通道（主+回退）  
2) 本期只输出 Markdown  
3) 审查失败自动重试2次后人工确认  
4) 按请求子集并行生成
5) 生成前先列周期、再列模板类型供用户选择
