# 产品定位与信息传递

## 角色定位

产品定位是市场洞察的收官模块，负责将竞争分析、市场细分、政策环境的洞察转化为清晰、可传播的品牌主张。核心任务是回答"我们给谁、带来什么、为什么可信"。

## 核心任务

- 构建利益阶梯：从产品属性→利益→价值层层递进
- 确定RTB（Reason To Believe）：支撑利益主张的证据链
- 制定差异化定位声明（Positioning Statement）
- 构建核心信息框架（Key Message Platform）
- 指导各传播渠道的信息一致性

## 工具与方法

### 2.1 利益阶梯分析

**用途**：从产品特征到客户价值的逻辑链  
**方法论**：利益阶梯 workshop + 医学顾问委员会验证  
**应用**：将产品属性（what）转化为客户利益（what it does for me），再升华为个人/机构价值（why it matters）

### 2.2 RTB证据链构建

**用途**：为定位主张提供可信支撑  
**方法论**：临床数据映射 + 真实世界证据  
**应用**：将临床III期关键数据、亚组分析结果与定位利益对应，建立证据-利益映射表

### 2.3 定位测试

**用途**：验证定位主张的市场接受度  
**方法论**：概念测试（Concept Test）+ 竞争模拟  
**应用**：通过定量问卷和深度访谈，测试定位的独特性、相关性和可信度

## 推导过程

### 3.1 定位开发
1. 汇总竞争差异化优势（来自竞争分析）
2. 对齐客户关注点（来自市场细分）
3. 构建利益阶梯初稿
4. 建立RTB证据链

### 3.2 定位精炼
1. 内部医学顾问委员会评审
2. 外部KOL概念测试反馈
3. 竞品对比验证差异化
4. 迭代优化定位声明

### 3.3 信息框架构建
1. 制定核心信息平台（Key Message Platform）
2. 分解各受众（医生/KOL/药师/患者）的差异信息
3. 制定医学证据传播计划

## 关键发现

- **F1**：利益阶梯完整链条
- **F2**：差异化定位声明
- **F3**：RTB证据链完整性评估
- **F4**：定位测试结果（独特性/相关性/可信度评分）

| RTB强度 | 证据链完整度XX%，最强证据：XX | 高/中/低 |

## 待讨论点

1. 利益阶梯的逻辑是否经过KOL验证？
2. RTB证据是否足以支撑差异化主张（是否有头对头数据）？
3. 定位是否在竞争上具有可持续性（竞品短期能否追赶）？
4. 医学传播团队是否具备执行信息框架的能力？
