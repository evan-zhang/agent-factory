# BP原型模板生成规则

> 本文件定义 bp-prototype 如何从 BP规范 + BP系统API 逆向推理生成四套汇报模板。
> **核心原则：不依赖任何模板参考文件，从 BP 体系本身逆向推导。**

---

## 一、两个实时数据源

### 1.1 BP 规范（GitHub，实时拉取）
来源：https://raw.githubusercontent.com/xgjk/dev-guide/main/02.%E4%BA%A7%E5%93%81%E4%B8%9A%E5%8A%A1AI%E6%96%87%E6%A1%A3/BP/BP%E7%B3%BB%E7%BB%9F%E4%B8%9A%E5%8A%A1%E8%AF%B4%E6%98%8E.md

包含内容：
- BP 的 G-R-A 三层结构定义（Goal-Result-Action）
- 三层结构的边界规则（目标只写结果状态、成果必须可衡量、举措必须可追踪）
- 衡量标准五要素（指标+阈值+口径+数据源+频率）
- 康哲实施规则（组织BP vs 个人BP 的使用边界）
- BP 层级拆解深度（集团2层、中心/一级部门/核心员工3层）
- 承接规则（成果→目标、举措→目标）

### 1.2 BP 数据（BP系统 API，实时拉取）

| 接口 | 用途 |
|------|------|
| `GET /bp/group-goals/detail` | 集团目标（Goal）列表 |
| `GET /bp/goals/results?parentGoalId=X` | 关键成果（Result）列表 |
| `GET /bp/org/centers` | 中心组织列表 |
| `GET /bp/goals/actions?goalId=X` | 关键举措（Action）列表 |

**关键字段提取规则**：
- `goalId` → BP 编码（如 G-1.1）
- `goalName` → BP 名称
- `targetValue` → 数字锚点
- `completionRate` → 当前进度
- `parentGoalId` → 承接关系

## 二、核心推理：从 API 数据到模板章节

### 2.1 BP 结构 → 汇报章节映射（固定规则）

```
BP Goal（目标）        →  模板第2章"BP目标承接与对齐情况"
BP Result（关键成果）  →  模板第3章"核心结果与经营表现"
BP Action（关键举措）  →  模板第4章"关键举措推进情况"
```

### 2.2 六大维度的自动归纳

从 API 返回的 G-1 / G-2 / ... / G-18 中，按 BP系统操作手册 的定义归纳为六大维度：

| 维度编号 | 维度名称 | G系列来源 |
|---------|---------|----------|
| 2.1 | 经营与利润目标达成 | G-1.x |
| 2.2 | 产品力发展战略与价值闭环 | G-7.x / G-8.x |
| 2.3 | 管理方式向机制与数据驱动转型 | G-3.x |
| 2.4 | 组织与价值分配稳定机制建设 | G-5.x |
| 2.5 | 院外业务第二增长曲线推进 | G-9.x |
| 2.6 | 重大风险与合规事件全过程可控 | G-6.x / G-18.x |

**归纳逻辑**：API 返回的目标按 G 系列编码分组，组内编码数量最多的前6个维度，即为六大维度。

### 2.3 六大维度的填写字段（按时间跨度）

| 模板 | 字段数 | 字段内容 |
|------|--------|---------|
| 月报 | 3项 | 承接重点 + 当前状态 + 是否偏离预期 |
| 季报 | 5项 | 承接重点 + 阶段成果 + 与年度目标关系 + 当前状态 + 是否需要校准 |
| 半年报 | 4项 | 承接重点 + 阶段成果 + 与全年目标差距 + 下半程校准方向 |
| 年报 | 5项 | 年度目标 + 年度成果 + 达成度判断 + 价值贡献 + 下一年度承接建议 |

## 三、四套模板的章节对照

| 章节 | 月报 | 季报 | 半年报 | 年报 |
|------|------|------|--------|------|
| 第1章 综述 | 4项（当月判断） | 4项（季度评价） | 4项（半年度判断） | 4项（年度结论） |
| 第2章 BP承接 | 6维度×3字段 | 6维度×5字段 | 6维度×4字段 | 6维度×5字段 |
| 第3章 核心结果 | 按事项列举 | 按重点事项 | 按半年度成果 | 按年度成果 |
| 第4章 举措推进 | 本月推进 | 清单+完成/推进/延迟 | 闭环判断 | 有效/失效判断 |
| 第5章 问题分析 | 偏差+原因 | 偏差+原因+年度影响 | 偏差+结构性判断 | 未达成+根本原因 |
| 第6章 风险预警 | 下月风险 | 下季度风险 | 下半程风险 | 跨年风险 |
| 第7章 下期安排 | 下月3-5项 | 下季度目标+举措 | H2校准方案 | 下年度完整安排 |
| 第8章 协同事项 | 3类 | 3类 | 3类 | 3类 |

## 四、模板头部元信息生成

每次生成自动生成元信息：

```
- 所属项目：P001
- 节点编号：P001-T001
- 文档类型：TPL
- 文档编号：P001-T001-{TYPE}-TPL-{VERSION}
- 文档标题：{周期}报模板
- 文档状态：DRAFT
- 责任 Agent：bp-prototype
- 版本：{VERSION}
- 创建时间：{YYYY-MM-DD}
- 更新时间：{YYYY-MM-DD}
- 标签：模板类,{周期}报,BP对齐,DRAFT
- 说明：本模板由 bp-prototype 自动生成，数据来源：BP规范(GitHub实时) + BP系统API(实时)
```
