# bp-prototype

> BP 原型模板制造 Skill — BP规范 + BP系统API → 自动推理生成 | 支持查询发送模板

---

## 两种工作模式

| 模式 | 触发词 | 说明 |
|------|--------|------|
| **生成模式** | 生成BP模板 / 更新BP模板 | 从 BP规范 + API 数据推理生成新模板 |
| **查询模式** | 查询模板 / 获取模板 / 发给我模板 | 从 versions/ 目录查找并发送模板文件 |

---

## 模式一：生成模式

### 触发

用户说：**"生成BP模板"** 或 **"更新BP模板"**

### 数据源

| 来源 | 说明 |
|------|------|
| BP规范 | GitHub xgjk/dev-guide（每次从 GitHub 拉取最新） |
| BP数据 | BP系统 API 接口（通过 `BP_APP_KEY` 实时拉取） |

BP系统 API 接口：

| 接口 | 用途 |
|------|------|
| `GET /bp/group-goals/detail` | 集团目标（Goal）列表 |
| `GET /bp/goals/results?parentGoalId=X` | 关键成果（Result）列表 |
| `GET /bp/org/centers` | 中心组织列表 |
| `GET /bp/goals/actions?goalId=X` | 关键举措（Action）列表 |

### 交互流程

**第一步：AI 汇报数据状态**

```
✅ 数据已拉取完毕：

BP规范（GitHub）：
- G-R-A三层结构规范
- 康哲实施规则（6大维度定义）

BP系统数据（实时）：
- 集团目标：G-1 等，共 [X] 个
- 中心组织：产品中心/院外业务中心等，共 [X] 个
- G-R-A 承接关系：[X] 条

版本：v[X] → v[X+1]，确认生成？
```

**第二步：用户确认后，AI 推理生成**

AI 读取 BP规范 + API数据，逆向推理：
- 从 G-1 ~ G-18 自动归纳六大维度
- 按时间跨度（月/季/半年/年）推理章节结构差异
- 生成带填写说明的四套空白母版模板

**第三步：输出**

AI 在对话中输出四套完整模板，同时写入版本目录：

```
versions/YYYYMMDD-HHMMSS-v{X}/
├── P001-T001-MONTH-TPL-v{X}_月报模板.md
├── P001-T001-QUARTER-TPL-v{X}_季报模板.md
├── P001-T001-HALFYEAR-TPL-v{X}_半年报模板.md
└── P001-T001-YEAR-TPL-v{X}_年报模板.md
```

## 模式二：查询模式

### 触发

用户说：**"查询模板"** / **"发给我模板"** / **"获取模板"**

### 三种情况

**情况A：用户指定了具体模板**

用户："发给我月报v1" / "查一下季报模板v2"

→ AI 读取对应文件，用 `message` 工具以文件形式发送：
```
发送：
  action: send
  message: "月报模板 v1，请查收"
  media: /path/to/月报模板.md
```

**情况B：用户没有指定**

→ AI 先列出版本清单：

```
🔍 当前可选模板：

版本 v1（2026-04-01）：
- 月报模板 v1
- 季报模板 v1
- 半年报模板 v1
- 年报模板 v1

你想查看哪一个？比如："月报v1" 或 "全部发给我"。
```

**情况C：用户说"全部发给我"**

→ AI 批量发送该版本下的四个模板文件。

## 核心推理逻辑

### BP 结构 → 模板章节映射

```
BP Goal（目标）        →  第2章 BP目标承接与对齐情况
BP Result（关键成果）  →  第3章 核心结果与经营表现
BP Action（关键举措）  →  第4章 关键举措推进情况
```

### 六大维度（从 G-1 ~ G-18 自动归纳）

| 维度编号 | 维度名称 | G系列来源 |
|---------|---------|----------|
| 2.1 | 经营与利润目标达成 | G-1.x |
| 2.2 | 产品力发展战略与价值闭环 | G-7.x / G-8.x |
| 2.3 | 管理方式向机制与数据驱动转型 | G-3.x |
| 2.4 | 组织与价值分配稳定机制建设 | G-5.x |
| 2.5 | 院外业务第二增长曲线推进 | G-9.x |
| 2.6 | 重大风险与合规事件全过程可控 | G-6.x / G-18.x |

### 四套模板核心差异

| 维度 | 月报 | 季报 | 半年报 | 年报 |
|------|------|------|--------|------|
| 时间 | 1个月 | 3个月 | 6个月 | 12个月 |
| 功能 | 过程监控 | 趋势复盘 | 中程校准 | 全年传承 |
| 2.X每维度字段 | 3项 | 5项 | 4项 | 5项 |
| 趋势分析 | 无 | 必须 | 必须 | 必须 |
| 中程校准 | 无 | 无 | 必须 | 无 |
| 独有内容 | — | 下季度安排 | H2校准 | 下年建议+价值贡献 |

## 版本管理

- 每次生成自动递增版本号
- 历史版本全部存档，不删除
- 版本号格式：`v{数字}`（如 v1, v2）
- 目录格式：`versions/YYYYMMDD-HHMMSS-v{X}/`
