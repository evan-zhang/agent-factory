# DISCOVERY（S1完成）

项目：AF-20260327-001  
来源：TPR-20260327-001 交付包（已接收）

## 1) 项目目标
将现有 `bp-reporting-templates` 从“仅 SKILL.md + 参考模板”的弱实现，升级为“可执行代码化 Skill”：
- 支持输入解析（组织名/个人名 + 生成类型）
- 支持 API 拉取 BP 数据（含 fallback）
- 支持并行生成（月报/季报/半年报/年报）
- 支持审查机制（编码对齐/数字锚点/预警规则）

## 2) 输入输出契约
### 输入
- 用户指令：如“生成四套 / 只做季报 / 月报和年报”
- BP 数据来源：API 获取或用户文件

### 输出
- 4类填写规范（Markdown）：月报、季报、半年报、年报

## 3) 关键业务规则
- 模板填充原则：**BP 原文怎么写，模板就怎么填**
- 禁止编造：编码、衡量标准、数字锚点必须来自 BP 原文
- 颜色预警：财务/得分/节点阈值 + 零容忍项（合规事故、签约率<100%、奖金延迟）

## 4) 方案形态（交付建议）
建议采用代码模块化结构：
- `input_handler.py` 输入解析
- `api_client.py` API调用
- `parser.py` BP解析
- `template_manager.py` 模板管理
- `filler.py` 模板填充
- `reviewer.py` 审查
- `main.py` 主入口 + `tests/test_all.py`

## 5) 风险与边界
- API鉴权与稳定性（401 fallback）是主风险
- 输入格式非标准时需容错策略
- 当前范围聚焦“生成规范模板”，不包含“自动代写业务内容”
