## 2. 功能拆解

### 2.1 输入解析

**需要识别的信息**：
1. BP 编码体系（P 系列 / A 系列）
2. BP 目标列表（每个目标包含：编码、名称、衡量标准、数字锚点、举措）
3. 用户需求（生成哪几套模板）

**输入格式**：
- Markdown 文件（BP 原文）
- 用户指令（"生成四套" / "只做季报" / "月报和年报"）

### 2.2 模板填充

**每个模板的章节结构**（8 章）：

| 章节 | 月报 | 季报 | 半年报 | 年报 |
|------|------|------|--------|------|
| 1. 汇报综述 | ✓ | ✓ | ✓ | ✓ |
| 2. BP目标承接 | ✓ | ✓ | ✓ | ✓ |
| 3. 核心结果 | ✓ | ✓ | ✓ | ✓ |
| 4. 关键举措 | ✓ | ✓ | ✓ | ✓ |
| 5. 问题偏差 | ✓ | ✓ | ✓ | ✓ |
| 6. 风险预警 | ✓ | ✓ | ✓ | ✓ |
| 7. 下期安排 | ✓ | ✓ | ✓ | ✓ |
| 8. 需决策事项 | ✓ | ✓ | ✓ | ✓ |

**填充规则**：
- BP 原文怎么写，模板就怎么填
- 数字/编码必须来自原文，不得编造
- 颜色预警按规则判断

### 2.3 颜色预警规则

| 指标类型 | 🔴 红 | 🟡 黄 | 🟢 绿 |
|---------|------|------|------|
| 财务类 | >5% | 3-5% | <3% |
| 得分类 | >10% | 5-10% | <5% |
| 节点类 | delay>2周 | delay 1-2周 | 正常 |

**零容忍项**（任何偏离即红）：
- 重大合规事故
- BP签约率<100%
- 奖金发放delay

### 2.4 审查机制

**Reviewer Agent 检查项**：
- BP 编码对齐检查
- 数字锚点检查
- 衡量标准检查
- 示例质量检查
- 颜色预警检查

## 3. 技术架构分析

### 3.1 当前实现方式

**无代码实现**：当前 Skill 只有 SKILL.md + 参考文件，依赖 AI Agent 理解指令并执行。

**优点**：
- 灵活，AI 可处理非标准输入
- 无需维护代码

**缺点**：
- 依赖 AI 理解能力，可能不稳定
- 无法处理大量 BP 数据
- 无自动化流程

### 3.2 建议的代码实现架构

```
bp-reporting-templates/
├── SKILL.md                    # Skill 定义
├── references/                 # 参考文件
│   ├── 基础模板_月报.md
│   ├── 基础模板_季报.md
│   ├── 基础模板_半年报.md
│   ├── 基础模板_年报.md
│   ├── 通用填写规范.md
│   └── BP编码速查表.md
├── scripts/
│   ├── main.py                 # 主入口
│   ├── parser.py               # BP 解析器
│   ├── filler.py               # 模板填充器
│   ├── reviewer.py             # 审查器
│   └── utils.py                # 工具函数
├── tests/
│   └── test_all.py
└── requirements.txt
```

### 3.3 核心数据结构

```python
# BP 目标结构
```

## 4. 需求边界

### 4.1 功能范围

**包含**：
- 解析 BP 文件（Markdown 格式）
- 生成 4 种时间维度的填写规范
- 自动填充 BP 编码、衡量标准、数字锚点
- 颜色预警判断
- 审查机制

**不包含**：
- 从 API 拉取 BP 数据（由 bp-data-fetcher 负责）
- 实际填写汇报内容（只生成规范模板）
- 多人并行处理

### 4.2 输入格式约定

**BP 文件格式**（来自 bp-data-fetcher 输出）：
```markdown
# {组织/个人名称} BP

- 周期：2026年度计划BP
- 类型：{org/personal}
- 编码：{levelNumber}

---

## 目标一：{名称}

**编码：** {id}
**状态：** {status}
**计划时间：** {startTime} ~ {endTime}

### 关键成果1.1：{名称}

**衡量标准：** {measureStandard}

#### 举措1.1.1：{名称}

**承接人：** {ownerName}
```
