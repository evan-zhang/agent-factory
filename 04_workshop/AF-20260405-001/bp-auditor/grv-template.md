# GRV：BP 两级联动审计框架

> 版本：v4.0
> 修正：加入诊断深度——每个问题必须分析根因、给出多方案、推荐方案、明确行动责任人
> 基于：BP 评估分析规范 v1.0

---

## 核心原则

**BP 结构认知**：
- BP（目标层）：无举措，落地依赖 KR
- KR（成果层）：有 measureStandard，是审计重点
- 举措（Initiative）：无 measureStandard，不检查此项

**报告目标**：不只是发现问题，而是分析问题根因、给出解决方案、明确行动责任人。

**特殊约束：顶级目标（无上游 BP）的处理**：
- G 层（或顶级）BP：跳过"对齐性"检查（无上游，不需要对齐）
- 非顶级 BP：正常检查对齐性
- 判断方法：若 upstream 为空且本 BP 为 G 层目标，标记为"顶级目标，无需上游对齐"，不扣分

## BP 节点评估模板

### N.1 目标层评估

| 维度 | 评估内容 | 标准 |
|------|---------|------|
| 清晰性 | 目标描述是否明确、无歧义 | 有一句话能说清楚 |
| 对齐性 | 目标是否与上游对齐 | 有明确的上游对应 |
| 完整性 | 目标是否覆盖所有核心业务方向 | 所有核心方向都有覆盖 |

### N.2 KR 层评估

| 维度 | 评估内容 | 标准 |
|------|---------|------|
| 充分性 | KR 集合是否足够支撑 BP 目标 | 所有目标方向都有 KR |
| 独立性 | KR 之间是否重叠或打架 | 边界清晰、不重叠 |
| SMART-S | measureStandard 是否定义业务边界 | 如"AI产品"有明确定义 |
| SMART-M | measureStandard 是否含数字指标 | 含数字 = PASS；null = P0 |
| SMART-A | 指标是否有依据/可达成 | 有推导依据 = PASS；无 = P1 |
| 匹配性 | 责任人是否与 KR 能力要求匹配 | 责任人能真正推动 |
| 支撑性 | KR 是否有举措或下游承接 | 有落地路径 |

### N.3 举措层评估

| 维度 | 评估内容 | 标准 |
|------|---------|------|
| 完整性 | 举措集合是否覆盖 KR 的所有方面 | 无遗漏的关键路径 |
| 科学性 | 举措的设计是否科学、合理 | 有逻辑支撑 |
| 标准性 | 举措是否有明确的执行标准 | 有交付物/完成定义 |
| 可执行性 | 举措是否有承接人/时间/资源 | 人、时间、工具都明确 |
| 有效性 | 举措完成后是否能支撑 KR 实现 | 因果关系清晰 |
| 延期风险 | 举措截止日期是否已过期 | 未过期 = PASS；过期>30天 = P0 |

## 两级联动审计流程

```
Step 1: fetch.py 获取根 BP 数据 → /tmp/bp-{CODE}.json
Step 2: 对根 BP 执行 Level 1 评估（Goal + KR + 举措）
Step 3: 识别根 BP 所有 KR 的 downstream（下游 BP 列表）
Step 4: 对每个下游 BP 执行 Level 2 评估（Goal + KR + 举措）
Step 5: 汇总所有 P0/P1/P2 问题
Step 6: 生成两级联动审计报告
```

## 诊断深度要求（核心）

每个发现的问题必须包含以下 5 个要素，缺一不可：

| 要素 | 说明 | 示例 |
|------|------|------|
| **问题描述** | 具体是什么问题 | A3-3.1 目标值 = 基准值（80%=80%），无变化要求 |
| **根因分析** | 为什么发生 | 制定 KR 时对"降占比"理解有误，将基准值当成了目标值 |
| **多方案** | 有哪些解决路径（至少2个） | 方案A：重新设定目标值；方案B：先定逻辑再定目标 |
| **建议方案** | 明确推荐哪个 | 建议方案B（先定逻辑再定目标） |
| **行动责任人** | 谁来做 | 李金金+张成鹏 |

## 问题定级标准

### P0（必须立即整改）
- KR 的 measureStandard 为 null
- 举措过期 >30 天
- 目标设计失效（基准 = 目标）
- 承接 BP 口径与上游不一致
- 责任人专业严重错位

### P1（尽快优化）
- 目标描述模糊
- KR 独立性无法判断
- KR 措施无支撑（无举措无下游）
- 举措标准不明确
- 举措与 KR 无因果链
- 责任人专业匹配存疑

### P2（后续迭代）
- 表达优化建议

## 报告输出结构

```
报告结构：
├── 诊断概览（P0/P1 数量、可实现性）
├── 第一级：根 BP 自身诊断
│   ├── Goal 层评估（判定 + 问题）
│   ├── KR 层诊断（判定 + 问题，每个 KR 单独一节）
│   └── 举措层评估（如有）
│   └── 第一级问题汇总（带根因+方案+责任人）
├── 第二级：下级承接 BP 深度诊断（每个下游 BP）
│   ├── 目标层评估
│   ├── KR 层诊断（每个 KR 单独分析 measureStandard + SMART + 因果链）
│   ├── 举措层诊断（每个举措单独一行：状态/过期/责任人）
│   ├── 承接有效性分析（举措→KR→BP 的因果链是否成立）
│   └── 问题汇总（带根因+方案+责任人）
├── 问题汇总总表（所有问题，带 5 要素）
└── 总体结论
    ├── 可实现性评估（定性判断）
    ├── 整改路线图（分三周安排）
    └── 最紧迫的 3 个问题
```
