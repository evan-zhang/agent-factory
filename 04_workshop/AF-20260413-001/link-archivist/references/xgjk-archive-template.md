# 档案生命周期管理

## 编号规则

- **格式**：`K-YYMMDD-NNN`，如 `K-260406-001`
- **YYMMDD**：建档日期（2位年+2位月+2位日）
- **NNN**：当日序号（001-999），每日从 001 开始
- **生成方式**：搜索知识库中当日已有编号，取最大 +1
- **排序**：编号天然按时间排序，一眼可看出建档日期和顺序

## 目录结构

```
个人知识库（根目录）
├── 知识档案/                    ← 主目录
│   ├── K-260406-001/
│   │   └── 调研报告.md
│   ├── K-260407-001/
│   │   └── ...
│   └── ...
└── （其他内容）
```

所有档案创建在 `知识档案/` 主目录下，folderName 格式：`知识档案/K-YYMMDD-NNN`

## YAML 元信息头

```yaml
---
archive: K-260406-001
source: https://example.com
source_type: 头条文章
platform: 今日头条
published_at: 2026-04-06
created_at: 2026-04-06T22:00:00+08:00
tags: [AI, 医疗]
---
```

## 生命周期

1. **创建**：Story1/Story2 → 分配编号 → 创建文件夹 → 存入首篇文档
2. **追加**：Story5 → 删除旧文件 → 重新上传合并内容（同一 folderName）
3. **永不结案**：档案持续开放，可无限追加
4. **编号不变**：追加不产生新编号；拆分新档案附标签 `原档案:K-原编号`

## 相关度算法（追加推荐）

- 基于关键词重叠 + 向量相似度
- 阈值 ≥ 0.6 才推荐追加
- 推荐时展示理由和相关度分数
